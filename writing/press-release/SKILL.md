# press-release

## 功能说明
创作专业的新闻稿和公关稿，包括产品发布、融资公布、活动宣传、危机公关等。

---

## 假设声明（Assumptions）

1. **事件真实**：用户描述的事件是真实发生的
2. **信息可公开**：用户提供的信息属于可对外公开的内容
3. **媒体渠道**：目标媒体是正式新闻媒体（不是自媒体）
4. **适用场景**：企业新闻稿、融资公告、产品发布、活动报道、危机声明
5. **不支持**：虚假信息、恶意攻击竞品的"新闻稿"

## 详细工作流程（Detailed Workflow）

### 阶段一：信息收集
1. **确认新闻要素**
   - What（什么事件）
   - When（何时）
   - Where（何地）
   - Who（谁）
   - Why（为什么）
   - How（如何发生）

2. **核实信息准确性**
   - 关键数据核实
   - 人名/公司名准确
   - 时间节点确认

### 阶段二：价值判断
3. **评估新闻价值**
   - 时效性：是否是新消息
   - 重要性：影响范围多大
   - 显著性：是否涉及名人/大公司
   - 接近性：与目标读者有何关联
   - 趣味性：是否有故事性

4. **确定报道角度**
   - 从哪个角度切入最能吸引读者
   - 如何与读者利益关联

### 阶段三：结构撰写
5. **撰写新闻稿**
   - 标题：简洁有力，含关键信息
   - 导语：第一段包含核心5W1H
   - 主体：按重要性/时间/逻辑排序
   - 背景：补充相关背景信息
   - 结尾：总结或展望

6. **审核优化**
   - 信息准确无误
   - 语言符合新闻规范
   - 篇幅适中（500-1000字）

### 检查点
- [ ] 标题吸引记者/编辑
- [ ] 导语包含核心信息
- [ ] 引用权威人士的话
- [ ] 数据有来源支撑
- [ ] 联系方式完整

## 边界情况（Edge Cases）

### 情况1：危机公关稿
**问题**：负面事件需要回应
**处理方式**：
- 坦诚面对，不回避问题
- 明确说明事实和解决方案
- 态度诚恳，不推卸责任
- 控制篇幅，不宜过长

### 情况2：信息不足
**问题**：用户只能提供有限信息
**处理方式**：
- 基于可用信息构建合理框架
- 对缺失信息使用"待公布"或"XXX表示"
- 建议用户补充关键信息

### 情况3：多公告合并
**问题**：如融资+产品发布同时发生
**处理方式**：
- 确定主次，分开发布或合并
- 建议主要公告优先
- 避免信息堆砌

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 数据夸大 | 引用数据无法核实 | 标注数据来源或降级表达 |
| 侵权风险 | 提及未授权第三方 | 删除或泛化处理 |
| 法规风险 | 可能违反广告法/证券法 | 调整措辞 |
| 敏感内容 | 涉及政治/争议话题 | 建议删除或淡化 |

### 失败案例及修正

**原始问题新闻稿**：
```
XXX公司最近推出了一个新产品，这个产品非常好，非常棒，是我们公司最优秀的产品，比市面上所有的产品都好，欢迎大家购买。
```

**问题分析**：
1. 缺乏具体信息 - 什么产品？什么功能？
2. 主观色彩过浓 - "非常棒"不是新闻语言
3. 缺少权威声音 - 没有引用/数据
4. 像广告而非新闻稿

**修正后**：
```
【新闻标题】XXX公司发布XX新品，专注解决年轻人"续航焦虑"

【新闻导语】
XXX公司今日正式发布XX产品，主打"7天续航"卖点，售价1299元起。该产品针对年轻人日常使用场景优化，宣称可实现"一周一充"。

【正文主体】
"市面上大多数智能手表续航只有1-2天，用户频繁充电反而带来焦虑。"XXX产品总监在发布会上表示，"我们希望通过技术突破，真正解决这个痛点。"

据悉，XXX Watch采用了全新低功耗芯片和系统优化方案，日常使用续航可达7天，关闭部分功能后续航最长可达14天。
```

## FAQ（常见问题）

**Q1：新闻稿一般多长合适？**
A：标准新闻稿500-800字（中文），重要事件可扩展至1000-1500字。

**Q2：新闻稿和公关稿有什么区别？**
A：
- 新闻稿：更客观，接近新闻报道，以传递信息为主
- 公关稿：可以更软性，融入品牌调性

**Q3：如何提高媒体采用率？**
A：
- 提供独家信息/数据
- 有新闻性（时间节点、首发等）
- 结构清晰，方便编辑使用
- 联系方式畅通

**Q4：可以帮写英文新闻稿吗？**
A：可以。请提供中文信息，我会按英文新闻稿规范撰写。

**Q5：新闻稿中引用的话是真的吗？**
A：是的，我们使用用户提供的信息。如需虚构引用，请明确说明，我会在输出中标注"[注：引用为示例，请替换为实际引语]"
