# report-writer

## 功能说明
帮助撰写各类分析报告，包括年度报告、调研报告、可行性研究报告等。

---

## 假设声明（Assumptions）

1. **数据可获取**：用户能提供或授权使用报告所需数据
2. **报告目的明确**：用户清楚报告的目标和受众
3. **信息基本完整**：用户能提供核心信息和素材
4. **适用场景**：经营分析、市场调研、可行性研究、项目复盘
5. **不支持**：高度专业化研究（如医学临床报告）

## 详细工作流程（Detailed Workflow）

### 阶段一：需求分析
1. **确认报告规格**
   - 报告类型和目的
   - 目标读者
   - 报告周期

2. **收集数据和素材**
   - 用户提供的核心数据
   - 需要补充的背景信息
   - 数据的可信度评估

### 阶段二：结构设计
3. **规划报告结构**
   - 确定章节划分
   - 规划各部分重点
   - 设计信息呈现顺序

4. **制定分析框架**
   - 选用分析模型
   - 确定对比维度
   - 设计图表类型

### 阶段三：内容撰写
5. **撰写各部分内容**
   - 执行摘要先写
   - 数据分析
   - 结论与建议

6. **制作数据图表**
   - 表格设计
   - 图表类型选择
   - 数据可视化描述

### 检查点
- [ ] 数据准确无误
- [ ] 分析逻辑清晰
- [ ] 结论有数据支撑
- [ ] 建议可执行

## 边界情况（Edge Cases）

### 情况1：数据不足
**问题**：用户无法提供完整数据
**处理方式**：
- 使用行业基准数据作为参考
- 明确标注"[待补充]"
- 提供数据收集模板

### 情况2：负面结果
**问题**：分析结果显示负面结果
**处理方式**：
- 客观呈现数据
- 深入分析原因
- 提出建设性建议
- 避免粉饰

### 情况3：多部门综合报告
**问题**：需要整合多个部门数据
**处理方式**：
- 提供统一模板
- 保持口径一致
- 突出跨部门洞察

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 数据不一致 | 多处数据矛盾 | 核实后统一 |
| 逻辑跳跃 | 结论与数据脱节 | 补充论证 |
| 建议不可行 | 建议脱离实际 | 调整为可执行 |
| 格式混乱 | 结构不清晰 | 重新组织 |

### 失败案例及修正

**原始问题**：
```
帮我写个报告，说一下我们公司去年业绩很好。
```

**问题分析**：
1. 目的不明确 - 好到什么程度？
2. 对象不清楚 - 给谁看？
3. 没有数据支撑
4. 缺乏分析和建议

**修正后**：
```
【需求确认】
为了撰写一份有价值的报告，请补充以下信息：

## 必填信息
1. **报告类型**：经营分析/年报/项目复盘？
2. **目标读者**：董事会/管理层/全员？
3. **核心数据**：
   - 营收/利润是多少？
   - 同比/环比变化？
   - 各业务线表现？
4. **报告目的**：总结成果/发现问题/规划未来？

## 可选信息
- 行业背景
- 竞争情况
- 下一步计划
```

## FAQ（常见问题）

**Q1：报告一般多长？**
A：视类型和需求：
- 经营分析报告：10-20页
- 市场调研报告：20-40页
- 可行性报告：15-30页
- 复盘报告：5-15页

**Q2：可以帮我做数据分析吗？**
A：可以。如果您有原始数据，我可以帮助分析并撰写报告。

**Q3：可以生成图表吗？**
A：我无法直接生成图表，但我会：
- 提供图表类型建议
- 描述图表内容
- 提供数据表格

**Q4：数据保密吗？**
A：是的，所有提供的数据仅用于撰写报告，不对外共享。

**Q5：可以定期生成报告吗？**
A：可以。建立模板后，可定期更新数据生成报告。

# 2024年上半年经营分析报告

## 1. 执行摘要

2024年上半年，公司整体业绩稳步增长，实现营收2.3亿元，同比增长35%，完成年度目标的52%。

**核心亮点**
- 核心产品线A营收增长58%，超预期
- 毛利率提升至68%（去年同期62%）
- 获得新一轮融资5000万元

**主要挑战**
- 产品线B营收下滑12%
- 客户流失率上升至8%

---

## 2. 整体业绩回顾

### 2.1 财务表现

| 指标 | 2024H1 | 2023H1 | 同比 |
|------|--------|--------|------|
| 营收 | 2.3亿 | 1.7亿 | +35% |
| 毛利 | 1.56亿 | 1.05亿 | +49% |
| 毛利率 | 68% | 62% | +6pct |
| 净利 | -1200万 | -2000万 | 减亏40% |

### 2.2 运营数据

- 活跃用户数：45万（+42% YoY）
- 客单价：510元（+15% YoY）
- 客户流失率：8%（去年同期5%）

---

## 3. 各业务线表现

### 3.1 产品线A（核心产品）
**表现**：超预期
- 营收1.6亿，占总营收70%
- 新增企业客户120家
- 复购率达到75%

### 3.2 产品线B（新业务）
**表现**：低于预期
- 营收0.4亿，占总营收17%
- 客户流失率高达15%
- 需评估是否继续投入

---

## 4. 关键问题与挑战

### 4.1 产品线B问题
经分析，产品线B下滑主要原因：
1. 定价策略失误：高于市场均价20%
2. 竞品压力：2家新进入者以低价抢占市场
3. 产品体验：客户反馈功能冗余、学习成本高

### 4.2 客户流失率上升
深入分析发现，流失主要集中在：
- 中小企业客户（流失率12%）
- 产品线B客户（流失率15%）

---

## 5. 下半年展望与建议

### 5.1 目标
- 下半年营收目标：2.5亿（+30% YoY）
- 全年目标达成率：55-58%

### 5.2 关键举措

1. **产品线B重组**
   - 建议降低定价15%
   - 简化产品功能，聚焦核心场景
   - 评估是否转型为解决方案

2. **提升客户留存**
   - 建立客户健康度监控体系
   - 推出客户成长期计划
   - 目标：将流失率降至5%

3. **探索新增长点**
   - 下半年推出产品线C（高端市场）
   - 目标：通过产品线C获取20%新客户
