# resume-optimize

## 功能说明
帮助用户诊断和优化简历，包括内容结构、语言表达、关键词优化、排版格式等。

---

## 假设声明（Assumptions）

1. **信息真实**：用户提供的简历信息是真实准确的
2. **目标岗位已知**：用户有明确的求职方向
3. **经验属实**：工作经历和项目经历基于真实经历
4. **适用场景**：社招、校招、转行、简历优化
5. **不支持**：完全造假、虚构不存在的经历

## 详细工作流程（Detailed Workflow）

### 阶段一：简历诊断
1. **信息收集**
   - 了解目标岗位
   - 收集简历全文
   - 了解工作年限和背景

2. **全面分析**
   - 结构评估
   - 内容评估
   - 措辞评估
   - 关键词评估
   - ATS友好度评估

### 阶段二：优化建议
3. **问题归类**
   - 致命问题（如：关键信息缺失）
   - 重要问题（如：措辞不够有力）
   - 细节问题（如：格式不统一）

4. **制定优化方案**
   - 结构调整建议
   - 内容补充建议
   - 措辞优化建议
   - 关键词植入建议

### 阶段三：示范修改
5. **修改示范**
   - 提供修改前/后对比
   - 说明修改理由
   - 给出具体可执行的表达

6. **提供完整框架**
   - 优化后的简历模板
   - 各模块写作要点
   - 注意事项提醒

### 检查点
- [ ] 核心卖点突出
- [ ] 成果用数据量化
- [ ] 关键词匹配岗位
- [ ] ATS可识别

## 边界情况（Edge Cases）

### 情况1：转行/跨行简历
**问题**：之前经历与目标岗位不相关
**处理方式**：
- 挖掘可迁移技能
- 强调通用能力
- 重新包装经历

### 情况2：应届生简历
**问题**：缺乏工作经历
**处理方式**：
- 突出实习/项目经历
- 强调教育背景相关课程
- 展示可迁移软技能

### 情况3：空窗期解释
**问题**：简历有较长空白期
**处理方式**：
- 提供合理解释方式
- 避免简历明显空白
- 强调期间的学习/准备

### 情况4：多年工作经历
**问题**：经历丰富需要取舍
**处理方式**：
- 聚焦与目标岗位相关的经历
- 按时间倒序排列
- 控制篇幅

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 信息虚假 | 明显不合逻辑的数据 | 标注风险，建议修改 |
| 过度夸大 | 成果描述明显超出合理范围 | 提供更客观的表达 |
| 关键词堆砌 | 硬植入岗位JD关键词 | 提供更自然的表达方式 |
| 格式混乱 | 排版严重影响可读性 | 提供模板建议 |

## 质量验证清单（Quality Checklist）

### 内容质量检查
- [ ] 个人信息简洁完整（姓名、联系方式）
- [ ] 工作经历按时间倒序
- [ ] 每条经历有具体成果
- [ ] 成果用数据量化

### 匹配度检查
- [ ] 关键词与目标岗位匹配
- [ ] 技能要求覆盖核心需求
- [ ] 项目经历体现岗位相关能力

### 可读性检查
- [ ] 排版整洁统一
- [ ] 语言精炼无废话
- [ ] 重要信息一眼可见
- [ ] PDF格式文件名规范

### ATS检查
- [ ] 关键词包含岗位JD中的核心词
- [ ] 避免特殊符号/图表
- [ ] 标准化工作经历描述
- [ ] 教育经历格式规范

## FAQ（常见问题）

**Q1：简历一般多长合适？**
A：视经历多少而定：
- 应届生：1页
- 3年以内：1-2页
- 3-5年：2页
- 5年以上：2-3页

**Q2：简历格式重要吗？**
A：非常重要。第一印象决定是否进入面试：
- 排版整洁统一
- 重点信息突出
- 避免花哨装饰
- 建议PDF格式

**Q3：简历需要针对每个岗位调整吗？**
A：建议分行业/岗位准备2-3个版本，针对性投递效果更好。

**Q4：简历写"精通"某技能会被质疑吗？**
A：建议诚实描述。区分：
- 精通：可独立解决复杂问题
- 熟悉：了解原理，有实践经验
- 了解：知道基础，能上手使用

**Q5：简历需要英文版吗？**
A：视情况：
- 外资/海外岗位：需要
- 国内岗位：一般不需要
- 技术类：可准备英文以备不时之需

## 示例
**输入**: 帮我看看这份简历有什么问题
**output**:
【目标岗位】产品经理（互联网）
【简历诊断】
总体评分：★★★☆☆（中等，有较大提升空间）

【问题分析】

1. 【结构问题】
   - 教育背景放在最前面，应该突出工作经历
   - 项目经历和工作经历混在一起，不清晰
   - 缺少数据化成果展示

2. 【内容问题】
   - 工作职责描述过于平淡，缺乏亮点
   - 缺少用户增长、收入增长等核心指标
   - 项目描述缺乏背景和结果

3. 【措辞问题】
   - 使用"负责"、"参与"等弱动词
   - 没有用数字量化成果
   - 描述过于笼统

4. 【关键词问题】
   - 缺少"用户研究"、"需求分析"、"PRD"等岗位关键词
   - 未体现数据分析能力

【修改建议】

1. 将工作经历放在最前面
2. 每条工作经历用"动词+任务+结果"的格式
3. 量化成果：用户、收入、效率提升的百分比
4. 添加岗位相关关键词

【修改示范】

原文：
"负责用户增长工作，通过各种方法提升用户活跃度"

修改后：
"主导用户增长策略，0-1搭建用户激励体系，通过A/B测试优化Push策略，DAU提升35%，用户留存率提升22%"

【优化后的简历框架】
1. 个人信息（简洁）
2. 工作经历（按时间倒序，3-5条）
3. 项目经历（2-3个核心项目）
4. 教育背景
5. 技能证书（精选）
