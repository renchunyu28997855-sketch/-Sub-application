# email-writer

## 功能说明
创作各类商务邮件，包括开发信、跟进邮件、邀请函、道歉信等，确保邮件专业、有礼、有效。

---

## 假设声明（Assumptions）

1. **沟通目的明确**：用户清楚写邮件的目的和期望结果
2. **收件人身份已知**：用户了解收件人的身份和与自己的关系
3. **基础信息完备**：用户提供必要的背景信息
4. **适用场景**：中英文商务邮件、工作沟通、客户维护
5. **不支持**：私人情感邮件、法律合同、正式法律文件

## 详细工作流程（Detailed Workflow）

### 阶段一：需求分析
1. **确认邮件类型和目的**
   - 开发新客户/维护老客户/处理问题
   - 明确期望的回复或行动

2. **分析收件人画像**
   - 职位和决策权
   - 与发件人的关系
   - 可能的关注点

### 阶段二：内容规划
3. **梳理邮件要点**
   - 必须传达的信息
   - 希望传达的次要信息
   - 需要避免的信息

4. **设计邮件结构**
   - 开头：背景/问候
   - 主体：核心信息
   - 结尾：行动号召/期待回复

### 阶段三：撰写优化
5. **撰写邮件正文**
   - 语言风格匹配收件人
   - 重点信息突出
   - 语言简洁明了

6. **检查与优化**
   - 语法和拼写检查
   - 信息完整性核对
   - 格式规范检查

### 检查点
- [ ] 邮件目的清晰
- [ ] 语言得体适度
- [ ] 重点信息突出
- [ ] 行动指引明确
- [ ] 时间节点清晰

## 边界情况（Edge Cases）

### 情况1：收件人职位较高
**问题**：如写给CEO或高层领导
**处理方式**：
- 语言更加简洁正式
- 开门见山，直击要点
- 控制邮件长度

### 情况2：情绪化邮件需求
**问题**：如客户投诉后需要回复
**处理方式**：
- 帮助用户以专业态度处理
- 提供化解矛盾的表达方式
- 避免激化冲突的表达

### 情况3：英文邮件撰写
**问题**：需要写英文商务邮件
**处理方式**：
- 按商务英文规范撰写
- 注意中英文表达差异
- 提供中英双语对照（如需要）

### 情况4：跟进邮件
**问题**：之前已发过邮件未收到回复
**处理方式**：
- 设计合适的跟进频率（3-7天）
- 提供多个跟进版本
- 包含"是否感兴趣"的便捷回复选项

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 语气不当 | 与收件人关系/场景不匹配 | 调整为更合适的语气 |
| 信息遗漏 | 缺少关键背景信息 | 询问补充或标注"[背景]" |
| 过长冗余 | 超过目标读者阅读耐心 | 精简至核心信息 |
| 格式混乱 | 缺少段落/重点不突出 | 重新组织结构 |

- [ ] 重点信息有高亮/加粗
- [ ] 结尾有署名和联系方式

- [ ] 无敏感词汇
- [ ] 无虚假承诺
- [ ] 个人信息准确

### 失败案例及修正

**原始问题邮件**：
```
王总你好：

我是XX公司的小李，我们公司有个产品特别好，很多人都在用，效果非常好。如果你有兴趣的话可以联系我，我的电话是138xxxx。

此致
敬礼
```

**问题分析**：
1. 目的不明确 - 不知道想让对方做什么
2. 自说自话 - 没有说明对方能获得什么
3. 联系方式缺失或混乱
4. 格式不规范

## FAQ（常见问题）

**Q1：邮件一般多长合适？**
A：视场景而定：
- 开发信：100-200字
- 正式商务沟通：200-500字
- 方案/汇报类：500-1000字
- 高管邮件：越短越好，聚焦核心

**Q2：邮件标题怎么写？**
A：原则是清晰+有价值：
- 明确说明邮件主题
- 可包含收件人关注的好处
- 避免纯广告/垃圾邮件感

**Q3：英文邮件开头用什么好？**
A：常见选择：
- "Hope this email finds you well."（较正式）
- "I hope you're doing well."（友好）
- "Quick question about..."（直接）
- "Following up on..."（跟进）

**Q4：没有回复怎么办？**
A：建议：
- 等待3-5个工作日
- 用不同角度再次触达
- 提供"便捷回复选项"
- 考虑其他联系方式

**Q5：如何处理敏感信息？**
A：
- 涉及报价、合同等可要求"如邮件有误请忽略"
- 敏感数据脱敏处理
- 重要信息用附件而非正文
