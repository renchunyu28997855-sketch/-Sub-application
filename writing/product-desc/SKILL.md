# product-desc

## 功能说明
创作电商平台的商品详情页文案，包括标题、主图文案、详情描述、买家秀文案等。

---

## 假设声明（Assumptions）

1. **产品真实存在**：用户提供真实产品的准确信息
2. **平台明确**：用户指定目标电商平台
3. **合规要求**：产品符合平台和广告法规范
4. **适用平台**：淘宝、天猫、京东、亚马逊、拼多多、小红书电商
5. **不支持**：违禁品、虚假产品、侵权产品

## 详细工作流程（Detailed Workflow）

### 阶段一：产品分析
1. **理解产品特点**
   - 核心功能/用途
   - 差异化卖点
   - 使用场景

2. **了解目标用户**
   - 核心用户画像
   - 购买决策因素
   - 关注点和顾虑

### 阶段二：平台适配
3. **匹配平台特点**
   - 各平台用户群体差异
   - 各平台内容规范
   - 各平台爆款模式

4. **制定内容策略**
   - 主图文案策略
   - 详情页结构
   - 买家秀引导

### 阶段三：内容创作
5. **撰写各位置文案**
   - 标题：含关键词，吸引点击
   - 主图文案：突出核心卖点
   - 详情页：层层递进说服

6. **优化与审核**
   - 检查违禁词
   - 核实数据准确性
   - 确保差异化表达

### 检查点
- [ ] 标题含核心关键词
- [ ] 主图突出差异化卖点
- [ ] 详情页结构清晰
- [ ] 行动引导明确
- [ ] 无违禁词

## 边界情况（Edge Cases）

### 情况1：多SKU产品
**问题**：如衣服多颜色/多尺码
**处理方式**：
- 提供SKU描述框架
- 突出可选性而非堆砌
- 引导选择合适的沟通

### 情况2：新品无评价
**问题**：新产品缺乏信任背书
**处理方式**：
- 强调产品本身卖点
- 提供品牌/资质背书
- 引导售前咨询

### 情况3：对比同类产品
**问题**：用户希望对比竞品
**处理方式**：
- 突出自身优势
- 不直接贬低竞品
- 避免违反广告法

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 违禁词 | 包含"最"、"第一"等绝对词 | 自动替换为合规表达 |
| 数据虚假 | 引用无法核实的功能数据 | 删除或改为模糊表达 |
| 侵权风险 | 提及未授权的品牌/专利 | 删除相关表述 |
| 平台违规 | 违反特定平台规则 | 调整至平台适配 |

### 失败案例及修正

**原始问题标题**：
```
最好的耳机，最便宜的价格，不买就后悔！
```

**问题分析**：
1. 违禁词："最好"、"最便宜"
2. 过度夸张：降低可信度
3. 无关键词：搜索不友好
4. 强制推销感强

**修正后**：
```
【标题备选】
1. 【-42dB降噪】XX无线耳机 36小时续航 蓝牙5.3 适用苹果华为小米
2. XX耳机蓝牙5.3降噪耳机运动游戏吃鸡无延迟 一年换新
3. 【官方旗舰】XX无线耳机 主动降噪 36H续航 蓝牙5.3
```

## FAQ（常见问题）

**Q1：标题多长合适？**
A：各平台不同：
- 淘宝/天猫：30-40个汉字
- 京东：50-60个汉字
- 亚马逊：200字符以内

**Q2：主图一般几张？**
A：
- 淘宝/天猫：5张主图
- 京东：第一张白底主图+4张场景图
- 亚马逊：1张主图+6张辅图

**Q3：详情页多长合适？**
A：一般8-15屏（手机端），内容充实但不冗余

**Q4：可以帮我生成图片吗？**
A：抱歉我无法直接生成图片，但我会提供：
- 主图文案和位置建议
- 详情页结构规划
- 需要的图片类型描述

**Q5：如何提高转化率？**
A：关键点：
- 前3张图解决80%的购买疑虑
- 突出差异化卖点
- 消除常见顾虑（如售后）
- 真实的买家秀
