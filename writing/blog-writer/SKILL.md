# blog-writer

## 功能说明
创作各类博客文章，包括教程类、观点类、行业分析类，提供吸引读者的开头和SEO优化建议。

---

## 假设声明（Assumptions）

1. **主题明确**：用户有明确的文章主题和大致方向
2. **目标读者可知**：用户大致了解文章面向的读者群体
3. **内容真实性**：用户提供的事实、数据、案例是真实可靠的
4. **适用场景**：内容营销博客、技术博客、个人博客、资讯博客
5. **不支持**：完全虚构的新闻报道、虚假数据支撑的观点文章

## 详细工作流程（Detailed Workflow）

### 阶段一：主题分析与素材收集
1. **理解主题和角度**
   - 与用户确认文章核心观点
   - 明确差异化角度
   - 确定文章能解决什么问题

2. **素材收集与整理**
   - 用户提供的资料/数据
   - 补充相关背景信息
   - 整理可用案例

### 阶段二：结构设计
3. **制定文章大纲**
   - 设计开头（吸引读者）
   - 规划主体结构（逻辑清晰）
   - 设计结尾（总结/行动引导）

4. **SEO关键词布局**
   - 确定核心关键词
   - 规划关键词分布
   - 设计标题和子标题

### 阶段三：内容创作
5. **撰写各部分内容**
   - 开头：痛点引入/惊人数据/个人故事
   - 主体：论点+论据+案例
   - 结尾：总结+行动建议

6. **优化标题和SEO元素**
   - 主标题：包含关键词，有吸引力
   - Meta描述：概括文章，诱导点击
   - 子标题：层级清晰，含关键词

### 检查点
- [ ] 主题是否突出
- [ ] 结构是否清晰
- [ ] 是否有干货价值
- [ ] SEO元素是否完整
- [ ] 是否引导互动

## 边界情况（Edge Cases）

### 情况1：主题过于宽泛
**问题**："帮我写一篇关于人工智能的文章"
**处理方式**：
- 提供选题方向建议
- 将大主题拆分为具体小主题
- 示例："人工智能" → "2024年普通人用AI提效的5个方法"

### 情况2：用户提供素材不足
**问题**：只有主题，没有具体素材
**处理方式**：
- 基于主题提供通用框架
- 使用示例/案例填充
- 标注"[可根据实际情况替换]"的部分

### 情况3：需要快速产出
**问题**：用户需要短时间完成文章
**处理方式**：
- 提供"极简版文章结构"
- 简化流程，聚焦核心观点
- 建议用户后续补充细节

### 情况4：文章类型混合
**问题**：如"既是教程又是观点"
**处理方式**：
- 确定主要类型
- 其他类型作为补充元素
- 明确文章主线

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 数据引用错误 | 提及虚假/不可查证数据 | 标注"[数据待核实]"或删除 |
| 事实性错误 | 与公认事实矛盾 | 修正并说明 |
| 敏感话题 | 涉及政治/争议内容 | 建议调整角度或标注风险 |
| 过度营销 | 明显硬广感觉 | 调整为更软性的表达 |

### 失败案例及修正

**原始问题**：文章写成流水账
```
"今天我去了一个地方。这个地方很好玩。有很多项目可以玩。我们玩了第一个项目，然后玩了第二个项目，然后玩了第三个项目..."
```

**问题分析**：
1. 缺乏核心观点 - 不知道想表达什么
2. 平铺直叙 - 没有重点和亮点
3. 缺乏读者价值 - 对读者没有帮助

**修正后**：
```
【调整后结构】
一、开头：那个让我差点哭出来的游乐场（悬念引入）
二、为什么这个游乐场与众不同（核心观点：体验经济时代的差异化竞争）
三、3个让人惊艳的细节设计（具体案例）
  1. 排队等待也被精心设计
  2. 员工比游客更投入
  3. 每个角落都是拍照点
四、对做产品/服务的启发（对读者的价值）
五、结尾：你多久没有真正"惊喜"过了？

【修正说明】增加核心观点、具体细节描写、对读者的启发
```

## FAQ（常见问题）

**Q1：文章一般多长合适？**
A：视平台和类型而定：
- 资讯类博客：800-1500字
- 深度干货：2000-4000字
- 教程类：1500-3000字
- 观点类：1000-2000字

**Q2：如何提高文章通过率？**
A：关键点：
- 开头有钩子，前100字抓住注意力
- 有具体案例/数据支撑
- 结尾有行动指引或讨论引导
- 避免过于营销化

**Q3：SEO关键词密度多少合适？**
A：建议2-3%，自然分布，避免堆砌。

**Q4：可以帮我配图吗？**
A：抱歉我无法直接生成图片，但我会提供：
- 配图位置建议
- 图片内容描述（可用于AI生图或配图参考）
- Alt文本建议

**Q5：文章署名和版权如何处理？**
A：文章内容由我根据您的需求创作，署名权归您所有。如需商用，请自行确认其中引用的第三方内容（如数据、品牌名）的版权。

# 远程办公效率提升指南：我是如何从每天加班到准时下班的

去年这个时候，我还在每天加班到晚上10点，项目总是delay，老板不满意，自己也很焦虑。转远程办公后，我花了3个月时间调整，现在每天6点准时下班，项目反而提前交付。

## 远程办公的3大效率杀手

**1. 边界模糊**
在家工作容易工作和生活混在一起，该休息时无法休息。

**2. 干扰太多**
家人、同事消息、快递外卖……专注时间被打碎。

**3. 缺乏反馈**
看不到同事的工作状态，不知道自己做得怎么样。

## 5个亲测有效的技巧

### 技巧1：固定工作和休息的物理空间
在家里划分专门的工作区域，换上工作服才进入。物理边界帮助心理边界建立。

### 技巧2：用番茄工作法管理时间
- 25分钟专注工作
- 5分钟休息
- 每4个番茄钟休息15-30分钟
推荐工具：Forest、Toggl

### 技巧3：异步沟通优先
不是所有事情都需要即时回复。建立明确的响应时间预期：
- 紧急：钉钉/电话
- 一般：企业微信（2小时内）
- 不紧急：邮件/文档（24小时内）

### 技巧4：每日三件事法则
每天早上确定3件最重要的事（MIT），必须完成。剩下的可以放到第二天。

### 技巧5：仪式感收工
每天固定时间"下班"，关电脑、收拾桌面，给自己一个明确的结束信号。
