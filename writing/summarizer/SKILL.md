# summarizer

## 功能说明
从长文本中提取核心信息，生成结构化摘要，保留关键要点。

---

## 假设声明（Assumptions）

1. **文本可分享**：用户提供的文本不涉及高度敏感/机密信息
2. **文本完整**：用户提供的是完整文本或足够代表性的段落
3. **主题明确**：文本有明确的主题，不是杂乱的混合内容
4. **适用场景**：新闻报道、学术论文、报告、书籍章节、长文
5. **不支持**：完全无结构的聊天记录、代码片段

## 详细工作流程（Detailed Workflow）

### 阶段一：文本理解
1. **阅读理解原文**
   - 确定文章类型
   - 识别主题和结构
   - 理解作者意图

2. **识别核心内容**
   - 主要论点/事件
   - 关键数据和事实
   - 重要引述/观点

### 阶段二：信息提炼
3. **提炼关键要点**
   - 区分核心信息和次要信息
   - 按逻辑分组整理
   - 保留最重要的细节

4. **组织摘要结构**
   - 确定摘要长度
   - 规划信息层次
   - 撰写摘要正文

### 阶段三：质量把控
5. **核对信息准确性**
   - 关键数据核实
   - 避免断章取义
   - 保持原文准确性

6. **优化表达**
   - 语言简洁清晰
   - 逻辑连贯
   - 保留原文风格

### 检查点
- [ ] 核心信息完整
- [ ] 无关键信息遗漏
- [ ] 表达简洁准确
- [ ] 结构清晰易读

## 边界情况（Edge Cases）

### 情况1：文本过长
**问题**：超过单次处理能力
**处理方式**：
- 分段处理，最后综合
- 提供分段摘要+整体摘要
- 说明如何分段处理

### 情况2：多主题文本
**问题**：文本涉及多个不相关主题
**处理方式**：
- 识别各主题
- 分别提取各主题要点
- 保持主题区分清晰

### 情况3：学术论文
**问题**：包含复杂方法和数据
**处理方式**：
- 突出研究目的、方法、结论
- 保留关键数据
- 说明研究局限性

### 情况4：无结构文本
**问题**：如聊天记录、分散的笔记
**处理方式**：
- 提取关键信息点
- 按主题/时间重新组织
- 标注信息来源

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 信息丢失 | 核心观点未体现 | 补充遗漏要点 |
| 断章取义 | 摘录与原意不符 | 重新理解并修正 |
| 数据错误 | 数字/事实偏差 | 核实后修正 |
| 过度简化 | 关键细节丢失 | 补充重要细节 |

### 失败案例及修正

**原始摘要问题**：
```
原文讨论了AI在医疗领域的应用，主要讲了AI可以帮助诊断，还有AI可以用于药物研发，还有医疗影像分析，还有一家叫DeepMind的公司做了什么。
```

**问题分析**：
1. 缺乏结构和重点
2. 流水账式罗列
3. 缺少具体数据和案例

**修正后**：
```
【优化摘要】

【核心主题】AI在医疗领域的应用现状与展望

【关键要点】
1. **诊断辅助**：AI在影像诊断（如X光、CT）中达到甚至超过人类医生水平
2. **药物研发**：AI大幅缩短新药研发周期，从数年缩短至数月
3. **核心公司**：DeepMind的AlphaFold解决了蛋白质结构预测难题

【重要数据】
- AI辅助诊断准确率：部分领域达95%以上
- 研发周期缩短：传统10年 → AI辅助2-3年

【总结】AI正在深刻改变医疗行业的各个环节，从诊断到药物研发都有显著价值。
```

## FAQ（常见问题）

**Q1：可以处理多长的文本？**
A：单次处理建议不超过5000字。超过可分段处理。

**Q2：可以提取特定方面的内容吗？**
A：可以。请在输入时说明侧重点，如"重点关注政策相关内容"。

**Q3：摘要和划重点有什么区别？**
A：
- 摘要：用简洁语言重新组织原文核心内容
- 划重点：保留原文结构，标记关键句子

**Q4：可以生成不同格式的摘要吗？**
A：可以。包括：
- 段落式摘要
- 要点列表
- 时间线
- 对比表格

**Q5：翻译后可以摘要吗？**
A：可以。如需翻译+摘要，请使用translator技能后再使用summarizer。
