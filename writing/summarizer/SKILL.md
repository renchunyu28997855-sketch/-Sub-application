# summarizer v3.0

长文摘要提取技能。2026年集成AI智能摘要、多角度摘要、关键提取能力。

## 功能说明
从长文本中提取核心信息，生成简洁准确的摘要，保留关键要点。

---

## 假设声明（Assumptions）

1. **文本可分享**：用户提供的文本不涉及高度敏感/机密信息
2. **文本完整**：用户提供的是完整文本或足够代表性的段落
3. **主题明确**：文本有明确的主题，不是杂乱的混合内容
4. **适用场景**：新闻报道、学术论文、报告、书籍章节、长文

---

## 输入输出（I/O Format）

### 输入格式
```
【文本内容】粘贴完整或大段文本
【摘要类型】简略摘要/详细摘要/要点列表/多角度摘要
【字数要求】（可选）XXX字左右
【侧重点】（可选）重点关注XXX方面
【分析角度】（可选）技术/商业/用户/风险角度
```

### 输出格式
```
【原文类型】XXX
【原文长度】约XXX字
【摘要长度】约XXX字
【核心主题】XXX
【关键要点】
1. ...
2. ...
3. ...
【详细摘要】（可选）
...
【重要数据/引述】（如有）
...
```

## 2026 AI新能力

| 能力 | 说明 |
|------|------|
| **AI智能摘要** | 深度语义理解，自动识别文章类型和风格，生成流畅可读摘要 |
| **多角度摘要** | 同时从技术/商业/用户/风险等多视角分析，发现单一角度忽略的信息 |
| **关键提取** | 智能提取最具价值的句子、段落和数据，保留核心引述 |

## 工作流程（Workflow）

### 阶段一：文本理解
1. **识别文章类型** - 新闻/论文/报告/其他
2. **理解主题结构** - 确定核心论点和逻辑

### 阶段二：信息提炼
3. **提取关键要点** - 区分核心/次要信息
4. **组织摘要结构** - 按类型生成对应格式

### 阶段三：质量把控
5. **核对信息准确性** - 关键数据核实
6. **优化表达** - 语言简洁，逻辑连贯

## 边界情况（Edge Cases）

| 情况 | 处理方式 |
|------|----------|
| 文本过长 | 分段处理，最后综合 |
| 多主题 | 分别提取各主题要点 |
| 学术论文 | 突出背景/方法/结果/结论 |
| 无结构文本 | 提取关键点，按主题重组 |
