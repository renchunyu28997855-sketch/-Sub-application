# tech-writer v3.0

技术文档编写技能。2026年集成智能结构生成、术语音义解释、多语言文档能力。

## 功能说明

创作各类技术文档，包括API文档、用户手册、开发指南等，确保技术信息清晰易懂。

---

## 输入输出（I/O Format）

### 输入格式
```
【文档类型】API文档/用户手册/开发指南/技术规范/Changelog
【目标读者】XXX（如：前端开发工程师、运维人员、普通用户）
【技术背景】
- 接口/功能描述：XXX
- 请求/响应格式：XXX
- 错误码说明：XXX
【特殊要求】XXX（如：需要Markdown格式、需要包含 UML 图）
```

### 输出格式
```
【文档类型】XXX
【目标读者】XXX
【文档结构】
1. 概述
2. ...
【正文内容】
...
【示例代码】
...
【注意事项】
...
```

---

## 2026年新功能

### 智能文档结构生成
- **自动识别文档类型**：根据输入信息自动匹配最佳文档结构
- **语义章节排序**：基于逻辑关系自动组织章节顺序
- **API路径优化**：建议符合RESTful规范的路径命名

### 技术术语解释
- **自动检测术语**：识别文档中的技术术语
- **提供简明解释**：为复杂术语提供通俗易懂的解释
- **生成术语表**：自动提取并生成术语表

### 多语言文档
- **一键生成中英文**：同步输出中英文版本
- **技术词汇对齐**：确保多语言版本术语一致
- **本地化适配**：适应不同地区的表达习惯

## 工作流程（Workflow）

### 阶段一：需求分析
1. **确认文档类型和目标读者**
2. **收集技术信息**（接口、参数、返回值）

### 阶段二：结构设计
3. **设计文档结构**（根据文档类型自动匹配）
4. **确定文档层级**

### 阶段三：内容编写
5. **撰写各部分内容**
6. **编写示例代码**

### 检查点
- [ ] 技术信息准确性
- [ ] 术语一致性
- [ ] 示例可执行
- [ ] 结构完整性

---

## 质量验证清单（Quality Checklist）

### 准确性检查
- [ ] 技术参数与用户提供信息一致
- [ ] 示例代码语法正确、可执行
- [ ] 错误码与实际场景匹配

### 可读性检查
- [ ] 结构清晰，层次分明
- [ ] 术语使用一致
- [ ] 表格格式整齐
