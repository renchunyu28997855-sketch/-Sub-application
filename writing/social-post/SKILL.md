# social-post

## 功能说明
为不同社交媒体平台创作内容，包括微博、小红书、知乎、抖音等，确保内容符合平台调性和用户喜好。

---

## 假设声明（Assumptions）

1. **平台明确**：用户明确指定目标平台
2. **账号定位已知**：用户的账号风格/粉丝群体大致了解
3. **内容方向清晰**：用户有大致的内容主题
4. **适用平台**：小红书、微博、知乎、抖音/视频号、公众号
5. **不支持**：涉及政治敏感、违法违规、恶意营销的内容

## 详细工作流程（Detailed Workflow）

### 阶段一：平台分析
1. **确认目标平台**
   - 各平台特点和用户群体
   - 内容形式和长度限制
   - 标签/话题使用习惯

2. **分析平台调性**
   - 小红书：种草、生活方式、真实分享
   - 微博：短平快、热点、话题感
   - 知乎：专业、深度、逻辑性
   - 抖音：视觉冲击、节奏感、娱乐性

### 阶段二：内容策划
3. **确定内容角度**
   - 如何切入主题
   - 差异化角度
   - 目标用户痛点/痒点

4. **匹配平台语言**
   - 用词风格
   - 排版格式
   - emoji使用

### 阶段三：创作优化
5. **撰写正文**
   - 开头吸引注意力
   - 中间干货/有趣
   - 结尾引导互动

6. **设计标签和互动**
   - 热门话题标签
   - 互动引导语
   - 发布时间建议

### 检查点
- [ ] 符合平台调性
- [ ] 内容真实可信
- [ ] 标签使用得当
- [ ] 有明确互动引导
- [ ] 无平台敏感词

## 边界情况（Edge Cases）

### 情况1：多平台同时发布
**问题**：同一内容需要适配多个平台
**处理方式**：
- 提供多平台版本
- 指出各平台差异点
- 节省跨平台运营成本

### 情况2：平台规则限制
**问题**：涉及平台敏感词或限制
**处理方式**：
- 自动规避敏感词
- 提供替代表达
- 标注风险内容

### 情况3：需要系列内容
**问题**：一个主题需要多篇内容
**处理方式**：
- 设计内容系列规划
- 每篇角度不同
- 形成连续性

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|----------|----------|----------|
| 敏感词 | 包含平台违禁词 | 自动替换或标注风险 |
| 平台不适配 | 内容风格与平台不符 | 调整至平台适合的风格 |
| 信息错误 | 事实性错误 | 核实后修正 |
| 过度营销 | 硬广感太强 | 调整为更软性的表达 |

### 失败案例及修正

**原始问题**：
```
【小红书内容】
今天去了一家店，非常好，推荐大家去，服务很好，环境不错，价格合理。
```

**问题分析**：
1. 缺乏具体信息 - 什么店？在哪里？
2. 没有差异化 - 为什么推荐这家？
3. 语言空泛 - "很好"没有说服力
4. 缺少互动引导

**修正后**：
```
【标题】在XX区发现了一家宝藏咖啡店☕

【正文】
📍店名：XXX Coffee Lab
📍地址：XX区XX路XX号（地铁XX站B口）

一进门就被工业风+绿植的装修吸引了，超级出片！

## FAQ（常见问题）

**Q1：各平台最佳发布时间？**
A：大致参考：
- 小红书：18:00-21:00（下班后）
- 微博：12:00-13:00、21:00-23:00
- 知乎：工作时间（深度内容）
- 抖音：12:00-14:00、18:00-20:00

**Q2：标签越多越好吗？**
A：不一定。各平台有差异：
- 小红书：3-8个相关标签
- 微博：1-3个热门话题
- 抖音：3-5个相关标签

**Q3：可以帮我生成图片吗？**
A：抱歉我无法直接生成图片，但我会提供：
- 内容文案和标题
- 配图建议/描述
- 排版格式参考

**Q4：如何提高互动率？**
A：技巧包括：
- 结尾提出问题引导评论
- 分享个人经历引发共鸣
- 提供收藏价值（干货整理）
- 使用有争议性的话题
