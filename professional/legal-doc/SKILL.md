# legal-doc v3.0

法律文书撰写与审核技能。2026年集成AI文档分析、风险识别、条款解释能力。

## 功能说明

帮助用户撰写、审核和理解基础法律文书，提供文书结构指导、条款库参考和风险提示。

## 重要声明

**免责声明**：
- 本技能提供基础法律文书模板和参考，不能替代专业律师的撰写和审核
- 重大法律文书（涉及大额交易、复杂权利义务）务必委托专业律师
- 法律文书具有法律效力，签署前请仔细阅读并理解全部条款
- 本模板基于中国大陆法律框架

---

## 假设声明（Assumptions）

- 文书类型属于常见的民事合同或协议
- 金额不大、关系不复杂
- 用户能够理解文书内容

### 不适用场景
- 涉及重大利益（金额较大、涉及重要权利）
- 复杂法律关系（多方当事人、多层嵌套）
- 需要公证或见证的文书
- 诉讼相关文书（起诉状、答辩状等）
- 知识产权、婚姻家庭等复杂领域

## 输入输出（I/O Format）

### 标准输入格式
```
【法律文书】
文书类型：<合同/协议/律师函/授权书等>
具体场景：<如：房屋租赁、股权转让、劳务合作等>
当事人信息：<甲乙方基本信息>
关键条款：<需要特别约定的事项>
交易金额：<如适用>
特殊要求：<其他需要说明的内容>
```

### 标准输出格式
```
【法律文书撰写/审核】

## 文书标题

## 重要提示
<签署前需要注意的事项>

## 正文内容
<完整文书正文或审核意见>

## 条款说明
<解释关键条款的含义>

## 风险提示
<该类文书的风险点>

## 后续建议
<是否需要律师审核、公证等>
```

## 工作流程（Workflow）

### 阶段一：需求确认
- [ ] 明确文书类型
- [ ] 了解当事人信息
- [ ] 确认标的/服务/交易内容
- [ ] 确认金额和支付方式

### 阶段二：文书起草/审核
- [ ] 按照标准结构起草
- [ ] 检查必要条款是否齐全
- [ ] 审核条款是否公平合理
- [ ] 识别潜在风险点

### 阶段三：输出交付
- [ ] 提供完整文书/审核报告
- [ ] 解释关键条款
- [ ] 提示签署注意事项
- [ ] 建议是否需要律师审核

---

## 质量验证清单（Quality Checklist）

- [ ] 文书类型是否正确
- [ ] 当事人信息是否完整
- [ ] 关键条款是否齐全
- [ ] 金额、日期是否明确
- [ ] 违约责任是否约定
- [ ] 争议解决条款是否包含
- [ ] 是否包含免责声明
