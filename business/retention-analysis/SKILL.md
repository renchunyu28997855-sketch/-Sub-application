# retention-analysis

## 功能说明
通过数据分析用户留存情况，识别流失原因，制定留存提升策略。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有可分析的用户行为数据
- 数据埋点完整（至少覆盖核心行为）
- 有一定用户量级（建议>1000用户）
- 数据时间跨度足够（建议>30天）

### 适用范围
- App用户留存分析
- 订阅服务用户留存分析
- 电商平台复购分析
- 会员体系留存分析
- 游戏用户留存分析

### 不适用场景
- 缺乏数据基础设施
- 用户量太少无法做统计分析
- 数据质量差（埋点缺失、数据不准确）
- 纯线下业务

### 分析假设
- 留存是用户价值的重要指标
- 留存分析需要结合定性和定量
- 留存提升需要产品+运营协同

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：数据准备（第1-2步）

```
步骤1：明确留存定义
├── 指标定义：次日留存、7日留存、30日留存
├── 行为定义：什么是"活跃"
├── 分子分母：用什么做基准
├── 时间窗口：按天/周/月
└── 检查点：留存定义是否合理

步骤2：获取和清洗数据
├── 数据源确认
├── 数据提取
├── 数据清洗（去重、异常值）
├── 数据验证
└── 检查点：数据质量是否可信
```

### 阶段二：留存分析（第3-5步）

```
步骤3：计算留存指标
├── 留存曲线计算
├── Cohort分析（按周/月/渠道）
├── 留存率表
├── 流失率计算
└── 检查点：计算是否正确

步骤4：分析留存曲线
├── 识别关键节点（1天、7天、30天）
├── 观察曲线斜率变化
├── 对比不同时期/渠道
├── 识别异常波动
└── 检查点：是否有洞察

步骤5：深度Cohort分析
├── 按用户群分析（新用户vs老用户）
├── 按渠道分析（不同来源用户留存）
├── 按行为分析（完成核心行为vs未完成）
└── 检查点：Cohort是否有差异
```

### 阶段三：流失分析（第6-7步）

```
步骤6：识别流失用户
├── 流失用户定义
├── 流失用户特征
├── 流失时间分布
├── 流失阶段分析
└── 检查点：流失定位是否准确

步骤7：分析流失原因
├── 数据分析（流失前的行为模式）
├── 用户调研（流失用户访谈/问卷）
├── 竞品对比（流失去向）
├── 产品变化（版本更新影响）
└── 检查点：原因分析是否有依据
```

### 阶段四：策略制定（第8-9步）

```
步骤8：制定留存策略
├── 新用户激活策略
├── 成长体系策略
├── 沉默用户召回策略
├── 流失用户召回策略
└── 检查点：策略是否有针对性

步骤9：优先级排序
├── ICE评分（Impact/Confidence/Ease）
├── ROI评估
├── 实施难度
└── 检查点：优先级是否合理
```

### 阶段五：监控复盘（第10步）

```
步骤10：建立监控体系
├── 留存指标监控
├── 预警机制
├── 复盘机制
└── 检查点：监控是否到位
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 没有留存数据 | 建议先完善埋点，建立数据基础 |
| 数据量太少 | 降低分析粒度，或定性分析为主 |
| 不知道核心行为 | 协助定义核心行为 |
| 跨平台产品 | 分别分析各平台，统一分析 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 季节性产品 | 对比同期数据 |
| 促销拉新 | 区分自然用户vs促销用户 |
| 平台迁移 | 考虑用户迁移影响 |
| 产品改版 | 分析改版前后留存变化 |

### 流失场景

| 场景 | 分析重点 |
|------|----------|
| 新用户流失高 | 引导体验、新手任务 |
| 老用户流失高 | 产品疲劳、竞品吸引 |
| 特定版本流失高 | 版本问题排查 |
| 特定渠道流失高 | 渠道质量问题 |

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 定义不一致 | 留存率计算结果差异大 | 明确分子分母定义 |
| 幸存者偏差 | 只看留存用户 | 同时分析流失用户 |
| 虚假繁荣 | 注册多但留存低 | 关注留存质量 |
| 归因错误 | 留存提升归因错误 | 控制变量分析 |
| 忽视外部因素 | 节假日/竞品影响 | 考虑外部变量 |

### 数据质量

| 问题 | 检测方式 | 处理方式 |
|------|----------|----------|
| 埋点缺失 | 数据完整性检查 | 补充埋点 |
| 数据延迟 | 时间戳分析 | 等待数据同步 |
| 异常值 | 分布分析 | 排除异常 |
| 重复数据 | 去重检查 | 数据清洗 |

## 4. 流失原因假设

### 假设1：新用户没有完成"首次购买"
数据显示，完成首次购买的用户7日留存达60%，而未完成购买的仅15%。

### 假设2：App体验不够流畅
新用户反馈中，"找不到想要的商品"和"加载太慢"排名前二。

### 假设3：促销吸引的并非目标用户
大促期间用户质量下降，需要区分高质量用户。

## 8. FAQ（常见问题）

### Q1：留存率多少算合格？
**A**：行业不同标准不同，常见基准：
- 社交App：次日留存45%+，30日留存20%+
- 电商App：次日留存40%+，30日留存15%+
- 工具App：次日留存30%+，30日留存10%+
- SaaS：周留存70%+，月留存50%+
建议与竞品对标，而非绝对值。

### Q2：留存和活跃是什么关系？
**A**：
- 留存：用户在某个时间周期后是否回来
- 活跃：用户在某天/周/月是否使用
- 关系：留存好的用户通常更活跃
- 注意：留存是比活跃更严格的质量指标

### Q3：如何提升次日留存？
**A**：
- 优化新用户引导
- 快速让用户体验核心价值
- 新手任务和奖励
- 推送召回
- 社交关系导入

### Q4：Cohort分析是什么？
**A**：Cohort分析是按用户群体分析留存的方法。
- 按注册时间分组（周Cohort、月Cohort）
- 观察不同群体的留存曲线
- 识别渠道或产品变化的影响

### Q5：留存提升多久见效？
**A**：
- 短期（1-2周）：推送召回、运营活动
- 中期（1-2月）：产品优化、新手引导
- 长期（3月+）：产品价值、生态建设
- 建议：多策略并行，持续优化

### Q6：如何区分自然流失和被竞品挖走？
**A**：
- 用户调研（流失用户访谈）
- 竞品数据（竞品用户增长）
- 行为分析（流失前是否搜索竞品）
- 问卷调查（用户去哪了）
