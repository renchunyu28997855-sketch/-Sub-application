# growth-hacking

## 功能说明
帮助制定用户增长策略，利用增长黑客思维，通过裂变、留存优化、召回等手段提升用户量。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有产品（已上线或接近上线）
- 有一定用户基础（可分析数据）
- 有产品迭代能力
- 愿意投入资源测试

### 适用范围
- 0-1阶段产品的增长策略
- 1-10阶段产品的增长加速
- 特定增长瓶颈突破
- 增长团队建设参考

### 不适用场景
- 纯线下业务（增长策略差异大）
- 平台型业务（需要双边增长策略）
- 强监管行业（合规优先）
- 产品尚未达到PMF（先做产品市场匹配）

### 增长阶段假设
- 早期（0-1）：产品驱动，病毒增长
- 成长期（1-10）：渠道+产品双驱动
- 成熟期（10-N）：精细化运营，品牌驱动

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：现状分析（第1-3步）

```
步骤1：定义北极星指标
├── 什么是衡量业务成功的核心指标
├── 这个指标能否反映用户价值
├── 指标是否简单易懂
└── 检查点：北极星指标是否正确

步骤2：分析增长漏斗
├── 获客漏斗（渠道→激活→注册）
├── 激活漏斗（注册→首次使用→核心行为）
├── 留存漏斗（日活→周活→月活）
├── 变现漏斗（免费→付费→复购）
└── 检查点：找到漏斗最大流失点

步骤3：识别增长瓶颈
├── 数据分析：哪里流失最多
├── 用户调研：为什么流失
├── 竞品分析：别人怎么做的
└── 检查点：瓶颈定位是否准确
```

### 阶段二：策略制定（第4-5步）

```
步骤4：制定增长策略
├── AARRR框架应用
│   ├── Acquisition：获客渠道优化
│   ├── Activation：激活率提升
│   ├── Retention：留存率提升
│   ├── Referral：推荐裂变
│   └── Revenue：变现优化
├── 分渠道策略
├── 分用户群策略
└── 检查点：策略是否系统化

步骤5：确定优先级
├── Impact（影响大小）
├── Effort（投入多少）
├── ICE评分（Impact/Confidence/Ease）
└── 检查点：优先级是否合理
```

### 阶段三：方案设计（第6-7步）

```
步骤6：设计增长实验
├── 实验假设
├── 实验设计（对照组/实验组）
├── 样本量计算
├── 实验周期
└── 检查点：实验设计是否科学

步骤7：制定技术方案
├── 功能需求
├── 数据埋点
├── 技术实现
├── AB测试框架
└── 检查点：技术是否可实现
```

### 阶段四：执行优化（第8-9步）

```
步骤8：执行增长实验
├── 按优先级执行实验
├── 数据监控
├── 异常处理
└── 检查点：执行是否到位

步骤9：分析实验结果
├── 统计显著性检验
├── 效果评估
├── 决策：推广/迭代/放弃
└── 检查点：决策是否数据驱动
```

### 阶段五：规模化（第10步）

```
步骤10：规模化成功实验
├── 扩大实验范围
├── 持续优化
├── 建立机制
└── 检查点：是否形成增长飞轮
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 用户只说"帮我增长" | 询问产品、阶段、目标 |
| 没有用户数据 | 建议先做基础数据埋点 |
| 不知道北极星指标 | 协助定义北极星指标 |
| 竞品情况未知 | 建议先做竞品分析 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 0用户阶段 | 先验证PMF，而非增长 |
| 增长停滞 | 分析留存和召回 |
| 增长作弊风险 | 设计防作弊机制 |
| 隐私合规（GDPR等） | 确保增长策略合规 |

### 阶段适配

| 阶段 | 核心策略 |
|------|----------|
| 0-1（PMF验证） | 口碑传播、小规模测试 |
| 1-10（快速增长） | 渠道投放、裂变增长 |
| 10-100（精细运营） | 分层运营、留存优化 |
| 100+（品牌驱动） | 品牌广告、市场公关 |

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 增长作弊 | 短期数据暴涨、留存极低 | 建立防作弊机制 |
| 忽视留存 | 只关注获客 | 同时优化留存 |
| 实验不科学 | 无对照组、样本不足 | 完善实验设计 |
| 短期思维 | 只看短期指标 | 关注LTV和长期价值 |
| 盲目复制 | 完全复制竞品策略 | 适配自身产品 |

### 增长陷阱

| 陷阱 | 说明 | 避免方式 |
|------|------|----------|
|虚荣指标 | 只看注册数不看质量 | 定义北极星指标 |
| 猴子陷阱 | 大量用户但无留存 | 提升留存优先 |
| 短视增长 | 牺牲体验换数据 | 平衡用户体验 |
| 数据偏见 | 只看数据忽视用户声音 | 结合定性调研 |

## 7. 多个示例（Multiple Examples）

### 典型案例：社交App增长策略

**输入**：
```
App类型：陌生人社交
当前：日活10万，增长停滞
目标：6个月日活50万
```

**增长策略片段**：
```
【App类型】陌生人社交
【当前问题】
- 日活10万，增长停滞
- 次日留存35%（行业平均40%）
- 没有自然增长，主要靠投放

【增长目标】
- 6个月内日活达到50万
- 次日留存提升至45%
- 获客成本降低至15元

【北极星指标】日活（DAU）

## P0策略1：裂变邀请

### 方案设计
- 邀请1个新用户 → 邀请者获得VIP会员7天
- 邀请3个 → 获得VIP会员30天
- 邀请10个 → 获得VIP会员90天 + 称号

### 技术实现
- 记录邀请关系链
- 自动发放奖励
- 防止刷量（设备指纹+行为检测）

### 预期效果
- 获客成本：约5元/人（远低于付费投放的30元）
- 转化率：预计10%的活跃用户会邀请

---

## P0策略2：分享有礼

### 方案设计
用户分享内容到朋友圈/微博：
- 分享动态卡片 → 获得3次免费匹配机会
- 分享成就 → 获得虚拟礼物
- 分享邀请码 → 双方获得优惠券

### 触发时机
- 完成首次匹配后引导分享
- 获得"匹配成功"成就时引导
- 每日首次使用时引导

---

## P1策略3：召回系统

### 用户分层

| 用户类型 | 定义 | 召回策略 |
|----------|------|---------|
| 流失预警 | 连续3天未打开 | Push召回 |
| 流失用户 | 7天未打开 | 短信+Push |
| 沉默用户 | 14天未打开 | 优惠券召回 |
| 流失用户 | 30天未打开 | 重定向广告 |

### 召回内容
- 流失1-3天：功能更新通知
- 流失7-14天：专属优惠
- 流失14天+：老用户活动
```

## 8. FAQ（常见问题）

### Q1：增长团队怎么建？
**A**：
- 早期：增长负责人+1-2个工程师
- 成长期：增长/留存/裂变分工
- 成熟期：增长产品+数据+工程
- 关键：数据驱动、实验文化

### Q2：北极星指标怎么选？
**A**：
- 原则：反映用户价值、可衡量、简单
- 常见：DAU、MAU、付费用户数
- 注意：不同阶段可能不同

### Q3：增长实验多久跑一次？
**A**：
- 实验周期：通常1-2周
- 样本要求：统计显著（通常95%置信度）
- 优先级：ICE评分高的先跑
- 节奏：团队每周可跑2-4个实验

### Q4：裂变活动怎么防作弊？
**A**：
- 设备指纹识别
- 行为检测（IP、设备异常）
- 奖励延迟发放
- 人工审核机制
- 举报和惩罚机制

### Q5：留存低怎么办？
**A**：
1. 分析流失节点（哪里流失最多）
2. 调研流失用户（为什么走）
3. 优化新用户引导
4. 提升核心体验
5. 建立召回机制

### Q6：增长遇到瓶颈怎么突破？
**A**：
1. 回到数据，找到最大瓶颈点
2. 竞品分析，学习有效策略
3. 用户调研，了解真实需求
4. 尝试新渠道或新产品功能
5. 考虑品牌建设
