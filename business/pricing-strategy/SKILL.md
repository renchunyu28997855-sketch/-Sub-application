# pricing-strategy

## 功能说明
帮助制定和优化产品/服务的定价策略，分析不同定价方法的优势和适用场景。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有明确的产品或服务
- 产品有明确的成本结构（或可估算）
- 目标市场已定义
- 竞品信息可获取

### 适用范围
- 新产品定价
- 现有产品调价
- 定价策略优化
- 订阅制定价
- SaaS产品定价
- B2B企业级定价
- B2C消费品定价

### 不适用场景
- 政府采购（受法规约束）
- 拍卖/竞价场景
- 垄断市场的政府定价
- 需要专业金融产品的复杂定价

### 市场假设
- 市场信息可获取（成本、竞品、用户）
- 用户价格敏感度可通过调研或数据获取
- 定价决策由商业目标驱动

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：信息收集（第1-3步）

```
步骤1：了解产品价值
├── 核心价值是什么
├── 与竞品的差异
├── 目标用户痛点
├── 用户支付意愿
└── 检查点：价值是否清晰可量化

步骤2：分析成本结构
├── 直接成本（可变成本）
├── 间接成本（固定成本分摊）
├── 边际成本（每增加一单位）
├── 成本优化空间
└── 检查点：成本数据是否准确

步骤3：研究竞品定价
├── 直接竞品定价
├── 间接竞品定价
├── 替代品定价
├── 竞品定价策略
└── 检查点：竞品信息是否完整
```

### 阶段二：策略选择（第4-5步）

```
步骤4：选择定价方法
├── 成本加成定价：适合成本稳定、产品标准化
├── 竞争定价：适合市场竞争激烈
├── 价值定价：适合差异化产品、高感知价值
├── 动态定价：适合需求波动、库存敏感
└── 检查点：方法是否适合产品特性

步骤5：确定定价策略
├── 价格层级（入门/主流/高级）
├── 免费增值模式（Freemium）
├── 捆绑定价
├── 促销定价
└── 检查点：策略是否匹配商业目标
```

### 阶段三：方案设计（第6-7步）

```
步骤6：设计价格结构
├── 版本设计（功能差异）
├── 价格锚定（引导选择）
├── 付款周期（月/年/终身）
├── 折扣策略（早鸟/团购/协议价）
└── 检查点：结构是否清晰、有吸引力

步骤7：设定具体价格
├── 基于价值的定价
├── 基于竞争的定价
├── 基于成本的定价
├── 综合确定最终价格
└── 检查点：价格是否合理、有竞争力
```

### 阶段四：验证与优化（第8-9步）

```
步骤8：价格验证
├── 用户调研验证
├── AB测试（如可能）
├── 盈亏平衡分析
├── 敏感度分析
└── 检查点：价格是否可接受

步骤9：制定执行计划
├── 上线节奏
├── 过渡策略（旧客户处理）
├── 沟通策略
├── 监控指标
└── 检查点：计划是否可执行
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 用户只说"帮我定个价" | 询问产品是什么、目标市场、成本结构 |
| 不知道成本 | 提供行业成本参考，或建议先做成本核算 |
| 没有竞品信息 | 研究行业报告、公开数据，或假设竞品价格带 |
| 用户坚持特定价格 | 分析价格可行性，讨论影响 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 完全新市场（无竞品） | 价值定价为主，参考相邻市场 |
| 垄断市场 | 成本+合理利润，考虑监管要求 |
| 价格战竞争 | 考虑差异化，避免纯价格竞争 |
| B2B复杂定价 | 提供报价模板，强调价值沟通 |
| 免费vs付费转换 | 设计合理的免费额度 |

### 调价场景

| 场景 | 处理方式 |
|------|----------|
| 成本上涨 | 评估消化能力、渐进调整或价值升级 |
| 市场份额下滑 | 谨慎分析原因，避免纯价格战 |
| 高增长目标 | 可牺牲短期利润换市场 |
| 品牌升级 | 同步提升价值感知 |

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 成本核算不全 | 漏掉隐性成本 | 全面核算所有成本项 |
| 过度关注竞品 | 竞品定多少我们就定多少 | 强调差异化价值 |
| 定价过高 | 转化率极低 | 重新评估价值或调整价格 |
| 定价过低 | 转化高但利润不足 | 提升价格敏感度测试 |
| 版本设计混乱 | 用户不知道选哪个 | 简化版本，突出差异 |

### 价格测试

| 测试类型 | 适用场景 | 风险 |
|----------|----------|------|
| 竞品参考 | 无历史数据 | 忽视自身价值 |
| 用户调研 | 了解意愿 | 实际行为与声称不符 |
| AB测试 | 可实施 | 需要足够流量 |
| 早期用户验证 | 预售场景 | 样本可能不具代表性 |

### 边界案例：硬件产品定价

**输入**：
```
产品：智能硬件（如智能音箱）
成本结构： BOM成本、模具成本、生产成本
特点：需要线下渠道
```

**定价要点**：
- 考虑渠道分润（通常30-40%）
- 考虑售后成本
- 竞品对标（小米、百度）
- 早期溢价、后期降价策略

---

### 失败案例：某定价问题分析

**问题特征**：
- 定价基于成本+20%利润
- 三个版本功能差异不大
- 年付价格是月付的12倍（无优惠）
- 无免费版本

**问题诊断**：
- 定价缺乏市场视角
- 版本设计失败，用户无选择理由
- 缺乏激励年付的机制
- 获客门槛高

**修正方向**：
- 参考竞品和用户价值
- 明确版本差异
- 年付折扣（通常是月付的8-10折）
- 增加免费试用

## 8. FAQ（常见问题）

### Q1：如何知道用户愿意付多少钱？
**A**：
- 用户调研（价格敏感度测试）
- 竞品对比（市场价位）
- 试销测试（小规模验证）
- 拍卖/竞价（揭示用户支付意愿）

### Q2：定价高了怎么调整？
**A**：
1. 分析原因：价值不足还是价格过高
2. 方案A：增加价值（功能/服务）
3. 方案B：降低价格（直接降价或增加版本）
4. 方案C：改变结构（订阅改买断）
5. 注意：调价影响品牌定位和用户预期

### Q3：如何应对价格战？
**A**：
1. 避免正面价格战
2. 差异化：价值而非价格竞争
3. 聚焦细分：不做全人群，做特定人群
4. 提升感知价值：品牌、服务、生态
5. 必要时：选择性降价+其他价值补偿

### Q4：订阅制价格多久调整一次？
**A**：
- 常规调整：年度审视
- 触发调整：成本大幅变化、竞品重大调价
- 原则：小步快跑，避免大起大落
- 通知：提前告知，给用户选择

### Q5：如何设计版本差异？
**A**：
- 入门版：满足基本需求，降低尝鲜门槛
- 主流版：性价比最高，主推
- 高级版：满足高需求，溢价空间
- 差异化点：用户数/功能数/存储/支持
- 避免：功能堆砌、价格混乱

### Q6：B2B定价和B2C有什么不同？
**A**：
- B2B：复杂决策、协议价、多年合同、关系驱动
- B2C：简单决策、标准价、短期合同、价格驱动
- B2B需要：销售团队、定制方案、ROI计算
