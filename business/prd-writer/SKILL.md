# prd-writer v3.0

## 核心能力

1. **PRD撰写** - 完整产品需求文档
2. **用户故事** - 角色/故事/验收条件
3. **验收标准** - 可测试的验收条件
4. **AI辅助** - 2026年AI能力集成

| 命令 | 说明 |
|------|------|

## 工作流程（渐进式披露）

### 阶段一：理解

```
1. 明确产品背景 → 解决什么问题
2. 确定目标用户 → 谁来用
3. 收集需求信息 → 来源/竞品/约束
```

### 阶段二：设计

```
4. 设计产品方案 → 功能架构/用户流程
5. 拆解用户故事 → 角色/故事/验收
6. 确定验收标准 → 可测试的条件
```

### 阶段三：撰写

```
7. 撰写PRD正文 → 概述/需求/非功能
8. 自审与修改 → 逻辑/完整性
9. 组织评审 → 产品/设计/技术/测试
```

## 常见问题

**Q: PRD应该多详细？**
核心功能详细写，通用功能简略写。原则：开发看了知道怎么做，测试看了知道怎么测。

**Q: 技术方案要写吗？**
不需要。PRD是产品视角，技术方案由技术团队输出。

**Q: 多个需求一起写还是分开？**
相关功能可合为1个PRD，独立功能建议分开。
