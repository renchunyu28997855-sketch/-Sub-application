## 工作流程（渐进式披露）

### 阶段一：信息收集

```
1. 明确项目定位 → 做什么、给谁做、独特价值
2. 收集商业信息 → 商业模式、核心竞争力
3. 市场数据 → 市场规模、增长趋势
```

### 阶段二：市场分析

```
4. 市场规模分析 → TAM/SAM/SOM
5. 竞争分析 → 直接/间接竞品
6. 用户分析 → 目标用户画像
```

### 阶段三：方案设计

```
7. 产品/服务设计 → 核心功能、独特价值
8. 商业模式设计 → 收入来源、定价策略
9. GTM策略 → 获客渠道、增长策略
```

### 阶段四：财务规划

```
10. 财务预测 → 营收/成本/现金流
11. 融资规划 → 融资金额、估值、用途
12. 风险分析 → 关键风险与应对
```

## 质量检查清单

### 内容完整性
- [ ] 执行摘要清晰有力（1-2页）
- [ ] 市场分析包含TAM/SAM/SOM
- [ ] 产品/服务描述具体可感知
- [ ] 商业模式说明如何盈利
- [ ] 竞争分析客观真实

### 逻辑一致性
- [ ] 痛点 → 解决方案 → 产品 → 市场 fit
- [ ] 市场规模 → 可获得份额 → 营收预测
- [ ] 融资用途 → 业务目标 → 财务回报

## 常见问题

**Q: BP应该多长？** 通常15-25页，核心是清晰有力。

**Q: 没有运营数据怎么办？** 早期项目侧重团队、赛道、模型验证。

**Q: 估值怎么定？** 参考同类融资案例，考虑阶段和团队。
