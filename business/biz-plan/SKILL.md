# biz-plan

## 功能说明
帮助撰写专业的商业计划书，包含执行摘要、市场分析、商业模式、竞争分析、财务预测等完整内容。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户已有明确的创业项目或业务方向
- 用户对目标市场有基本认知（可提供行业数据来源）
- 用户能提供产品/服务的基本描述
- 涉及融资时，用户需说明当前融资阶段和需求

### 适用范围
- 创业公司早期融资（天使轮、Pre-A、A轮）
- 内部项目立项评审
- 战略规划汇报
- 合作洽谈材料
- 商业竞赛/创业大赛

### 不适用场景
- 上市公司正式年报
- 监管机构要求的正式招股说明书
- 需要法律效力的正式商业文档

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：信息收集（第1-2步）
```
步骤1：明确项目定位
├── 确认项目名称和slogan
├── 了解所属行业和细分市场
├── 确认目标用户群体
└── 检查点：是否清楚回答"做什么、给谁做"

步骤2：收集商业信息
├── 商业模式：如何赚钱
├── 核心竞争力：为什么能赢
├── 团队背景：为什么能做
└── 检查点：是否有独特的 insight
```

### 阶段二：市场分析（第3-4步）
```
步骤3：市场规模分析
├── TAM（Total Addressable Market）：潜在市场总量
├── SAM（Serviceable Addressable Market）：可服务市场
├── SOM（Serviceable Obtainable Market）：可获得市场
└── 检查点：数据来源是否可靠

步骤4：竞争分析
├── 直接竞品（功能相似）
├── 间接竞品（用户相似）
├── 替代品（满足同类需求）
└── 检查点：是否诚实分析竞争劣势
```

### 阶段三：方案设计（第5-6步）
```
步骤5：产品/服务设计
├── 核心功能描述
├── 独特价值主张
├── 发展阶段规划
└── 检查点：是否清晰可执行

步骤6：商业模式设计
├── 收入来源
├── 定价策略
├── 成本结构
└── 检查点：单位经济模型是否成立
```

### 阶段四：财务规划（第7-8步）
```
步骤7：财务预测
├── 营收预测（3-5年）
├── 成本预测
├── 现金流预测
└── 检查点：假设是否合理，数字是否自洽

步骤8：融资规划
├── 融资金额
├── 资金用途
├── 估值逻辑
└── 检查点：估值是否有依据
```

### 阶段五：文档输出（第9步）
```
步骤9：撰写商业计划书
├── 执行摘要（最后写）
├── 市场分析
├── 产品/服务
├── 商业模式
├── 竞争分析
├── 团队
├── 财务预测
├── 融资需求
└── 检查点：整体逻辑是否通顺
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 用户只说"帮我写BP" | 询问项目基本信息：项目是做什么的？目标融资多少？ |
| 没有产品描述 | 提供通用产品框架，引导用户填充 |
| 没有行业数据 | 使用公开数据源（艾瑞、36kr等），注明估算 |
| 没有团队信息 | 先写"核心团队待组建"，后续补充 |

### 输入格式错误

| 场景 | 处理方式 |
|------|----------|
| 行业分类模糊 | 提供行业细分选项 |
| 融资阶段不明确 | 解释各阶段特点，帮助确认 |
| 数据明显不合理 | 提出质疑，请用户确认或修正 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 纯概念阶段（无产品） | 侧重市场分析和团队背景，突出想象空间 |
| 成熟业务融资 | 侧重财务数据和增长曲线，强调确定性 |
| 多元化业务 | 拆分为独立业务单元分别分析 |
| 已有融资历史 | 需说明历史估值和股东情况 |

## 5. 错误处理（Error Handling）

### 常见错误及处理

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 市场数据夸大 | "千亿级市场"、"万亿级蓝海" | 引入第三方数据源，区分TAM/SAM/SOM |
| 竞争分析失真 | "没有竞争对手"、"遥遥领先" | 引导客观分析竞品优势 |
| 财务预测过于乐观 | 每年200%增长 | 引入行业基准，对标成熟公司 |
| 团队经历虚构 | CEO"连续成功创业者" | 强调可验证的背景 |
| 商业模式不清晰 | "生态化反"、"闭环"等空洞词 | 用具体数字解释如何赚钱 |

### 数据验证清单

- [ ] 市场规模数据有明确来源
- [ ] 竞品信息与公开资料一致
- [ ] 财务假设有行业对标
- [ ] 团队背景可验证
- [ ] 时间线符合业务逻辑

## 7. 多个示例（Multiple Examples）

### 典型案例：SaaS产品融资

**输入**：
```
产品：企业协同办公SaaS
行业：企业服务
阶段：Pre-A
融资需求：1500万
已有数据：付费客户50家，MRR 15万
```

**输出片段**：
```
【核心亮点】
1. 差异化：专注制造业协同场景，解决ERP与办公软件割裂问题
2. 数据验证：50家付费客户，续费率95%，NPS 60+
3. 团队：核心成员来自用友、金蝶

【财务预测】
| 指标 | 2024 | 2025 | 2026 |
| 营收 | 300万 | 1200万 | 3600万 |
| 客户数 | 120 | 400 | 1000 |
| ARPU | 2.5万 | 3万 | 3.6万 |
```

### 边界案例：纯概念阶段项目

**输入**：
```
项目：AI驱动的智能投顾平台
阶段：天使轮
需求：500万
现状：只有idea，团队3人
```

**处理策略**：
- 侧重团队背景和技术壁垒
- 市场规模分析为主
- 财务预测标注"基于假设"
- 强调团队可执行性

---

### 失败案例：某生鲜电商BP问题分析

**问题BP特征**：
- 市场数据："千亿级生鲜市场"
- 竞争优势："我们比叮咚便宜"
- 财务预测：月复合增长30%
- 团队："暂无"

**问题诊断**：
- 市场数据未区分SAM
- 竞争优势不可持续
- 增长假设违背商业常识
- 团队空白是大减分项

**修正建议**：
- 聚焦细分市场（如母婴群体有机食材）
- 说明具体差异化（如产地直采、冷链优势）
- 引入行业基准调整预测
- 补充团队信息或顾问资源

## 8. FAQ（常见问题）

### Q1：BP长度多少合适？
**A**：种子轮/天使轮8-15页，Pre-A/A轮15-25页。核心原则：足够展示关键信息，不冗余。执行摘要永远不超过2页。

### Q2：没有财务数据怎么写？
**A**：
1. 提供行业基准数据（如SaaS的LTV/CAC基准）
2. 基于假设建模，标注"预测"
3. 随着业务发展迭代更新

### Q3：竞品分析会不会暴露劣势？
**A**：不会。投资人见多识广，刻意回避反而显得不专业。客观分析竞品，展示差异化思考，更能赢得信任。

### Q4：估值怎么定？
**A**：
- 早期：参考同类公司融资估值
- 成长期：使用营收/用户量倍数
- 成熟期：DCF、PE等方法
- 原则：有依据、可解释

### Q5：BP需要哪些附件？
**A**：
- 详细财务模型（Excel）
- 专利/资质证书
- 合同/订单意向
- 团队简历
- 产品截图/演示视频
