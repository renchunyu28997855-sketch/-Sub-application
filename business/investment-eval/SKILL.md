# investment-eval

## 功能说明
帮助评估投资标的的价值，包括早期项目和成熟企业。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有明确的评估标的（公司/项目）
- 有一定的财务或运营数据
- 评估目的明确（投资/融资/收购）
- 市场信息可获取

### 适用范围
- 早期项目估值（天使轮、A轮、B轮）
- 成长期公司估值
- 投资尽职调查
- 融资定价参考
- 收购估值参考
- 内部项目评估

### 不适用场景
- 上市公司正式估值（需专业机构）
- 复杂金融工具估值
- 需要审计的正式估值报告
- 法律诉讼相关的估值

### 数据假设
- 财务/运营数据真实可靠
- 市场数据具有参考价值
- 未来预测基于合理假设

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：信息收集（第1-2步）

```
步骤1：收集基础信息
├── 公司/项目基本情况
├── 商业模式
├── 团队背景
├── 发展历程
└── 检查点：基本信息是否完整

步骤2：收集财务/运营数据
├── 历史财务数据
├── 当前运营数据
├── 未来预测数据
├── 融资历史
└── 检查点：数据是否可信
```

### 阶段二：行业分析（第3-4步）

```
步骤3：行业分析
├── 行业规模与增长
├── 行业竞争格局
├── 行业发展趋势
├── 行业估值基准
└── 检查点：行业定位是否清晰

步骤4：可比公司分析
├── 选择可比公司
├── 收集可比公司数据
├── 计算可比指标
└── 检查点：可比公司是否合适
```

### 阶段三：估值计算（第5-7步）

```
步骤5：选择估值方法
├── 早期项目：用户量、GMV、早期倍数
├── 成长型：P/S、EV/EBITDA
├── 成熟期：P/E、DCF
├── 特殊情况：资产法、清算价值
└── 检查点：方法是否适合阶段

步骤6：多方法估值
├── 方法1估值及计算过程
├── 方法2估值及计算过程
├── 方法3估值及计算过程
└── 检查点：计算是否正确

步骤7：综合估值
├── 各方法加权
├── 估值区间确定
├── 最终估值结论
└── 检查点：估值是否合理
```

### 阶段四：投资分析（第8-9步）

```
步骤8：投资条款建议
├── 估值建议
├── 股权比例建议
├── 投资保护条款
├── 退出路径
└── 检查点：条款是否合理

步骤9：风险分析
├── 业务风险
├── 财务风险
├── 团队风险
├── 市场风险
└── 检查点：风险是否识别完整
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 只有概念没有数据 | 侧重团队和赛道，用类比法估值 |
| 数据不完整 | 标注数据缺口，给出区间估值 |
| 无法获取竞品数据 | 扩大可比范围或用行业平均 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 纯早期项目（无收入） | 用户数×单用户价值，PS值 |
| 亏损公司 | PS、EV/EBITDA、DCF |
| 成熟稳定公司 | PE、DCF为主 |
| 平台型公司 | GMV、交易笔数 |
| SaaS公司 | ARR倍数 |

### 不同阶段估值

| 阶段 | 常用方法 |
|------|----------|
| 种子轮 | 类比估值、团队估值 |
| 天使轮 | 用户估值、PS估值 |
| A轮 | PS、ARR倍数 |
| B轮 | PS、EV/EBITDA |
| 成长期 | PS、PE、DCF |
| 成熟期 | PE、DCF、PB |

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 方法选择错误 | 成熟方法估早期项目 | 根据阶段选择方法 |
| 可比公司不当 | 可比公司差异大 | 调整可比公司 |
| 假设过于乐观 | 增长假设违背常识 | 保守估计 |
| 忽视风险 | 只看收益 | 风险调整 |
| 忽视稀释 | 不考虑后续融资 | 考虑完全稀释 |

### 估值陷阱

| 陷阱 | 说明 | 避免方式 |
|------|------|----------|
| 虚假繁荣 | 用户数虚高 | 验证数据真实性 |
| 对标错误 | 和巨头对标 | 选择合适可比 |
| 线性外推 | 假设持续高增长 | 考虑市场天花板 |
| 忽视稀释 | 只看本轮稀释 | 考虑后续稀释 |

## 3. 详细估值计算

### 3.1 ARR倍数法（主要方法）

**目标公司ARR**：2023年实际ARR = 2000万
**行业平均倍数**：15x
**估值** = 2000万 × 15 = 3亿

### 3.2 用户估值法（辅助验证）

| 指标 | 数值 |
|------|------|
| 注册用户 | 50万 |
| 付费用户 | 2万 |
| 付费率 | 4% |
| ARPU | 1000元/年 |
| 单用户价值（LTV） | 5000元 |
| 用户总价值 | 50万 × 4% × 5000 = 1亿 |

### 3.3 DCF验证（保守估计）

假设：
- 未来3年营收增长80%/50%/40%
- 永续增长率3%
- 折现率15%

| 年份 | 营收 | 折现因子 | 现值 |
|------|------|---------|------|
| 2024 | 3600万 | 0.87 | 3132万 |
| 2025 | 5400万 | 0.76 | 4104万 |
| 2026 | 7560万 | 0.66 | 4989万 |
| 永续 | - | - | 2亿 |
| **DCF估值** | | | **3.2亿** |

---

## 4. 估值结论

| 方法 | 估值 | 权重 | 加权估值 |
|------|------|------|---------|
| ARR倍数法 | 3亿 | 50% | 1.5亿 |
| DCF | 3.2亿 | 30% | 0.96亿 |
| 用户估值 | 1亿 | 20% | 0.2亿 |

**综合估值**：约2.7亿，**建议本轮估值2.5-3亿**

### 失败案例：某估值报告问题

**问题特征**：
- 用PE给亏损公司估值
- 可比公司选择BAT
- 假设年增长100%持续5年
- 没有风险分析

**问题诊断**：
- 方法选择错误
- 可比公司不当
- 假设过于乐观
- 风险识别缺失

**修正方向**：
- 根据阶段选择方法
- 选择可比公司
- 假设回归理性
- 补充风险分析

## 8. FAQ（常见问题）

### Q1：早期项目怎么估值？
**A**：
- 团队因素占比高
- 赛道和市场规模
- 技术壁垒
- 类比法（参考同类融资）
- 常见倍数：用户数×单用户价值

### Q2：估值多久调整一次？
**A**：
- 每轮融资重新估值
- 业务重大变化时重估
- 外部环境重大变化时评估

### Q3：估值和投资条款什么关系？
**A**：
- 估值决定股权比例
- 投资条款保护投资人权利
- 估值高：创始团队股权保留多，但条件可能苛刻
- 估值低：股权稀释多，但条款可能宽松

### Q4：如何避免估值过高？
**A**：
- 合理假设增长
- 选择合适的可比公司
- 考虑风险因素
- 考虑后续融资稀释

### Q5：估值方法怎么选？
**A**：
| 阶段 | 推荐方法 |
|------|----------|
| 早期 | 类比估值、用户估值 |
| 成长 | PS、EV/EBITDA |
| 成熟 | PE、DCF |

### Q6：如何判断估值是否合理？
**A**：
- 与可比公司对比
- 与历史融资对比
- 与市场环境对比
- 考虑增长预期
