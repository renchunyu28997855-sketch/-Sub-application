# contract-review

## 功能说明
帮助审查合同条款，识别法律风险和商业风险，提供修改建议。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有明确的合同文本
- 合同类型已知
- 签约目的和背景明确
- 签约双方主体资格可验证

### 适用范围
- 商务合作合同
- 技术服务合同
- 软件开发合同
- 采购合同
- 销售合同
- 租赁合同
- 人力资源相关合同
- 投资/融资相关合同

### 不适用场景
- 需要律师执业的法律服务（应转介专业律师）
- 重大复杂的并购/IPO合同
- 知识产权诉讼相关
- 劳动仲裁/诉讼案件
- 需要公证/见证的正式文件

### 审查假设
- 审查是基于一般商业实践
- 不构成正式法律意见
- 最终法律风险由专业律师判断
- 涉及重大利益时应咨询律师

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：合同了解（第1-2步）

```
步骤1：了解合同背景
├── 签约目的和背景
├── 合同类型和性质
├── 涉及金额和重要程度
├── 双方关系和谈判地位
└── 检查点：是否理解合同背景

步骤2：获取合同文本
├── 获取完整合同文本
├── 确认版本（草稿/正式版）
├── 确认附件和补充协议
└── 检查点：文本是否完整
```

### 阶段二：合同审查（第3-5步）

```
步骤3：形式审查
├── 合同结构完整性
├── 签章是否齐全
├── 生效条件是否满足
├── 主体资格是否合法
└── 检查点：形式是否符合要求

步骤4：实质条款审查
├── 标的条款（货物/服务描述）
├── 价格条款（金额、付款方式）
├── 履行条款（时间、地点、方式）
├── 违约条款（责任、赔偿）
├── 终止条款（解除条件）
├── 争议条款（管辖、仲裁）
└── 检查点：条款是否清晰合理

步骤5：风险识别
├── 识别高风险条款
├── 识别中等风险条款
├── 识别低风险条款
├── 识别不明确条款
└── 检查点：风险是否识别完整
```

### 阶段三：修改建议（第6-7步）

```
步骤6：制定修改建议
├── 针对高风险条款
├── 针对中风险条款
├── 补充必要条款
└── 检查点：建议是否可行

步骤7：优先级排序
├── P0级（必须修改）
├── P1级（强烈建议）
├── P2级（建议考虑）
└── 检查点：优先级是否合理
```

### 阶段四：报告输出（第8步）

```
步骤8：生成审查报告
├── 审查结论概述
├── 风险清单详述
├── 修改建议
├── 签约建议
└── 检查点：报告是否完整清晰
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 只有口头协议 | 提醒用户需要书面合同 |
| 合同文本不完整 | 指出缺失部分，建议补充 |
| 对方提供的是格式合同 | 提醒注意格式条款风险 |
| 合同已签署后发现问题 | 评估补救措施的可能性 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 合同金额巨大 | 建议聘请专业律师 |
| 涉及知识产权 | 重点审查IP条款 |
| 涉及保密条款 | 审查保密范围和期限 |
| 涉及竞业限制 | 审查合理性 |
| 跨境合同 | 注意法律适用和管辖 |

### 条款争议场景

| 场景 | 处理方式 |
|------|----------|
| 条款模糊 | 要求明确或提供解释 |
| 显失公平 | 指出问题，要求修改 |
| 违法条款 | 指出违法性，建议删除 |
| 缺失必要条款 | 建议补充 |

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 遗漏重要条款 | 签约后产生争议 | 建立审查清单 |
| 条款理解偏差 | 对条款含义理解不一致 | 明确条款含义 |
| 口头承诺未入合同 | 对方否认 | 补充书面确认 |
| 格式条款风险 | 对方提供格式合同 | 重点审查格式条款 |

### 风险等级判断

| 风险等级 | 判断标准 | 处理方式 |
|----------|----------|----------|
| 🔴高 | 可能导致重大损失 | 必须修改 |
| 🟡中 | 可能导致一般损失 | 建议修改 |
| 🟢低 | 影响有限 | 注意即可 |

```
## 风险清单

### 🔴 高风险（必须修改）

**风险1：知识产权归属模糊**
- **条款内容**：乙方完成的工作成果归甲方所有
- **问题**：未明确"工作成果"定义，范围过宽可能损害乙方权益
- **建议修改**：
  ```
  乙方为完成本合同工作成果所形成的源代码、文档、设计等，
  在甲方支付全部合同款项后，归甲方所有。
  乙方在开发过程中使用的自有技术、通用工具不在此限。
  ```

### 🟡 中风险（建议修改）

**风险2：违约责任过重**
- **条款内容**：任何一方违约，违约方需赔偿对方全部损失
- **问题**："全部损失"无上限，可能导致责任过重
- **建议修改**：
  ```
  任何一方违约给对方造成损失的，违约方应赔偿对方直接损失，
  赔偿金额不超过本合同总金额的200%。
  ```

**风险3：验收标准不明确**
- **条款内容**：甲方应在收到工作成果后7日内验收
- **问题**：未明确验收标准和流程
- **建议修改**：
  ```
  乙方交付工作成果后，甲方应在5个工作日内完成验收测试。
  验收标准以双方确认的《功能需求文档》为准。
  如有缺陷，乙方应在5个工作日内修复。
  ```

---

## 修改建议汇总

| 条款 | 原内容 | 建议修改 | 优先级 |
|------|--------|---------|--------|
| 知识产权 | 模糊 | 明确界定自有技术vs项目成果 | P0 |
| 违约责任 | 无上限 | 增加赔偿上限 | P1 |
| 验收标准 | 不明确 | 以需求文档为准 | P1 |
| 付款节点 | 不清晰 | 明确里程碑付款 | P2 |
| 保密条款 | 无期限 | 增加保密期限 | P2 |

---

## 签约建议

1. **法务审核**：涉及金额超过50万的合同，建议法务参与
2. **印章管理**：公章使用需登记，最好双人在场
3. **合同保管**：签署后双方各留一份，原件妥善保管
```

## 8. FAQ（常见问题）

### Q1：合同应该审查哪些方面？
**A**：
- 主体资格（是否有签约资格）
- 标的条款（货物/服务是否明确）
- 价格条款（金额、付款方式）
- 履行条款（时间、地点、方式）
- 质量条款（验收标准）
- 违约条款（责任分配）
- 终止条款（解除条件）
- 争议条款（管辖、仲裁）

### Q2：哪些条款是必须有的？
**A**：
- 合同双方
- 标的（货物/服务）
- 数量
- 价格
- 履行时间
- 违约责任
- 争议解决

### Q3：格式合同要注意什么？
**A**：
- 仔细阅读格式条款
- 注意限制责任条款
- 注意免责条款
- 对方已填写的条款
- 争取修改不平等条款

### Q4：口头承诺有法律效力吗？
**A**：
- 口头承诺也有法律效力，但举证困难
- 重要承诺应书面确认
- 合同签订前的往来邮件也可能是合同一部分

### Q5：签章要注意什么？
**A**：
- 确认对方签章人有权代表
- 企业公章需确认真实性
- 合同多页应盖骑缝章
- 签署后保留原件

### Q6：发现合同风险怎么办？
**A**：
1. 评估风险等级
2. 提出修改建议
3. 与对方协商
4. 必要时拒绝签约
5. 重大风险咨询律师
