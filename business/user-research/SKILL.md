# user-research

## 功能说明
帮助设计用户调研问卷和访谈提纲，优化问题设计，确保收集到高质量的用户洞察。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有明确的调研目的
- 用户能定义目标用户群体
- 用户有获取用户的渠道或预算
- 调研结果将用于实际决策（非学术研究）

### 适用范围
- 新产品开发前的需求调研
- 产品迭代优化方向的探索
- 用户满意度评估
- 付费意愿和定价研究
- 用户画像构建
- 竞品用户体验对比

### 不适用场景
- 需要统计学显著性的学术研究
- 需要随机对照实验的因果推断
- 需要政府审批的敏感调研（如医疗、金融合规）
- 需要长期纵向追踪的纵向研究

### 调研设计假设
- 问卷调研适用于大样本定量分析（n≥100）
- 深度访谈适用于探索性定性研究（n=5-15）
- 两者结合可获得更全面的洞察

## 2. 输入输出（I/O Format）

### 输入格式

**必需信息**：
```
调研目的：[描述]

调研方法：[问卷/访谈/可用性测试/混合]
样本量要求：[如：有效问卷≥300份]
```

**可选补充**：

- 竞品或行业参考

- 已有用户数据

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：调研规划（第1-3步）

```
步骤1：明确调研目的
├── 核心问题是什么？
├── 调研结果用于什么决策？
├── 需要回答的关键假设？
└── 检查点：目的是否具体可衡量

步骤2：定义目标用户
├── 用户画像（人口统计特征）
├── 行为特征（使用习惯、付费行为）
├── 渠道特征（从哪里触达）
└── 检查点：用户定义是否清晰

步骤3：选择调研方法
├── 定量调研（问卷）：验证假设、测量比例
├── 定性调研（访谈）：探索原因、深度洞察
├── 可用性测试：评估产品易用性
├── 混合方法：先定性探索，后定量验证
└── 检查点：方法是否适合目的
```

### 阶段二：问卷设计（第4-6步）

```
步骤4：设计问卷结构
├── 筛选问题（确认目标用户）
├── 基本信息（人口统计）
├── 核心问题（直接回答目的）
├── 辅助问题（增加洞察）
├── 开放问题（收集意外发现）
└── 检查点：问题是否覆盖所有目的

步骤5：优化问题设计
├── 问题类型选择（单选/多选/量表/排序）
├── 选项设计（穷尽、互斥、平衡）
├── 措辞优化（中立、清晰、无引导）
├── 顺序效应（逻辑分组、难度递进）
└── 检查点：是否有引导性/模糊问题

步骤6：设计辅助内容
├── 问卷说明/开场白
├── 进度指示
├── 感谢语/奖励设置
└── 检查点：用户体验是否顺畅
```

### 阶段三：访谈设计（第7-8步）

```
步骤7：设计访谈提纲
├── 暖场问题（建立 rapport）
├── 核心问题（围绕目的）
├── 探索问题（追问深入）
├── 收尾问题（补充或展望）
└── 检查点：时间控制（30分钟/60分钟）

步骤8：访谈技巧准备
├── 追问技巧（5个为什么）
├── 中立引导（避免暗示）
├── 记录方式（录音/笔记）
└── 检查点：访谈员是否培训到位
```

### 阶段四：小范围测试（第9步）

```
步骤9：预测试与修改
├── 问卷：小范围投放（20-30份）
├── 访谈：2-3人试访
├── 识别问题：
│   - 选项不完善
│   - 问题歧义
│   - 时间过长
│   - 逻辑跳转问题
└── 检查点：正式执行前修改到位
```

### 阶段五：执行与收集（第10步）

```
步骤10：执行调研
├── 问卷投放：渠道选择、激励设置
├── 访谈邀约：筛选条件、预约安排
├── 质量控制：筛选无效问卷、录音转文字
└── 检查点：样本是否满足要求
```

### 阶段六：分析与输出（第11步）

```
步骤11：数据分析与洞察
├── 定量：描述统计、交叉分析
├── 定性：编码、主题归纳
├──  triangulation：多来源交叉验证
└── 检查点：结论是否有数据支撑
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 用户说"随便做一下调研" | 询问具体目的，明确调研将用于什么决策 |
| 没有目标用户定义 | 提供用户画像模板，引导用户填充 |
| 调研目的模糊 | 提供常见调研目的选项 |
| 无法触达目标用户 | 提供用户获取渠道建议或样本服务信息 |

### 输入格式错误

| 场景 | 处理方式 |
|------|----------|
| 问题过于宽泛 | 拆分为具体问题 |
| 问题有逻辑矛盾 | 指出并修正 |
| 选项不穷尽 | 添加"其他"选项 |
| 选项互相包含 | 重新分类确保互斥 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 敏感话题（收入、政治） | 使用模糊区间，解释收集目的 |
| 专业领域（B2B） | 增加筛选条件，确保受访者资格 |
| 小众用户群体 | 降低样本量要求，深度访谈为主 |
| 需要对比用户群 | 设计对比维度，控制样本结构 |

## 5. 错误处理（Error Handling）

### 常见问题

| 问题类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 问卷过长 | 完成率<70% | 精简题目，控制在10分钟内 |
| 选项偏差 | 一边倒选择 | 检查选项平衡性和中立性 |
| 引导性问题 | 暗示期望答案 | 改写为中性表述 |
| 幸存者偏差 | 样本都是活跃用户 | 设置筛选问题排除 |
| 社交期望偏差 | 都选正向选项 | 加入反向问题验证 |

### 质量控制

| 控制点 | 方法 |
|--------|------|
| 问卷有效性 | 设置注意力检测题 |
| 访谈有效性 | 追问验证理解一致性 |
| 样本代表性 | 与目标用户结构对比 |
| 回答真实性 | 交叉验证多个问题 |

## 6. 质量验证清单（Quality Checklist）

### 问卷设计

- [ ] 问卷时长≤10分钟（除非用户接受更长）
- [ ] 开头有明确说明和激励说明
- [ ] 筛选问题能有效识别目标用户
- [ ] 问题无引导性、无歧义
- [ ] 选项穷尽且互斥
- [ ] 有逻辑跳转且正确
- [ ] 有质量检测题（陷阱题/反向题）
- [ ] 结束语友好且有后续引导

### 访谈设计

- [ ] 提纲结构清晰（暖场→核心→探索→收尾）
- [ ] 核心问题直接回答调研目的
- [ ] 有追问提示和备用问题
- [ ] 预计时长合理（30/60分钟）
- [ ] 访谈员了解业务背景

### 执行设计

- [ ] 样本量满足统计要求
- [ ] 渠道覆盖目标用户
- [ ] 有激励措施促进完成
- [ ] 匿名性/隐私保护说明清楚
- [ ] 发放时间考虑用户习惯

### 分析计划

- [ ] 预设分析维度和交叉项
- [ ] 开放题有编码框架
- [ ] 定量问题有图表呈现计划

## 7. 多个示例（Multiple Examples）

### 典型案例：知识付费产品调研

**输入**：
```
目的：了解用户付费意愿和影响因素
目标用户：22-40岁，有学习需求的人群
方法：问卷（300份）+访谈（10人）
```

**问卷设计片段**：
```
【问卷设计】（约15题，10分钟）

### 第一部分：基本信息（3题）
1. 您的年龄范围？
2. 您的职业？
3. 您每月可支配收入（不含必要开支）？

### 第二部分：知识付费行为（5题）
4. 您是否购买过知识付费产品？
5. 您主要购买哪些类型？
6. 您每年在知识付费上的支出大约是？
7. 您选择知识付费产品时最看重什么？（排序题）
8. 您不购买/少购买知识付费产品的原因是什么？

### 第三部分：付费意愿（4题）
9. 您能接受一门系统课的最高价格是？
10. 您更倾向于哪种付费方式？
11. 如果有"不满意退款"保障，您的付费意愿会提高多少？
12. 您愿意为哪些增值服务付费？

### 第四部分：开放问题（2题）
13. 您觉得目前知识付费产品最大的问题是什么？
14. 您理想中的知识付费产品是什么样的？
```

**访谈提纲片段**：
```
【访谈提纲】（6个核心问题，30分钟）
1. 先自我介绍一下，包括您的职业和学习习惯
2. 您平时是怎么学习的？有没有用过知识付费产品？
3. 什么样的内容会让您愿意付费？什么样的会让您觉得不值？
4. 您买过的最值的知识产品是什么？为什么觉得值？
5. 如果我们要做一个新的知识产品，您有什么建议？
6. 有什么想补充的吗？
```

## 8. FAQ（常见问题）

### Q1：问卷和访谈各适合什么场景？
**A**：
- **问卷**：验证假设、测量比例、对比选项、收集统计数据
- **访谈**：探索原因、深度洞察、发现意外、复杂问题追问
- **原则**：先定性探索（5-10人访谈），再定量验证（问卷）

### Q2：样本量多少才够？
**A**：
- **问卷**：定量分析通常需要n≥100，交叉分析需要更多
- **访谈**：定性研究5-15人通常足够，主题饱和后停止
- **计算器**：如需误差±5%，置信度95%，需要约400份

### Q3：如何提高问卷回收率？
**A**：
- 精简问卷（≤10分钟）
- 设置激励（抽奖/优惠券/资料）
- 选择合适渠道（用户聚集地）
- 提醒和跟进（24小时/72小时）
- 匿名保证

### Q4：访谈数据怎么分析？
**A**：
1. 转录录音为文字
2. 编码：标注关键主题
3. 归类：合并相似主题
4. 量化：统计各主题出现频次
5. 验证：多编码者交叉验证
6. 呈现：引用原话+统计

### Q5：调研结果和预期不符怎么办？
**A**：
- 首先检查数据质量（样本是否偏差）
- 其次检查问题设计（是否有效测量）
- 最后诚实呈现：可能发现了新洞察
- 不要强行拟合预期

### Q6：用户说一套做一套怎么办？
**A**：
- 结合行为数据验证（如果有）
- 访谈中追问具体场景而非抽象态度
- 使用假设情境问题
- 注意"政治正确"回答
