# competitor-analysis

## 功能说明
系统性地分析竞争对手，了解市场竞争格局，为产品决策提供支持。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有明确的分析目的（产品规划、融资、竞争策略等）
- 用户需说明分析范围（直接竞品/间接竞品/全行业）
- 涉及特定公司时，用户需确认信息来源的授权
- 市场研究场景下，用户需说明目标市场定义

### 适用范围
- 新产品立项前的市场调研
- 现有产品的竞争策略制定
- 融资或战略规划中的竞争分析章节
- 行业研究和企业对标分析
- 定价策略参考

### 不适用场景
- 需要实地商业间谍行为的情况
- 需要获取竞争对手内部机密信息
- 法律诉讼所需的证据收集

### 数据来源假设
- 公开信息：官网、新闻稿、行业报告、用户评价
- 用户提供的信息：直接体验、用户反馈、内部调研
- 估算推演：基于公开数据的合理推断

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：范围界定（第1-2步）

```
步骤1：明确分析边界
├── 确定目标市场定义
├── 识别直接竞品（满足同需求）
├── 识别间接竞品（满足同用户群）
├── 识别潜在替代品
└── 检查点：竞品列表是否完整

步骤2：信息来源规划
├── 官方渠道：官网、产品、PR
├── 用户渠道：评价、论坛、社媒
├── 行业渠道：报告、新闻、研究
└── 检查点：信息渠道是否多元可靠
```

### 阶段二：信息收集（第3-4步）

```
步骤3：基础信息收集
├── 产品功能清单
├── 定价方案
├── 目标用户定位
├── 商业模式
├── 公司背景（融资、团队）
└── 检查点：基本信息是否完整

步骤4：深度信息挖掘
├── 用户评价分析（App Store/知乎）
├── 功能细节体验
├── 内容/社区质量
├── 技术架构推测
└── 检查点：是否有独家洞察
```

### 阶段三：分析框架（第5-7步）

```
步骤5：多维度对比分析
├── 产品维度：功能、体验、技术
├── 市场维度：定位、渠道、品牌
├── 商业维度：定价、收入、盈利
├── 用户维度：评价、满意度、忠诚度
└── 检查点：对比是否客观全面

步骤6：SWOT分析
├── 各竞品优势（Strengths）
├── 各竞品劣势（Weaknesses）
├── 市场机会（Opportunities）
├── 市场威胁（Threats）
└── 检查点：分析是否有洞察

步骤7：竞争格局绘制
├── 市场地图（定位图）
├── 竞争壁垒分析
├── 差异化空间识别
└── 检查点：是否清晰呈现格局
```

### 阶段四：洞察输出（第8-9步）

```
步骤8：机会与策略建议
├── 市场空白点
├── 差异化切入点
├── 威胁规避策略
└── 检查点：建议是否可执行

步骤9：风险提示
├── 来自竞品的威胁
├── 来自新进入者的威胁
├── 来自替代品的威胁
└── 检查点：风险是否已识别
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 用户只说"分析竞品" | 询问：什么产品/市场？分析目的是什么？ |
| 无明确竞品名单 | 从行业分类出发，列出主要玩家 |
| 只有一家竞品 | 扩大范围，引入间接竞品和替代品 |
| 竞品是大型生态 | 聚焦与自身定位重叠的部分 |

### 输入格式错误

| 场景 | 处理方式 |
|------|----------|
| 竞品名称错误 | 通过谐音或描述识别正确竞品 |
| 市场定义过宽 | 建议聚焦细分市场 |
| 分析目的模糊 | 列举常见目的供选择 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 竞品是上市公司 | 可用财报数据，分析更深入 |
| 竞品是内部产品 | 只能基于公开信息和推测 |
| 竞品是海外产品 | 关注国内落地情况和本地化 |
| 行业处于剧变期 | 强调不确定性和持续跟踪 |

---

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 信息来源单一 | 全部来自竞品官网 | 引入用户评价、第三方报告 |
| 缺乏时间维度 | "竞品现状"无历史变化 | 加入时间轴，对比演变 |
| 结论过于主观 | "我们比竞品好很多" | 用数据和用户评价支撑 |
| 忽视间接竞品 | "这个赛道只有我们" | 扩大竞品定义 |
| 分析深度不足 | 停留在功能列表 | 深入到体验、策略层面 |

### 数据验证

| 验证项 | 方法 |
|--------|------|
| 定价信息 | 多渠道核实（官网/客服/第三方） |
| 用户数据 | 引用权威来源，标注置信度 |
| 功能描述 | 实际体验验证或多方印证 |
| 融资信息 | 天眼查/IT桔子/新闻 |

### 边界案例：垄断市场分析

**输入**：
```
目的：寻找切入机会
市场：某垂直领域（假设：在线教育语文赛道）
现状：一家独大（70%份额）
```

**处理策略**：
- 分析巨头优势来源（是先发还是渠道？）
- 识别巨头不愿做/做不好的细分
- 寻找差异化定位（年级/地区/内容类型）
- 评估切入成本和风险

---

### 失败案例：某竞品分析报告问题

**问题特征**：
- 竞品列表只有3家，遗漏重要玩家
- 功能对比表只有"有/无"
- 结论："我们完胜竞品"
- 无数据来源标注

**问题诊断**：
- 竞品范围界定过窄
- 分析深度不足
- 缺乏客观性
- 报告可信度低

**修正建议**：
- 扩大竞品范围
- 细化功能对比维度
- 引入用户评价数据
- 客观呈现优劣势

## 8. FAQ（常见问题）

### Q1：竞品分析需要分析几家？
**A**：直接竞品3-5家必须覆盖，间接竞品和替代品可列出。重点不在数量，在于分析深度和洞察质量。

### Q2：信息从哪来？
**A**：
- 一手信息：产品体验、用户访谈
- 二手信息：官网、新闻、行业报告、App评价
- 推演信息：基于公开数据的合理推断
- 注意：不获取也不使用非法获取的信息

### Q3：分析深度怎么把握？
**A**：
- 快速概览：功能、定价、定位（1-2天）
- 深度分析：加用户体验、商业模式、策略（1周）
- 全面调研：加入技术架构、团队、组织架构（2周+）

### Q4：竞品分析多久更新一次？
**A**：
- 重点竞品：季度更新
- 一般竞品：半年度更新
- 行业变化快时：月度跟踪

### Q5：如何发现蓝海市场？
**A**：
- 分析现有竞品的共同弱点
- 寻找被忽视的用户群体
- 发现新兴需求或技术变革
- 考虑非连续性创新机会
