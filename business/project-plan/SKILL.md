## 工作流程（渐进式披露）

### 阶段一：理解

```
1. 明确项目背景 → 解决什么问题
2. 确定项目范围 → 包含/不包含
3. 识别干系人 → 团队/决策者/业务方
```

### 阶段二：分解

```
4. WBS任务分解 → 阶段/任务/子任务
5. 任务关联分析 → 依赖关系/关键路径
6. 工期估算 → 工作量/资源需求
```

### 阶段三：规划

```
7. 制定时间表 → 甘特图/里程碑
8. 资源分配 → 人力/预算
9. 风险识别 → 风险清单/应对策略
```

## 常见问题

**Q: 计划应该多详细？**
分解到2-5天任务，有明确交付物。不需要到小时级。

**Q: 敏捷项目还需要详细计划吗？**
Epic/里程碑仍需提前确定，Sprint内可灵活调整。

**Q: 需求不确定怎么办？**
分期交付，先做MVP验证方向。计划中标注"待确认"。
