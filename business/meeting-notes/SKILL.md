# meeting-notes

## 功能说明
帮助整理会议内容，生成清晰的会议纪要，包括讨论要点、决策结论、行动项和责任人。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有会议相关信息（录音、笔记、议程）
- 会议有明确的参与者和目的
- 会议涉及决策或行动事项
- 用户需要生成正式纪要

### 适用范围
- 周会/月会纪要
- 项目评审会议
- 客户沟通会议
- 头脑风暴会议
- 产品评审会议
- 技术方案评审
- 商务谈判会议
- 培训/分享会议

### 不适用场景
- 高度机密的董事会会议
- 需要法律效力的正式会议记录
- 心理咨询或医疗相关会议
- 需要公证的会议记录

### 信息来源假设
- 用户可以提供会议录音/视频/笔记
- 用户可以补充会议背景信息
- 参与者信息可获取

## 2. 输入输出（I/O Format）

### 输入格式

**必需信息**：
```
会议主题：[主题名称]
会议时间：[日期时间]
参会人员：[人员列表]
会议目的：[简要说明]
```

**可选补充**：
- 会议议程
- 会议录音/视频
- 会议前的相关文档
- 之前的会议纪要（延续性会议）

### 输出格式

```
【会议主题】XXX
【会议时间】XXX
【参会人员】
【缺席人员】
【会议纪要】
【决策汇总】
【行动项】
【下次会议】
...
```

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：信息收集（第1-2步）

```
步骤1：收集会议信息
├── 获取会议录音/视频
├── 获取会议笔记（如有）
├── 获取会议议程
├── 了解会议背景
└── 检查点：信息是否完整

步骤2：确认基本信息
├── 会议主题
├── 时间地点
├── 参会人员（确认名单）
├── 会议目的
└── 检查点：基本信息是否准确
```

### 阶段二：内容整理（第3-4步）

```
步骤3：梳理讨论内容
├── 按议程或时间顺序整理
├── 提取每个议题的讨论要点
├── 记录关键发言（必要时）
├── 标注争议点和共识点
└── 检查点：内容是否完整

步骤4：提取决策事项
├── 识别明确的决策
├── 记录决策内容和依据
├── 确认决策人
└── 检查点：决策是否清晰
```

### 阶段三：行动项整理（第5-6步）

```
步骤5：识别行动项
├── 识别所有行动项
├── 明确行动内容
├── 确定负责人
├── 确定截止日期
└── 检查点：行动项是否明确

步骤6：确认行动项
├── 与相关人员确认
├── 检查依赖关系
├── 确认优先级
└── 检查点：行动项是否可执行
```

### 阶段四：纪要生成（第7-8步）

```
步骤7：生成正式纪要
├── 按模板格式整理
├── 清晰呈现决策和行动项
├── 明确标记负责人和截止日期
├── 添加必要的上下文
└── 检查点：格式是否规范

步骤8：审核和发送
├── 自审一遍
├── 发送相关人员确认
├── 根据反馈修改
├── 最终发送并存档
└── 检查点：是否获得确认
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 只有录音 | 转录后整理，标注不确定部分 |
| 只有笔记 | 基于笔记整理，请用户确认细节 |
| 没有任何材料 | 询问会议基本信息，根据记忆整理 |
| 会议已结束很久 | 尽快整理，标注可能的信息损失 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 多人参与讨论 | 整理各方观点，标注分歧 |
| 会议结论模糊 | 如实记录，标注需进一步确认 |
| 行动项不明确 | 记录讨论内容，标注需明确 |
| 会议中断/延期 | 如实记录，标注会议未完成 |

### 决策场景

| 场景 | 处理方式 |
|------|----------|
| 决策已明确 | 清晰记录决策内容和决策人 |
| 决策需进一步讨论 | 记录各方观点，标注待决议题 |
| 决策被推翻 | 记录原决策和新决策，标注变更原因 |
| 决策有争议 | 记录各方观点，标注争议点 |

## 5. 错误处理（Error Handling）

### 常见问题

| 问题类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 信息不完整 | 缺失关键讨论点 | 标注"待确认" |
| 记录不准确 | 与参会人记忆不符 | 请参会人确认 |
| 责任人不明确 | 行动项无负责人 | 标注需指定 |
| 截止日期模糊 | "尽快"、"尽快完成" | 建议明确日期 |

### 质量控制

| 控制点 | 方法 |
|--------|------|
| 信息准确性 | 请参会人确认关键内容 |
| 行动项明确性 | 确认负责人和截止日期 |
| 格式规范性 | 按模板检查 |
| 发送及时性 | 会议后24小时内发送 |

## 8. FAQ（常见问题）

### Q1：会议纪要应该包含哪些内容？
**A**：
- 会议基本信息（主题、时间、地点、参会人）
- 讨论要点（按议题整理）
- 决策事项（明确结论）
- 行动项（负责人+截止日期）
- 下次会议安排

### Q2：会议纪要应该谁来写？
**A**：
- 一般由会议主持人指定
- 或由项目经理/助理负责
- 或轮流记录

### Q3：会议纪要多久发送？
**A**：
- 建议：会议结束后24小时内发送初稿
- 重要会议：48小时内发送最终版
- 紧急会议：当天发送

### Q4：如何确保记录完整？
**A**：
- 提前获取议程
- 会议中做笔记
- 录音作为备份
- 会后及时整理

### Q5：行动项如何跟踪？
**A**：
- 在纪要中明确列出
- 定期回顾（如下次会议前）
- 配合项目管理工具
- 未完成项在下次会议中跟进

### Q6：会议纪要如何归档？
**A**：
- 统一存储位置
- 按会议类型/项目分类
- 命名规范（日期+主题）
- 保留历史版本
