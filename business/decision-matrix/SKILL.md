# decision-matrix

## 功能说明
帮助进行系统性决策分析，使用多种分析框架支持决策。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有明确的决策问题
- 有2个或以上的可选方案
- 决策标准和权重可确定
- 有足够的信息支持评估

### 适用范围
- 产品策略决策（功能/技术选型）
- 供应商/合作伙伴选择
- 投资决策（项目/供应商）
- 人员招聘决策
- 战略方向选择
- 资源配置决策

### 不适用场景
- 需要直觉或经验的创意决策
- 信息完全不足的决策
- 需要专业资质的决策（如医疗、法律）
- 紧急危机决策（需走应急流程）

### 分析假设
- 决策因素可以量化或排序
- 决策者偏好是稳定的
- 分析结果仅供参考，最终决策由人做出

## 2. 输入输出（I/O Format）

### 输入格式

**必需信息**：
```
决策问题：[描述要解决的问题]
可选方案：[列出所有选项]
决策目的：[优化什么]
```

**可选补充**：
- 已有方案信息
- 决策约束（预算、时间等）
- 利益相关者
- 风险偏好

### 输出格式

```
【决策问题】XXX
【可选方案】XXX
【分析方法】XXX
【分析过程】
【分析结果】
【决策建议】
...
```

## 3. 详细工作流程（Detailed Workflow）

### 阶段一：问题定义（第1-2步）

```
步骤1：明确决策问题
├── 描述要解决的问题
├── 确定决策目标
├── 明确决策范围
├── 识别决策约束
└── 检查点：问题是否清晰

步骤2：收集方案信息
├── 列出所有可行方案
├── 收集各方案信息
├── 确认方案完整性
├── 排除明显不可行方案
└── 检查点：方案是否完整
```

### 阶段二：分析方法选择（第3-4步）

```
步骤3：选择分析方法
├── 加权评分法：多因素综合评估
├── 利弊分析：列出优缺点
├── 决策树分析：概率和期望值
├── 情景分析：不同情景下的结果
├── SWOT分析：内外部因素
└── 检查点：方法是否适合问题

步骤4：确定评估因素
├── 识别关键评估维度
├── 确定因素权重
├── 确认评估标准
└── 检查点：因素是否全面
```

### 阶段三：分析执行（第5-6步）

```
步骤5：执行分析
├── 加权评分：打分和计算
├── 利弊分析：列出优缺点
├── 情景分析：不同情景预测
├── 风险评估：识别风险
└── 检查点：分析是否客观

步骤6：敏感性分析
├── 测试权重变化影响
├── 测试假设变化影响
├── 识别关键决策因素
└── 检查点：结论是否稳健
```

### 阶段四：结论输出（第7-8步）

```
步骤7：综合结论
├── 汇总各方法结论
├── 考虑决策者偏好
├── 识别不确定因素
├── 提出建议
└── 检查点：建议是否可执行

步骤8：风险和后续
├── 识别决策风险
├── 制定监控计划
├── 确定决策审查机制
└── 检查点：是否有应对措施
```

## 4. 边界情况（Edge Cases）

### 输入为空的情况

| 场景 | 处理方式 |
|------|----------|
| 只有一个方案 | 将"不做"作为备选方案 |
| 方案信息不足 | 标注信息缺口，建议收集 |
| 无法量化因素 | 使用定性排序 |
| 利益冲突 | 识别并标注 |

### 特殊情况

| 场景 | 处理方式 |
|------|----------|
| 多目标冲突 | 使用多目标决策方法 |
| 时间压力 | 简化分析，快速决策 |
| 高风险决策 | 增加分析深度和审核 |
| 团队意见分歧 | 使用Delphi法或投票 |

### 分析异常

| 场景 | 处理方式 |
|------|----------|
| 得分差距很小 | 建议进一步分析或测试 |
| 专家打分差异大 | 讨论并对齐 |
| 结论违背常识 | 检查假设和数据 |

## 5. 错误处理（Error Handling）

### 常见错误

| 错误类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 因素选择偏差 | 只选自己偏好的因素 | 引入多方意见 |
| 权重不合理 | 权重与目标不符 | 重新讨论权重 |
| 打分主观 | 打分无依据 | 明确评分标准 |
| 忽视风险 | 只看收益不看风险 | 加入风险分析 |

### 分析陷阱

| 陷阱 | 说明 | 避免方式 |
|------|------|----------|
| 锚定效应 | 过度依赖初始信息 | 多角度思考 |
| 确认偏差 | 只看支持自己观点的信息 | 主动寻找反面证据 |
| 群体思维 | 团队讨论随大流 | 引入反对意见 |
| 过度自信 | 低估不确定性 | 敏感性分析 |

```
## 1. 加权评分分析

### 1.1 确定评估因素和权重

| 因素 | 权重 | 说明 |
|------|------|------|
| 成本 | 25% | 初始+运维成本 |
| 时间 | 20% | 上线速度 |
| 功能 | 20% | 功能满足度 |
| 安全 | 15% | 数据安全 |
| 扩展 | 10% | 未来扩展性 |
| 维护 | 10% | 运维复杂度 |

### 1.2 评分（1-10分）

| 因素 | 权重 | A自研 | B SaaS | C 开源 |
|------|------|-------|--------|--------|
| 成本 | 25% | 4 | 7 | 8 |
| 时间 | 20% | 3 | 9 | 5 |
| 功能 | 20% | 8 | 6 | 6 |
| 安全 | 15% | 9 | 6 | 7 |
| 扩展 | 10% | 9 | 5 | 8 |
| 维护 | 10% | 4 | 8 | 3 |

### 1.3 加权得分计算

| 方案 | 计算 | 总分 |
|------|------|------|
| A自研 | 4×0.25+3×0.2+8×0.2+9×0.15+9×0.1+4×0.1 | 5.7 |
| B SaaS | 7×0.25+9×0.2+6×0.2+6×0.15+5×0.1+8×0.1 | 6.9 |
| C 开源 | 8×0.25+5×0.2+6×0.2+7×0.15+8×0.1+3×0.1 | 6.3 |

**结论**：SaaS方案综合得分最高

## 2. 利弊分析

### A：自研
**利**：
- 数据完全自主可控
- 可深度定制
- 长期成本可能更低

**弊**：
- 开发周期长（预计6个月）
- 需要专职团队
- 维护成本高
- 技术风险

### B：SaaS
**利**：
- 上线快（1周）
- 无运维成本
- 功能持续更新

**弊**：
- 数据在第三方
- 长期费用可能增加
- 定制受限
- 服务稳定性依赖

### C：开源
**利**：
- 成本低
- 数据自主
- 可定制

**弊**：
- 需要技术团队配置
- 社区版功能有限
- 无官方支持
- 安全漏洞需自行处理

## 3. 情景分析

| 情景 | 概率 | 自研 | SaaS | 开源 |
|------|------|------|------|------|
| 业务快速发展 | 40% | 扩展成本高 | 灵活扩展 | 扩展成本中等 |
| 数据合规要求 | 30% | 最优 | 需评估 | 中等 |
| 预算收紧 | 20% | 成本压力大 | 成本可控 | 中等 |
| 技术人员流失 | 10% | 风险高 | 不受影响 | 风险高 |

---

## 4. 决策建议

### 综合建议：**B（SaaS）**

**理由**：
1. 加权评分最高（6.9分）
2. 上线速度快，符合快速验证需求
3. 无运维成本和团队压力
4. 数据安全可通过合同条款约束

**风险缓解**：
- 选择通过等保认证的SaaS厂商
- 合同中明确数据归属和退出条款
- 重要数据定期导出备份

**如果选择自研**：
- 需要确认有足够的研发资源
- 做好6个月以上的准备
- 建立专门的安全审查机制
```

### 失败案例：某决策分析问题

**问题特征**：
- 评估因素只有成本
- 权重都是平均分配
- 没有敏感性分析
- 结论过于果断

**问题诊断**：
- 分析维度单一
- 缺乏判断
- 忽视不确定性
- 过度自信

**修正方向**：
- 完善评估因素
- 合理设定权重
- 加入敏感性分析
- 承认不确定性

## 8. FAQ（常见问题）

### Q1：如何确定因素权重？
**A**：
- 专家讨论法：专家团队讨论确定
- 层次分析法（AHP）：两两比较确定
- 参照历史：参考类似决策的权重
- 利益相关者访谈：了解各方关注点

### Q2：评分主观怎么办？
**A**：
- 明确评分标准（什么算高分）
- 多方参与评分
- 敏感性分析测试影响
- 定性+定量结合

### Q3：多个方法结论矛盾怎么办？
**A**：
- 检查各方法假设
- 识别核心差异
- 尝试加权综合
- 最终结合判断

### Q4：如何处理高不确定性？
**A**：
- 情景分析（多个情景）
- 敏感性分析（变量波动）
- 区间估计（给出范围）
- 决策树（概率+期望值）

### Q5：团队意见分歧怎么处理？
**A**：
- 引入魔鬼辩护人
- Delphi法匿名投票
- 讨论对齐假设
- 最终决策者决定

### Q6：决策后如何跟踪？
**A**：
- 建立监控指标
- 定期回顾决策
- 记录实际结果
- 总结经验教训
