# excel-formula

## 功能说明
帮助编写复杂的Excel公式，解决实际工作中的数据处理需求。

---

## 假设声明（Assumptions）

1. **Excel版本已知**：不同版本支持不同函数（如XLOOKUP需要Office 365）
2. **数据布局确定**：源数据区域已明确，便于引用
3. **数据类型正确**：数值/文本/日期类型已验证，不会混用
4. **无循环引用**：公式不会导致Excel循环引用错误
5. **性能要求合理**：公式复杂度在Excel性能承受范围内
6. **目标受众**：用户需具备基础Excel操作能力

## 详细工作流程（Detailed Workflow）

### 第一阶段：需求分析
1. **理解业务规则**
   - 明确计算的业务逻辑
   - 确认分段条件、分组条件
   - 识别特殊处理情况（空值、边界值）

2. **分析数据结构**
   - 确认源数据区域
   - 判断是否需要固定引用（$符号）
   - 确定是否需要命名区域

3. **选择函数策略**
   - 基础函数 vs 高级函数
   - 单公式 vs 辅助列
   - 数组公式 vs 普通公式

**检查点**：理解计算逻辑，可复述需求

### 第二阶段：公式编写
4. **构建基础框架**
   - 确定外层函数（如IF、SUM）
   - 确定内层逻辑
   - 逐步填充

5. **处理特殊情况**
   - 使用IFERROR处理错误
   - 使用IF处理空值
   - 使用AND/OR处理复合条件

6. **优化公式结构**
   - 减少嵌套层级
   - 使用LET函数提升可读性（Office 365）
   - 考虑性能影响

### 第三阶段：测试验证
7. **边界值测试**
   - 空值情况
   - 零值情况
   - 负值情况
   - 最大/最小边界

8. **多场景验证**
   - 正常情况
   - 极端情况
   - 错误输入情况

9. **性能检查**
   - 大数据量时的计算速度
   - 是否需要禁用自动计算

## 边界情况（Edge Cases）

### 输入为空的情况
- **空单元格**：用 IF(cell="", ...) 判断并返回预设值
- **整列为空**：使用 IFERROR 避免报错
- **部分数据为空**：使用 AGGREGATE 函数忽略空值

### 输入格式错误的情况
- **文本数字**：用 VALUE() 或 -- 前缀转换
- **日期格式混乱**：用 DATEVALUE() 统一转换
- **多余空格**：用 TRIM() 清除
- **大小写不一致**：用 UPPER()/LOWER() 统一

### 特殊情况处理
- **分段计算**：使用 IFS 或多个 IF 嵌套
- **跨表引用**：使用 INDIRECT 或 3D引用
- **动态范围**：使用 OFFSET、INDEX 或命名管理器
- **数组运算**：使用 SUMPRODUCT 或 Ctrl+Shift+Enter 数组公式

## 错误处理（Error Handling）

| 错误类型 | 可能原因 | 处理方式 |
|---------|---------|---------|
| #N/A | 查找值不存在 | 用 IFERROR 包裹，提供默认值 |
| #DIV/0! | 除数为零 | 用 IF 判断分母，或 IFERROR 捕获 |
| #REF! | 引用无效单元格 | 检查并修正引用范围 |
| #VALUE! | 数据类型不匹配 | 转换数据类型或用 VALUE() |
| #NAME? | 函数名拼写错误 | 检查函数名是否正确 |
| #NUM! | 数值超出范围 | 检查输入值是否合理 |
| #NULL! | 范围引用错误 | 检查是否有多余空格或符号错误 |

## 多个示例（Multiple Examples）

### 典型案例

**场景**：销售提成阶梯计算

**需求**：
```
0-10万：提成5%
10-30万：提成8%
30万以上：提成12%
```

**公式**：
```excel
=IF(A2>=300000, A2*12%, IF(A2>=100000, A2*8%, A2*5%))
```

**更优方案（使用IFS）**：
```excel
=IFS(A2>=300000, A2*12%, A2>=100000, A2*8%, A2<100000, A2*5%)
```

---

### 边界/异常案例

**场景**：处理空值和文本数字

**原始数据**：
```
A列：100, "", 200, "300", 0
```

**安全求和公式**：
```excel
=SUMPRODUCT(IFERROR(VALUE(A1:A5), 0))
```

**分情况处理**：
```excel
=IF(A2="", "无数据", IF(ISNUMBER(A2)=FALSE, "非数值", A2*2))
```

---

### 失败案例及修正

**场景**：VLOOKUP 找不到值时返回错误

**问题公式**：
```excel
=VLOOKUP(A2, $B$2:$D$100, 3, FALSE)
-- 当A2值不存在时返回 #N/A
```

**修正方案**：
```excel
=IFERROR(VLOOKUP(A2, $B$2:$D$100, 3, FALSE), "未找到")
```

## 常用公式速查

### 统计类
```excel
// 单条件求和
=SUMIF(A:A, "华东", C:C)

// 多条件求和
=SUMIFS(C:C, A:A, "华东", B:B, ">=2024-01-01")

// 单条件计数
=COUNTIF(A:A, ">100")

// 多条件计数
=COUNTIFS(A:A, "北京", B:B, ">30")

// 忽略空值的平均值
=AVERAGEIF(A:A, "<>", B:B)
```

### 查找类
```excel
// VLOOKUP（从左到右查找）
=VLOOKUP(A2, $B$2:$D$100, 3, FALSE)

// INDEX+MATCH（灵活查找）
=INDEX($C$2:$C$100, MATCH(A2, $B$2:$B$100, 0))

// XLOOKUP（Office 365，替代VLOOKUP）
=XLOOKUP(A2, $B$2:$B$100, $C$2:$C$100, "未找到")

// 双向查找
=INDEX($B$2:$D$100, MATCH(A2, $A$2:$A$100, 0), MATCH(B2, $B$1:$D$1, 0))
```

### 日期类
```excel
// 计算年龄
=DATEDIF(B2, TODAY(), "Y")

// 月末日期
=EOMONTH(TODAY(), 0)

// 工作日天数
=NETWORKDAYS(A2, B2)

// 下个月第一天
=DATE(YEAR(TODAY()), MONTH(TODAY())+1, 1)
```

### 文本类
```excel
// 合并文本
=CONCATENATE(A2, "-", B2)

// 提取姓名
=LEFT(A2, FIND(" ", A2)-1)

// 提取手机号（只留数字）
=TEXTJOIN("", TRUE, IFERROR(MID(A2, SEQUENCE(LEN(A2)), 1)*1, ""))

// 条件显示
=IF(LEN(A2)>10, LEFT(A2, 10)&"...", A2)
```

### 逻辑类
```excel
// 多条件判断
=IFS(A2>=90, "A", A2>=80, "B", A2>=60, "C", TRUE, "D")

// 数组条件求和
=SUMPRODUCT((A2:A100="华东")*(B2:B100>=1000)*C2:C100)

// 判断是否重复
=IF(COUNTIF($A$2:A2, A2)>1, "重复", "唯一")
```

## FAQ（常见问题）

**Q1：VLOOKUP 和 INDEX+MATCH 哪个更好？**
A：VLOOKUP 简单易用，但只能从左向右查找；INDEX+MATCH 更灵活，可从任意方向查找，且性能更好。建议优先使用 INDEX+MATCH。

**Q2：为什么公式拉取后引用会错位？**
A：检查是否需要使用 $ 符号固定行或列。如需固定A列但行可变，使用 $A2；固定整个区域，使用 $A$2:$B$100。

**Q3：如何让公式在大数据量时更快？**
A：避免使用整列引用（如 A:A），改为具体范围；减少数组公式嵌套；关闭自动计算改为手动计算；使用 SUMPRODUCT 替代部分数组公式。

**Q4：如何处理带条件的复杂统计？**
A：使用 SUMIFS/COUNTIFS/AVERAGEIFS 处理多条件统计；使用 SUMPRODUCT 处理更复杂的数组条件。

**Q5：日期显示为数字怎么办？**
A：这是因为单元格格式是"常规"，右键设置单元格格式为"日期"即可正确显示。
