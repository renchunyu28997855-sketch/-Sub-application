# chart-designer

## 功能说明
帮助选择最合适的数据可视化方式，设计清晰、美观、有说服力的图表。

---

## 假设声明（Assumptions）

1. **数据已准备好**：数据已经过清洗和预处理，可直接用于可视化
2. **目标受众明确**：图表的读者角色和关注点已知
3. **展示场景确定**：图表用于PPT/报告/Dashboard/论文等场景
4. **数据量适中**：数据量不会导致渲染性能问题（若数据量过大需先聚合）
5. **色彩环境兼容**：最终展示环境支持所选色彩方案（考虑色盲群体）
6. **图表目的清晰**：明确图表要回答的问题或传达的信息

---

## 输入输出（I/O Format）

### 标准输入格式

```
【可视化目标】展示什么/回答什么问题
【数据类型】
  - 定量数据（数值）
  - 定类数据（类别）
  - 时间序列数据
  - 地理空间数据
【数据规模】数据点数量/维度数量
【目标受众】管理层/业务方/技术团队/公众
【展示场景】PPT/报告/Dashboard/论文/网页
【约束条件】配色要求/尺寸限制/品牌色
```

### 标准输出格式

```
【推荐图表类型】首选 + 备选方案
【图表设计说明】
  - X轴/Y轴定义
  - 色彩编码说明
  - 标签和图例
  - 交互功能（如适用）
【图表代码/配置】可执行的代码示例
【设计原则应用】为什么这个图表适合此场景
【常见误区提醒】
```

## 详细工作流程（Detailed Workflow）

### 第一阶段：需求理解
1. **明确可视化目的**
   - 比较数量？展示趋势？显示分布？揭示关系？
   - 强调单个重点还是展示整体概览？
   - 需要精确数值还是感知相对差异即可？

2. **分析数据结构**
   - 变量数量（单变量/双变量/多变量）
   - 数据类型（连续/离散/分类/时间）
   - 数据量级和密度
   - 是否包含地理信息

3. **了解受众和场景**
   - 受众的专业程度
   - 展示环境（投影/打印/屏幕）
   - 交互需求（静态/可筛选/可钻取）

**检查点**：确认对可视化需求有清晰理解

### 第二阶段：图表选择
4. **匹配图表类型**
   - 比较类：柱状图、条形图、饼图、雷达图
   - 趋势类：折线图、面积图
   - 分布类：直方图、箱线图、散点图
   - 关系类：散点图、气泡图、热力图
   - 地理类：地图、 Choropleth
   - 流程类：漏斗图、桑基图

5. **考虑数据编码优先级**
   - 位置 > 长度 > 角度 > 面积 > 颜色 > 体积
   - 选择最容易被视觉系统识别的编码方式

6. **评估备选方案**
   - 同一数据尝试2-3种图表类型
   - 选择最能清晰传达信息的方案

### 第三阶段：详细设计
7. **坐标轴设计**
   - 坐标轴从0开始（避免截断Y轴误导）
   - 选择合适的刻度间隔
   - 添加必要的参考线（平均线、目标线）

8. **色彩设计**
   - 配色不超过3种主色
   - 使用语义化颜色（红色=负面，绿色=正面）
   - 考虑色盲友好配色（避免红绿组合）
   - 使用渐变色时保持一致性

9. **标签和注释**
   - 直接在数据点上标注数值
   - 图例清晰易懂
   - 标题简洁明确（避免模糊标题如"数据趋势"）

### 第四阶段：实现与优化
10. **代码实现**
    - 使用推荐的图表库
    - 保持代码可复用
    - 添加适当的动画（可选）

11. **视觉优化**
    - 去除不必要的网格线
    - 减少视觉干扰
    - 保持足够的留白
    - 确保文字可读

12. **Accessibility检查**
    - 颜色对比度足够
    - 图形配合文字说明
    - 考虑打印效果

## 边界情况（Edge Cases）

### 数据为空的情况
- **完全无数据**：显示"暂无数据"占位符，不留空白图表
- **部分维度无数据**：使用"-"或"0"标注，避免误导

### 数据格式错误的情况
- **数值显示为文本**：转换数据类型后再绘图
- **日期格式不统一**：先统一日期格式
- **数据颗粒度不一致**：按统一维度聚合后再展示

### 特殊情况处理
- **数据量过大**（>10万点）：先采样或聚合，避免性能问题
- **数据分布极度不均**：使用对数刻度或分箱处理
- **负值展示**：使用双轴或调整基准线
- **多类别对比**（>10类）：考虑合并小类别或使用不同图表类型

---

## 错误处理（Error Handling）

| 错误类型 | 可能原因 | 处理方式 |
|---------|---------|---------|
| 饼图切片过多 | 类别>7个 | 限制5-7个，其他归为"其他" |
| 3D图表扭曲数据 | 滥用3D效果 | 改用2D图表 |
| Y轴截断 | 故意夸大差异 | Y轴从0开始 |
| 颜色误导 | 色觉编码不当 | 改用位置/长度编码 |
| 图表加载慢 | 数据量过大 | 采样或聚合 |
| 图例不清晰 | 颜色过多难区分 | 简化分类或使用交互图例 |

- [ ] 对色盲群体友好
- [ ] 导出格式和分辨率满足需求
- [ ] 数据与图表一致，无错误呈现

## 多个示例（Multiple Examples）

### 典型案例

**场景**：展示月度销售额及增长率

**输入**：
```
【可视化目标】展示12个月销售趋势，同时显示环比增长率
【数据类型】时间序列（月份）+ 两个数值指标（销售额、增长率）
【目标受众】管理层
【场景】季度业务汇报PPT
```

**推荐方案**：组合图
```
【设计说明】
- 主轴（柱状图）：月度销售额，清晰展示数量
- 次轴（折线图）：环比增长率，连接各点显示趋势
- 参考线：月度目标（灰色虚线）
- 颜色：柱状图用品牌蓝色，增长率用对比色（橙色）

【代码示例-Python】
import matplotlib.pyplot as plt

fig, ax1 = plt.subplots(figsize=(12, 6))
bars = ax1.bar(months, sales, color='#4A90E2', label='销售额')
ax1.axhline(y=target, color='gray', linestyle='--', label='目标')
ax1.set_ylabel('销售额（万元）', color='#4A90E2')
ax1.tick_params(axis='y', labelcolor='#4A90E2')

ax2 = ax1.twinx()
ax2.plot(months, growth, color='#E24A4A', marker='o', label='增长率')
ax2.set_ylabel('增长率（%）', color='#E24A4A')
ax2.tick_params(axis='y', labelcolor='#E24A4A')

plt.title('2024年月度销售趋势分析')
fig.legend(loc='upper right')
```

### 边界/异常案例

**场景**：收入分布严重右偏（少数高收入者拉高均值）

**输入**：
```
【数据】用户月均收入分布，90%用户<1万，1%用户>100万
【目标】展示收入分布特征
```

**问题发现**：
- 直方图会因极端值导致大部分区间几乎无数据
- 饼图无法清晰展示分布差异

**优化方案**：
```
【推荐图表】箱线图 + 截断直方图
【处理方式】
1. 使用箱线图展示整体分布特征（中位数、四分位数、异常值）
2. 直方图使用对数刻度，或将高收入段单独展示
3. 添加文字说明："90%用户月收入低于1万元，1%高收入用户单独标注"
```

### 失败案例及修正

**场景**：PPT中饼图比例失真

**原设计**：
```
问题：A公司市场份额饼图，60%和40%两个数据，用3D饼图展示
视觉感受：60%的扇区看起来只有30%
原因：3D饼图的视角扭曲了实际比例
```

**修正方案**：
```
【改用】2D环形图或柱状图
【理由】消除视觉扭曲，比例展示更准确
```

## FAQ（常见问题）

**Q1：什么情况下应该用柱状图而不是饼图？**
A：当需要精确比较各部分大小，或类别超过5个，或各部分比例接近难以用角度区分时，柱状图更合适。饼图适合展示整体与部分的关系，且类别不超过5个。

**Q2：多组数据如何展示？**
A：3组以内可用分组柱状图；3-5组考虑使用折线图（多系列）；5组以上使用热力图或雷达图；或考虑分面展示。

**Q3：如何选择颜色方案？**
A：优先使用品牌色或中性色；系列数据用渐变色；对比数据用互补色；确保打印后仍可区分；考虑色盲友好（避免纯红绿）。

**Q4：Dashboard中的图表和PPT中的图表有什么不同？**
A：Dashboard强调实时性和交互性，需要加载快、支持筛选；PPT强调视觉冲击力和叙事性，可以更复杂精美。

**Q5：数据标签太多看不清怎么办？**
A：只标注关键数据点；使用数据标签向导选择性显示；改用图例+参考线方式；或拆分多个小图表。
