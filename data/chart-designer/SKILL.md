# chart-designer v3.0

数据可视化图表设计技能。2026年集成AI自动图表推荐、交互式可视化、多模态数据展示能力。

## 功能说明

帮助选择最合适的数据可视化方式，设计清晰、美观、有说服力的图表。

---

## 假设声明（Assumptions）

1. **数据已准备好**：数据已经过清洗和预处理
2. **目标受众明确**：图表的读者角色和关注点已知
3. **展示场景确定**：图表用于PPT/报告/Dashboard/论文等
4. **图表目的清晰**：明确图表要回答的问题或传达的信息

---

## 输入输出（I/O Format）

### 标准输入格式
```
【可视化目标】展示什么/回答什么问题
【数据类型】定量/定类/时间序列/地理空间
【数据规模】数据点数量/维度数量
【目标受众】管理层/业务方/技术团队/公众
【展示场景】PPT/报告/Dashboard/论文/网页
```

### 标准输出格式
```
【推荐图表类型】首选 + 备选方案
【图表设计说明】X轴/Y轴定义、色彩编码、标签图例
【图表代码/配置】可执行的代码示例
【设计原则应用】为什么这个图表适合此场景
```

## 工作流程（Workflow）

### 第一阶段：需求理解
1. 明确可视化目的 - 比较数量/展示趋势/显示分布/揭示关系
2. 分析数据结构 - 变量数量、数据类型、数据量级
3. 了解受众和场景 - 专业程度、展示环境、交互需求

### 第二阶段：图表选择
4. 匹配图表类型 - 比较类/趋势类/分布类/关系类/地理类
5. 考虑数据编码优先级 - 位置>长度>角度>面积>颜色
6. 评估备选方案 - 尝试2-3种图表类型

### 第三阶段：详细设计
7. 坐标轴设计 - Y轴从0开始、合适刻度、参考线
8. 色彩设计 - 不超过3种主色、语义化颜色
9. 标签和注释 - 直接标注数值、图例清晰

## 质量验证清单（Quality Checklist）

- [ ] 图表类型选择正确，能回答核心问题
- [ ] 坐标轴设置合理（Y轴从0开始，刻度适当）
- [ ] 色彩使用恰当（不超过3种主色）
- [ ] 标签直接标注在数据上
- [ ] 无不必要的视觉元素干扰
