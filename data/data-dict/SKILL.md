# data-dict v3.0

## 核心能力

1. **字典生成** - 表结构字段说明
2. **血缘追溯** - 表级/字段级血缘
3. **元数据管理** - 统计信息、口径
4. **质量检查** - 完整性、一致性
5. **AI字段推荐** - 2026年智能推荐
6. **自动血缘分析** - 自动绘制血缘图

## 工作流程（渐进式披露）

### 阶段一：信息收集

```
1. 确定范围 → 识别需要生成字典的表
2. 获取元数据 → 表结构/字段/类型/注释
3. 收集业务定义 → 字段业务含义
```

### 阶段二：字典编制

```
4. 基础结构设计 → 确定收录内容
5. 逐表编写 → 表基本信息、字段说明
6. 血缘标注 → 表级/字段级血缘
```

### 阶段三：审核发布

```
7. 内容审核 → 业务方确认
8. 格式优化 → 排版/搜索
9. 发布分发 → 知识库/通知
```

### 阶段四：维护更新

```
10. 变更追踪 → 新增/修改/删除
11. 定期审查 → 季度审查
12. AI自动更新 → 2026年自动同步
```

## 常见问题

**Q: 数据字典应该谁来维护？** 数据团队/DBA统一维护，业务方提供定义。

**Q: 如何处理敏感字段？** 标注敏感等级(L1-L4)，不同等级有不同访问要求。

**Q: 微服务架构下血缘如何维护？** 使用DataHub/Amundsen等工具自动采集。
