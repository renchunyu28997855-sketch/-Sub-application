# data-viz-story

## 功能说明
帮助将数据分析结果转化为有说服力的故事，让数据汇报更具影响力。

---

## 假设声明（Assumptions）

1. **数据已分析完毕**：有明确的数据发现和洞察
2. **受众明确**：知道听众是谁、关注什么
3. **时间有限**：汇报时间有限（通常5-30分钟）
4. **目标清晰**：汇报有明确目的（说服/告知/决策）
5. **场景合适**：适合用故事化的方式呈现
6. **支持可视化**：有条件制作图表或PPT

---

## 输入输出（I/O Format）

### 标准输入格式

```
【汇报主题】核心主题
【数据分析结果】关键发现列表
【受众】谁在听
【时间限制】汇报时长
【目标】说服/告知/激发行动
【场景】会议/演讲/报告/文档
```

### 标准输出格式

```
【故事框架】
- 开场：吸引注意力
- 主体：层层递进的发现
- 高潮：核心洞察
- 结尾：行动号召

【PPT结构】
- 每页主题
- 关键信息
- 图表建议
- 讲述要点
```

## 详细工作流程（Detailed Workflow）

### 第一阶段：信息梳理
1. **明确汇报目标**
   - 核心信息是什么
   - 希望听众做什么
   - 衡量成功的标准

2. **梳理数据发现**
   - 列出所有数据发现
   - 按重要性排序
   - 识别关键洞察

3. **了解受众**
   - 受众关心什么
   - 他们的痛点
   - 决策权限

4. **确定叙事主线**
   - 选择核心故事线
   - 决定展示顺序
   - 准备转折点

**检查点**：明确核心信息和叙事逻辑

### 第二阶段：故事构建
5. **设计开场**
   - 制造"哇时刻"
   - 设立悬念
   - 引发共鸣

6. **构建主体**
   - 问题揭示
   - 分析过程
   - 发现呈现

7. **设计高潮**
   - 核心洞察
   - 价值量化
   - 意义升华

8. **收尾设计**
   - 总结要点
   - 明确行动
   - 留下印象

### 第三阶段：视觉呈现
9. **图表选择**
   - 每页一个核心图表
   - 图表占页面60%以上
   - 简化信息，突出重点

10. **文字精简**
    - 每页不超过3个要点
    - 使用短句
    - 避免术语

11. **视觉优化**
    - 色彩统一
    - 留白适当
    - 引导视线

### 第四阶段：演练优化
12. **模拟演练**
    - 控制时间
    - 检查逻辑
    - 准备问答

13. **反馈修改**
    - 调整节奏
    - 优化表达
    - 补充支撑

## 数据故事框架

### 框架一：问题-方案-价值
```
1. 背景：描述当前状况
2. 问题：揭示痛点或机会
3. 分析：如何深入了解
4. 发现：关键洞察
5. 方案：建议采取的行动
6. 价值：预期收益
```

### 框架二：英雄之旅
```
1. 平凡世界：介绍现状
2. 冒险召唤：问题出现
3. 遇见导师：分析方法
4. 跨越门槛：开始探索
5. 严峻考验：困难和挑战
6. 满载而归：发现和洞察
7. 凯旋归来：建议和行动
```

### 框架三：起承转合
```
1. 起：吸引注意，引入主题
2. 承：铺陈背景，描述现状
3. 转：转折发现，揭示洞察
4. 合：总结要点，呼吁行动
```

## 边界情况（Edge Cases）

### 内容问题
- **数据单薄**：选择更小的切入点，或增加数据来源
- **发现平淡**：从不同角度挖掘，寻找异常或对比
- **结论冲突**：坦诚呈现，由受众决定

### 呈现问题
- **时间不够**：精简内容，突出核心
- **受众不专注**：增加互动，制造悬念
- **被质疑数据**：展示数据来源和方法论

### 场景问题
- **无PPT条件**：用口述+白板/纸笔
- **远程汇报**：增加视觉元素，语速放慢
- **问答环节长**：提前准备FAQ，控制节奏

## 错误处理（Error Handling）

| 问题类型 | 可能原因 | 处理方式 |
|---------|---------|---------|
| 数据被质疑 | 数据来源不明确 | 准备数据溯源文档 |
| 逻辑被挑战 | 论证不严密 | 展示多维度验证 |
| 受众不感兴趣 | 内容不切要害 | 调整开场和重点 |
| 时间失控 | 内容过多 | 提前演练，果断删减 |

## 多个示例（Multiple Examples）

### 典型案例

**主题**：用户留存分析汇报

**原版（纯数据）**：
```
- 次日留存率42%
- 7日留存率25%
- 30日留存率12%
- 流失用户10万/月
```

**故事版（有力说服）**：

```
【开场 - 制造震撼】
"我们分析了100万用户的行为，发现了一个价值千万的问题。"

【问题揭示】
"每月有10万用户在注册后30天内流失。
这意味着，如果不做任何改变，我们每年将损失120万潜在用户。"

【深入分析】
"我们发现，流失的用户中有78%从未完成首次购买。
而完成首次购买的用户，30日留存率高达65%。
关键差距就在'首次购买'这个环节。"

【洞察提炼】
"更深入的分析显示，影响首次购买的核心因素是：
1. 新用户引导流程（影响30%）
2. 首单优惠力度（影响25%）
3. 商品推荐精准度（影响20%）"

【解决方案】
"我们建议：优化新用户首次购买体验
目标：将首次购买率从22%提升至40%"

【预期价值】
"如果目标达成：
- 预计每年可挽回60万用户
- 创造约6000万额外收入
- 30日留存率提升至20%"
```

### 边界/异常案例

**场景**：数据发现与业务预期完全相反

**原发现**：产品改版后各项指标都在下降

**故事构建**：
```
【开场 - 坦诚揭示】
"这次改版的数据结果，可能和大家预期不太一样。"

【客观呈现】
"改版后7日活跃率下降15%
用户时长下降20%
但同时，负面反馈下降了60%"

【深入分析】
"这说明改版确实牺牲了部分'凑热闹'的用户
但留下来了真正的目标用户
这是'净化'而非'流失'"

【重新定义成功】
"如果用'留下真正用户'这个角度衡量
改版是成功的"

【建议】
"建议调整考核指标
增加'用户质量'维度的评估"
```

---

### 失败案例及修正

**场景**：汇报后没有产生任何行动

**问题分析**：
1. 只陈述问题，没有提出解决方案
2. 建议太抽象，无法执行
3. 没有量化价值，无法打动决策者

**修正方案**：
```
【修正前】
"建议优化新用户流程"

【修正后】
"建议实施'首单提升计划'，包含3项具体措施：
1. 改版新用户引导流程（预计投入2周开发）
2. 发放首单专属优惠券（预算10万元/月）
3. 增加个性化商品推荐（预计提升转化15%）

预期效果：
- 首次购买率：22% -> 35%
- 30日留存率：12% -> 18%
- ROI: 1:4.5

建议本周启动，优先排期"
```

## PPT结构建议

### 单页结构模板

```
┌────────────────────────────────────────┐
│  核心观点（标题，1句话）                │
├────────────────────────────────────────┤
│                                        │
│           【主图表】                    │
│         占页面60%面积                   │
│                                        │
├────────────────────────────────────────┤
│  3个支撑要点（简短）                   │
│  · 要点1                               │
│  · 要点2                               │
│  · 要点3                               │
└────────────────────────────────────────┘
```

### 开场5页结构
1. 封面：主题 + 演讲者
2. Agenda：今天要讲什么
3. 背景：为什么这个问题重要
4. 挑战/机会：当前面临的问题
5. 核心洞察：你的主要发现

### 结尾5页结构
1. 总结：今天的关键发现
2. 建议：具体行动方案
3. 价值：预期收益/价值
4. 计划：下一步是什么
5. Q&A：开放讨论

## FAQ（常见问题）

**Q1：数据故事和数据分析报告有什么区别？**
A：报告是给专业人士看的详细分析，强调数据和方法；故事是给决策者看的精简呈现，强调洞察和行动。一个是"证明"，一个是"说服"。

**Q2：如何让数据更有说服力？**
A：量化价值（不用"很多"用"1200万"）；对比参照（与行业/竞品/历史对比）；展示因果（不仅说是什么，还要说为什么）。

**Q3：被问到数据来源怎么办？**
A：准备数据溯源文档；能回答"数据是XX系统XX时间XX维度的原始数据"；对数据质量要有信心。

**Q4：时间不够讲不完怎么办？**
A：提前演练，果断删减；留出互动时间；准备"详细版"作为附录。

**Q5：如何应对质疑？**
A：保持冷静，用数据回应；承认局限性比强辩更有力；准备第二层数据应对追问。
