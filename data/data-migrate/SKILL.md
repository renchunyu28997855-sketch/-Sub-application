## 工作流程（渐进式披露）

### 阶段一：迁移准备

```
1. 需求分析 → 迁移目的、范围、数据量
2. 环境检查 → 源端/目标端/网络/权限
3. 方案设计 → 迁移方式、ETL、校验
```

### 阶段二：全量迁移

```
4. 数据抽取 → 全量导出、格式转换
5. 数据导入 → 批量导入、索引重建
6. 初步校验 → 数据量对比、抽样核对
```

### 阶段三：增量同步

```
7. CDC配置 → 变更数据捕获配置
8. 实时同步 → 启动增量任务
9. 切流准备 → 灰度切流验证
```

### 阶段四：正式切换

```
10. 业务切换 → 停止写入、追平增量
11. 最终校验 → 全量校验、功能验证
12. 监控告警 → 配置监控
```

### 阶段五：收尾

```
13. 原系统处理 → 归档/保留/下线
14. 文档归档 → 记录/总结/经验
```

## 常见问题

**Q: 全量迁移期间业务如何处理？** 通常采用"双写"或"停服维护"策略。

**Q: 如何保证迁移后数据一致性？** 全量校验+增量校验双重保障。

**Q: 迁移失败了怎么办？** 立即启动回滚方案，恢复原系统。
