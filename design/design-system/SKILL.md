# design-system v3.0

设计系统规范建议技能。2026年集成AI组件推荐、设计令牌自动生成、多平台适配检测能力。

## 功能说明

帮助建立完整的设计系统，从设计令牌到原子设计，从组件规范到模式库。

---

## 假设声明（Assumptions）

- 品牌定位和视觉系统已建立
- 产品类型和平台已明确
- 设计工具已选定

---

## 输入输出（I/O Format）

### 标准输入格式

```
【设计系统请求】
## 产品信息
- 产品类型：[App/Web/多平台]
- 产品数量：[单一/多产品线]
- 目标平台：[iOS/Android/Web]
## 团队情况
- 设计团队规模：[人数]
```

### 标准输出格式

```
【设计系统规范手册】
## 1. 设计原则
[核心理念和设计原则]
## 2. 设计令牌
| 令牌名 | 色值 | 用途 |
## 3. 原子设计
[原子/分子/有机体列表]
## 4. 组件规范
[核心组件规范]
```

## 2026年新功能

### AI辅助设计系统建设
| 功能 | 说明 |
|------|------|
| **智能组件推荐** | 基于场景推荐组件组合 |
| **令牌自动生成** | 从品牌色自动生成令牌 |
| **多平台检测** | 自动检测跨平台一致性 |
| **使用率分析** | 分析组件使用情况 |

### 系统增强
- 增加暗色模式自动适配
- 增加设计-代码同步检测
- 增加系统健康度诊断

3. 有机体组件

### 阶段4：文档建设
1. 设计原则文档
2. 令牌文档
3. 组件文档
