# slide-deck

## 功能说明

帮助规划和制作演示文稿，提供结构设计、内容组织和演讲建议。从受众分析到信息架构，从视觉设计到演讲技巧，提供完整的演示文稿解决方案。

---

## 假设声明（Assumptions）

### 前提条件
- 演示目标已明确
- 目标受众已了解
- 演示时长已确定
- 主题内容已准备

### 适用范围
- 商业路演（融资、产品发布）
- 工作汇报（项目汇报、季度总结）
- 培训演讲（知识分享、技能培训）
- 学术报告（论文答辩、研究分享）
- 营销演示（方案提案、销售演示）

### 不适用场景
- 纯娱乐性质的演示
- 需要实时协作编辑的场景
- 完整的设计稿制作（需要设计工具）

## 详细工作流程（Detailed Workflow）

### 阶段1：受众与目标分析

**步骤1：受众分析**
```
检查点：
□ 受众是谁已明确
□ 受众关注点已了解
□ 受众决策权已知
□ 演讲场景已确认
```

1. 确定受众身份和背景
2. 分析受众的核心关注点和痛点
3. 了解受众的决策影响力
4. 确认演讲场景（线下/线上/录播）

**步骤2：目标设定**
1. 确定演示的最终目标
2. 设定可衡量的子目标
3. 确定需要传递的核心信息
4. 明确演示后的期望行动

### 阶段2：内容结构设计

**步骤1：选择演示框架**

| 框架类型 | 适用场景 | 结构特点 |
|----------|----------|----------|
| 痛点-方案 | 产品推销 | 问题→解决方案→价值 |
| 故事线 | 情感说服 | 开端→冲突→高潮→结局 |
| 数字论证 | 数据汇报 | 结论→数据支撑 |
| 时间线 | 历程回顾 | 过去→现在→未来 |
| 要素组合 | 全面介绍 | 分类讲解 |

**步骤2：页面规划**
1. 确定总页数（根据时长）
2. 规划每页的核心信息
3. 设计页面间的逻辑关系
4. 确保信息密度适中

### 阶段3：每页内容设计

**步骤1：封面设计**
1. 标题清晰传达主题
2. 副标题补充说明
3. 演讲者信息（如果适用）
4. 视觉元素吸引注意

**步骤2：内容页设计**
1. 每页一个核心观点
2. 使用图表代替文字
3. 预留演讲发挥空间
4. 设计视觉层次

**步骤3：结尾设计**
1. 总结核心要点
2. 明确的行动号召
3. 感谢和Q&A提示

### 阶段4：演讲准备

**步骤1：演讲稿撰写**
1. 每页的要点提示
2. 过渡语准备
3. 数据引用准备
4. 应对问题准备

**步骤2：时间把控**
1. 计算每页平均时长
2. 标记需要延长时间的页面
3. 预留Q&A时间
4. 准备压缩方案

## 边界情况（Edge Cases）

### 情况1：超短时间演示（3-5分钟）
**场景：** 时间非常有限，需高效传达
**处理：**
```
【超短演示规划】

时间约束：3-5分钟，约5-8页

策略：
1. 只讲一件事
   - 聚焦1个核心信息
   - 其他内容忍痛割舍

2. 结构简化
   - 背景（1页）：30秒
   - 核心内容（3-4页）：每页45秒
   - 结论+行动（1页）：30秒

3. 技巧
   - 用一句话概括你的观点
   - 用一个故事/案例支撑
   - 结尾明确告诉受众要做什么

核心原则：少即是多
```

### 情况2：超长演示（30分钟以上）
**场景：** 时间充裕，需丰富内容
**处理：**
```
【超长演示规划】

时间约束：30分钟以上，20+页

策略：
1. 分段设计
   - 分为3-4个章节
   - 每个章节有独立主题
   - 章节间有过渡页

2. 内容深度
   - 数据分析有详情
   - 案例有完整故事
   - 给受众消化时间

3. 互动设计
   - 每章节后留Q&A
   - 适当提问互动
   - 保持注意力

4. 节奏把控
   - 每15分钟换一种形式
   - 避免连续长时间讲述
```

### 情况3：纯线上演示
**场景：** 通过视频会议远程演示
**处理：**
```
【线上演示策略】

线上特点：
- 注意力更分散
- 缺乏眼神交流
- 视觉疲劳更快

策略：
1. 内容调整
   - 页面更简洁
   - 文字更少
   - 图表更清晰

2. 互动设计
   - 增加提问互动
   - 使用投票/问答工具
   - 定期小结确认理解

3. 技巧
   - 语速稍慢
   - 重要内容重复强调
   - 眼神看镜头

4. 技术准备
   - 提前测试设备和网络
   - 准备备选方案
```

### 情况4：学术/技术演示
**场景：** 专业性强，需要展示技术细节
**处理：**
```
【学术演示策略】

学术演示特点：
- 专业观众多
- 细节要求高
- 逻辑严密性重要

策略：
1. 结构严谨
   - 问题定义清晰
   - 方法论明确
   - 证据充分

2. 深度把控
   - 核心内容深入
   - 周边内容可略
   - 给参考文献

3. 应对专业问题
   - 准备详细附录
   - 诚实回答不知道
   - 表示后续跟进
```

## 错误处理（Error Handling）

### 错误类型1：内容过载
**识别：** 单页信息过多，文字密密麻麻
**处理：**
```
【内容精简建议】

当前页面信息过多，建议：

1. 拆分内容
   - 一页变两页
   - 每页一个核心点

2. 删除次要信息
   - 保留最重要的3点
   - 其他放入附录

3. 图表化
   - 文字转为图表
   - 一图胜千言

4. 演讲补充
   - 口头讲述详细内容
   - 页面只放关键点
```

### 错误类型2：逻辑跳跃
**识别：** 页面间逻辑关系不清晰
**处理：**
```
【逻辑梳理建议】

页面逻辑问题：

建议使用过渡页：
- "为什么这个问题重要？" → 过渡到痛点
- "我们如何解决？" → 过渡到方案
- "这对你意味着什么？" → 过渡到价值

检查：
- 每页承上启下
- 有逻辑连接词
- 避免突然跳转
```

### 错误类型3：数据引用不规范
**识别：** 数据来源不明或格式混乱
**处理：**
```
【数据规范建议】

数据引用要求：

1. 来源明确
   - 数据来源：XXX机构
   - 调研时间：2024年X月
   - 样本量：N=XXX

2. 格式统一
   - 数字格式：千分位分隔
   - 百分比：保留1位小数
   - 货币：标明货币单位

3. 可验证性
   - 提供数据来源链接
   - 标注可以深入了解的方式
```

## FAQ（常见问题）

### Q1：PPT应该有多少页？
**A：** 根据时长调整：1分钟约1页，10分钟约8-10页，20分钟约15-18页。原则是"宁少勿多"，给每页足够演讲时间。

### Q2：封面应该包含什么？
**A：** 封面基本要素：1）标题（清晰传达主题）；2）副标题（可选，补充说明）；3）演讲者信息（姓名、职位）；4）日期/场合；5）公司/项目Logo（如果适用）。

### Q3：如何让数据图表更有说服力？
**A：** 数据展示技巧：1）标注关键数据点；2）使用对比（vs竞品、vs过去）；3）标注数据来源；4）一句话解读数据含义；5）避免过于复杂的图表。

### Q4：远程演示有什么技巧？
**A：** 线上演示技巧：1）PPT页面更简洁（远程注意力更分散）；2）语速稍慢（网络延迟）；3）重要内容重复；4）增加互动（投票、问答）；5）眼睛看摄像头。

### Q5：如何应对Q&A环节？
**A：** Q&A准备：1）预测可能问题并准备答案；2）不懂就说，不要硬撑；3）复杂问题可约定会后跟进；4）准备好总结页可以再讲一遍核心信息。

### Q6：是否需要动画效果？
**A：** 动画使用建议：1）适度使用，不要花哨；2）用动画引导注意力；3）避免过多文字动画；4）线上演示减少复杂动画（兼容性差）；5）关键信息不要依赖动画。
