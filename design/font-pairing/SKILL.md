# font-pairing

## 功能说明

帮助选择和搭配字体，建立统一的视觉语言。涵盖字体选择原则、中英文搭配、字体层级设计、字号规范等内容，确保字体的可读性和美观性。

---

## 假设声明（Assumptions）

### 前提条件
- 产品类型已明确（App/网站/印刷品）
- 目标平台已确定
- 品牌调性已定义
- 目标用户已了解

### 适用范围
- 移动App字体系统
- Web页面字体系统
- 品牌字体系统
- 印刷品字体系统
- 演示文稿字体

### 不适用场景
- 需要专业排版的中文长文档
- 涉及字体版权的复杂情况
- 专业字体定制设计

## 详细工作流程（Detailed Workflow）

### 阶段1：需求分析

**步骤1：场景分析**
```
检查点：
□ 产品类型已确认
□ 目标平台已确定
□ 品牌调性已了解
□ 用户特征已清楚
```

1. 分析产品类型和用途
2. 了解目标平台特性
3. 确定品牌调性
4. 考虑目标用户的阅读习惯

**步骤2：字体需求梳理**
1. 确定需要哪些用途的字体
2. 分析不同场景的字体需求
3. 识别特殊字体需求（如数字、等宽）
4. 考虑无障碍需求

### 阶段2：字体选择

**步骤1：选择基础字体**

| 字体分类 | 推荐字体 | 适用场景 | 特点 |
|----------|----------|----------|------|
| 现代简洁 | 思源黑体、苹方、Noto Sans | 数字产品 | 清晰、现代 |
| 传统稳重 | 宋体、思源宋体 | 正式场合 | 文化感、权威 |
| 活泼友好 | 圆体、站酷快乐体 | 儿童/娱乐 | 亲切、有趣 |
| 专业技术 | JetBrains Mono、Source Code Pro | 代码/数据 | 等宽、易辨认 |

**步骤2：字体搭配原则**
1. 选择风格相近的字体
2. 区分标题字体和正文字体
3. 确保中英文风格协调
4. 考虑字体的可扩展性

**步骤3：层级设计**
1. 确定字体层级数量（通常4-6级）
2. 定义每层的字号、字重
3. 确保层级间对比足够
4. 保持使用一致性

### 阶段3：规范制定

**步骤1：制定使用规范**
1. 明确每种字体的使用场景
2. 定义字号范围
3. 规定字间距和行高
4. 制定中英文混排规则

**步骤2：制定技术规范**
1. 字体文件格式（TTF/OTF/WOFF2）
2. 字体加载策略
3. 回退字体设置
4. 无障碍字体设置

## 边界情况（Edge Cases）

### 情况1：只有中文字体需求
**场景：** 纯中文产品或主要面向中文用户
**处理：**
```
【中文字体方案】

纯中文场景字体建议：

1. 现代风格
   - 首选：思源黑体（Source Han Sans）
   - 备选：苹方（PingFang）、Noto Sans CJK
   - 特点：简洁、清晰、屏幕优化

2. 传统风格
   - 首选：思源宋体（Source Han Serif）
   - 备选：宋体（系统自带）
   - 特点：文化感、适合长文阅读

3. 字体层级建议
   | 层级 | 字号 | 字重 |
   |------|------|------|
   | 标题 | 24-32px | Bold |
   | 副标题 | 18-22px | Medium |
   | 正文 | 15-17px | Regular |
   | 辅助 | 12-14px | Regular |
```

### 情况2：多语言支持
**场景：** 需要支持中文+英文+日文等多语言
**处理：**
```
【多语言字体方案】

多语言支持策略：

1. 字体选择原则
   - 选择支持多语言的字体家族
   - 确保各语言字体风格协调
   - 考虑各语言的阅读方向

2. 推荐方案

   中文：Noto Sans SC / 思源黑体
   英文：Inter / Roboto
   日文：Noto Sans JP
   韩文：Noto Sans KR

3. 风格统一技巧
   - 使用同一字体的不同字重
   - 或选择风格相近的字体
   - 确保各语言视觉重量相当

4. 技术处理
   - 单独加载各语言字体
   - 使用font-display: swap
   - 设置合适的fallback
```

### 情况3：数字展示为主
**场景：** 金融、数据类产品，数字展示重要
**处理：**
```
【数字字体方案】

数字展示优先级场景：

1. 数字字体选择
   - Tabular Lining（等宽数字）：金额、数据对齐
   - Oldstyle Figures：文本中的数字，有上下起伏
   - DIN Alternate：统计数据，清晰有力
   - SF Pro Rounded：界面数字，友好感

2. 数字使用场景
   | 场景 | 推荐字体 | 特点 |
   |------|----------|------|
   | 金额 | Tabular Lining | 等宽对齐 |
   | 统计 | DIN Alternate | 清晰有力 |
   | 界面 | SF Pro Rounded | 友好现代 |
   | 图表 | 同正文 | 统一风格 |

3. 数字层级
   - 大数字（数据看板）：48-72px，Bold
   - 中数字（卡片金额）：24-32px，Semibold
   - 小数字（表格）：14-16px，Regular

4. 注意
   - 金额数字必须使用等宽字体
   - 千分位分隔符要清晰
   - 小数点后位数统一
```

### 情况4：无障碍/老年用户
**场景：** 需要考虑视力不好的用户
**处理：**
```
【无障碍字体方案】

无障碍字体设计：

1. 字号放大
   - 正文字号：≥18px
   - 标题字号：≥24px
   - 最小字号：≥14px

2. 字体选择
   - 首选：清晰、无装饰的字体
   - 推荐：思源黑体、苹方
   - 避免：过于纤细的字体

3. 行高增加
   - 行高建议：字号的1.6-1.8倍
   - 段落间距：字号×1.2

4. 对比度要求
   - 文字与背景对比度≥4.5:1
   - 重要信息对比度≥7:1
   - 使用深色文字（#1A1A2E）而非纯黑

5. 字间距
   - 可适当增加字间距
   - 建议：0.02-0.05em
```

## 错误处理（Error Handling）

### 错误类型1：字体数量过多
**识别：** 使用了3种以上不同字体
**处理：**
```
【字体数量控制建议】

当前问题：使用了X种字体，建议精简

原则：同一产品最多使用2-3种字体

精简方案：
1. 选择一套字体家族
   - 思源黑体（支持多字重）
   - Inter（英文）
   - 通过字重区分层级

2. 区分使用
   - 标题：Bold/Semibold
   - 正文：Regular
   - 辅助：Light

3. 字重层级代替字体层级
   - 用2-3种字重表达层级
   - 避免引入新字体

推荐结构：
- 中文：思源黑体（Bold + Regular + Light）
- 英文：Inter（Bold + Regular）
```

### 错误类型2：字重使用混乱
**识别：** 同一层级的文字使用不同字重
**处理：**
```
【字重使用规范】

字重使用混乱问题：
- 同一层级的标题有时Bold有时Medium
- 导致视觉层级不一致

规范建议：
| 层级 | 字重 | 说明 |
|------|------|------|
| 大标题 H1 | Bold (700) | 最重 |
| 标题 H2 | Bold/600 | 次重 |
| 副标题 H3 | Medium (500) | 中等 |
| 正文 Body | Regular (400) | 正常 |
| 辅助文字 | Light/300 | 较轻 |

规则：
- 同一层级必须使用相同字重
- 新增层级必须定义字重
- 建立字重使用对照表
```

### 错误类型3：字号层级不明显
**识别：** 标题和正文差不多大，看不出层级
**处理：**
```
【字号层级诊断】

层级不清问题：
- H1标题24px，正文16px（对比1.5:1）
- 层级对比不足，难以区分

字号对比标准：
- 最小对比：H1 vs Body ≥ 1.5:1
- 推荐对比：H1 vs Body ≥ 2:1
- H1 vs Caption ≥ 2.5:1

调整建议：
| 层级 | 当前字号 | 建议字号 |
|------|----------|----------|
| H1 | 20px | 28-32px |
| H2 | 17px | 22-24px |
| Body | 15px | 15-16px |
| Caption | 13px | 12-13px |

注意：避免字号过小，正文不小于14px
```

## FAQ（常见问题）

### Q1：如何选择标题字体和正文字体？
**A：** 标题字体特点：字重较重（Bold）、笔画清晰、有视觉冲击力。正文字体特点：易读性优先、长时间阅读不累、细节处理到位。搭配原则：风格统一但有对比（如同家族的黑体和宋体）。

### Q2：思源黑体和苹方哪个更好？
**A：** 两者都是优秀的屏幕字体：思源黑体开源免费、字重选择多；苹方是苹果系统字体、在iOS设备上显示最佳。建议：iOS为主用苹方，跨平台用思源黑体，或两者配合使用。

### Q3：可以同时使用衬线和非衬线字体吗？
**A：** 可以但要谨慎。常见做法：1）衬线标题+无衬线正文（传统搭配）；2）不同衬线程度区分层级；3）避免在同一界面混用过多字体风格。如无把握，保持风格统一更安全。

### Q4：字体文件很大如何处理？
**A：** 优化方法：1）使用WOFF2格式（体积小）；2）使用font-display: swap实现渐进显示；3）只加载需要的字重；4）使用系统字体作为fallback；5）考虑使用font-spider等工具压缩字体的使用字符。

### Q5：如何在设计稿中标注字体？
**A：** 字体标注建议：1）标注字体名称和字重；2）标注具体字号和行高；3）标注文字颜色；4）提供fallback建议；5）附上字体文件或获取方式。格式可参考：思源黑体 / Bold / 24px / #1A1A2E / 行高1.3。

### Q6：英文字体如何选？
**A：** 英文字体选择：1）屏幕优化：Inter、Roboto、San Francisco；2）正式文档：Source Serif Pro、Playfair Display；3）代码：JetBrains Mono、Fira Code；4）无衬线更现代，衬线更有格调。
