# brand-design

## 功能说明

帮助建立完整的品牌视觉系统，提供设计规范和指导。包括Logo系统、色彩系统、字体系统、图像风格、图标系统、组件规范等，确保品牌视觉的一致性和可复用性。

---

## 假设声明（Assumptions）

### 前提条件
- 品牌定位和核心价值已明确
- Logo设计已完成或有基础版本
- 目标用户群体已定义
- 主要应用场景已了解

### 适用范围
- 初创品牌视觉系统建立
- 品牌视觉升级
- 产品品牌系统
- 子品牌系统
- 设计系统中的品牌规范

### 不适用场景
- 纯技术产品标识设计
- 需要法律合规审查的特定行业（如金融、医疗）
- 已有强制执行的品牌指南

## 详细工作流程（Detailed Workflow）

### 阶段1：品牌理解

**步骤1：品牌资产梳理**
```
检查点：
□ 品牌定位已明确
□ 已有品牌资产已收集
□ 目标用户已定义
□ 应用场景已了解
```

1. 收集现有品牌资产（Logo、配色、字体等）
2. 梳理品牌定位和核心价值
3. 了解目标用户特征
4. 确认主要应用场景

**步骤2：缺口分析**
1. 识别已有品牌资产
2. 识别缺失的规范
3. 确定系统复杂度需求
4. 制定输出计划

### 阶段2：规范制定

**步骤1：Logo规范**
1. 整理Logo版本（完整、简化、单色）
2. 制定安全空间规范
3. 定义禁止事项
4. 提供正确使用示例

**步骤2：色彩系统**
1. 确定主色（来自Logo或品牌）
2. 构建辅色系统
3. 定义功能色
4. 建立中性色体系

**步骤3：字体系统**
1. 选择主字体（中英文）
2. 定义字体层级
3. 制定字体使用规范
4. 考虑备选字体

**步骤4：图像风格**
1. 确定图片风格
2. 制定图片使用规范
3. 定义插画风格（如适用）
4. 建立图片来源标准

**步骤5：图标规范**
1. 确定图标风格
2. 制定尺寸规范
3. 定义颜色规范
4. 选择图标库

**步骤6：组件规范**
1. 定义核心组件样式
2. 制定组件变体
3. 建立状态规范
4. 提供使用指南

### 阶段3：输出整理

**步骤1：文档整理**
1. 按规范模块整理
2. 添加图示和示例
3. 确保逻辑清晰
4. 便于后续查阅

**步骤2：使用指南**
1. 提供常见场景应用
2. 标注注意事项
3. 给出正确/错误示例
4. 便于设计师和开发者使用

## 边界情况（Edge Cases）

### 情况1：已有零散的视觉资产但不成系统
**场景：** 有Logo、有颜色，但缺乏统一规范
**处理：**
```
【视觉资产整合建议】

已有资产：
- Logo：✓ 完成
- 配色：✓ 部分完成
- 字体：✗ 未定义
- 图像风格：✗ 未定义

建议整合步骤：

1. 资产审计
   - 梳理现有所有视觉资产
   - 评估质量和一致性

2. 缺失补充
   - 定义缺失的字体规范
   - 确定图像风格方向
   - 建立图标使用规范

3. 整合输出
   - 将零散资产整合为系统
   - 编写使用规范文档
   - 提供应用示例
```

### 情况2：多品牌/多产品线视觉系统
**场景：** 需要建立主品牌+多个子品牌的视觉系统
**处理：**
```
【多品牌视觉系统建议】

品牌架构：
- 主品牌：[名称]
- 子品牌A：[名称]
- 子品牌B：[名称]

视觉系统策略选择：

1. 统一架构
   - 所有品牌使用同一套规范
   - 通过颜色/Logo区分
   - 适合关联度高的品牌

2. 独立架构
   - 每个品牌独立系统
   - 仅共享部分设计原则
   - 适合差异大的品牌

3. 混合架构
   - 共享基础规范（字体、图标）
   - 品牌特有元素（Logo、配色）独立
   - 平衡一致性与灵活性

建议：根据品牌架构选择适合的策略
```

### 情况3：数字化产品品牌
**场景：** 主要应用场景是App/网站等数字产品
**处理：**
```
【数字产品品牌规范要点】

重点考虑：

1. 暗色模式
   - 定义暗色模式配色
   - 确保Logo在深色背景下清晰
   - 考虑对比度要求

2. 响应式适配
   - Logo在不同尺寸下的表现
   - 字体层级的响应式调整
   - 组件的响应式行为

3. 动效规范
   - 品牌元素的动效风格
   - 转场动画风格
   - 交互反馈风格

4. 无障碍
   - 色彩对比度要求
   - 字体可读性要求
   - 交互可访问性

请提供数字产品的使用场景详情。
```

### 情况4：跨文化品牌
**场景：** 品牌需要面向不同国家和文化
**处理：**
```
【跨文化品牌规范要点】

目标市场：[市场列表]

需考虑：

1. 字体适配
   - 中文变体（简体/繁体）
   - 日文/韩文字体支持
   - 阿拉伯文等RTL语言

2. 色彩文化差异
   - 颜色在不同文化中的含义
   - 避免文化禁忌色
   - 当地偏好色

3. 图像内容
   - 人物面孔的代表性
   - 文化敏感元素
   - 当地化图片需求

4. Logo适用性
   - 在不同语言的适用性
   - 文化背景下的解读
   - 跨文化测试建议
```

## 错误处理（Error Handling）

### 错误类型1：规范过于复杂难以执行
**识别：** 规范内容过多，执行成本高
**处理：**
```
【规范简化建议】

当前规范复杂度较高，建议分层：

第一层：核心规范（必须遵守）
- Logo使用
- 主色使用
- 字体层级

第二层：扩展规范（建议遵守）
- 功能色使用
- 组件变体
- 图像风格

第三层：参考规范（按需使用）
- 动效风格
- 插画规范
- 复杂组件

分层输出：
- 核心规范：简洁单页，设计师必须掌握
- 完整规范：详细文档，可按需查阅
```

### 错误类型2：规范与实际执行脱节
**识别：** 规范美好但实际无法执行
**处理：**
```
【规范落地性审查】

请评估以下问题：

1. 字体是否免费可商用？
   - 如有版权字体，建议提供替代方案

2. 颜色数量是否过多？
   - 如超过10种，建议精简

3. 组件状态是否完整？
   - 必须包含：默认、悬停、激活、禁用

4. 规范是否考虑开发实现？
   - 建议与开发团队确认实现可行性

根据评估结果调整规范。
```

### 错误类型3：多平台兼容性问题
**识别：** 规范在某些平台无法实现
**处理：**
```
【多平台兼容性提示】

以下规范可能存在兼容性问题：

[规范项]：在[平台]上可能无法实现
- 原因：[说明]
- 建议：[调整方案]

建议：
1. 在输出规范前与开发团队沟通
2. 提供各平台的替代方案
3. 标注各平台差异
```

## FAQ（常见问题）

### Q1：品牌视觉系统需要包含哪些内容？
**A：** 基础系统应包含：1）Logo使用规范；2）色彩系统；3）字体系统；4）图像风格；5）图标规范。完整系统可增加：动效规范、插画规范、版式系统、数据可视化规范等。

### Q2：中小企业是否需要完整的品牌视觉系统？
**A：** 不需要一步到位。建议分阶段建设：1）MVP阶段：Logo+主色+字体（可执行即可）；2）发展阶段：补充图标、组件规范；3）成熟阶段：完整系统+动效+详细指南。根据资源情况灵活调整。

### Q3：如何确保团队执行品牌规范？
**A：** 确保执行的方法：1）规范文档要简洁易懂；2）提供可直接使用的设计资源（组件库、图标库）；3）与开发团队对齐，提供代码层面的规范；4）定期检查执行情况；5）将规范作为入职培训内容。

### Q4：品牌视觉系统需要多久更新一次？
**A：** 建议：1）品牌初创期，规范制定后1-2年内保持稳定；2）品牌升级时同步更新视觉系统；3）日常小调整可发布补充说明；4）避免频繁变动，影响品牌资产积累。

### Q5：如何平衡品牌一致性和创意自由？
**A：** 平衡方法：1）核心元素（Logo、主色）严格规范；2）创意场景给予灵活空间；3）明确哪些必须规范，哪些可以灵活；4）提供"规范内创意"的优秀示例。

### Q6：品牌视觉系统与设计系统有什么区别？
**A：** 区别：1）品牌视觉系统关注品牌识别的视觉表达；2）设计系统关注产品设计和开发的效率。品牌视觉系统更偏品牌层面，设计系统更偏产品层面。两者有重叠，通常设计系统会继承品牌视觉系统。
