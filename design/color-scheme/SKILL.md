# color-scheme

## 功能说明

提供专业的配色方案建议，帮助建立视觉统一的设计语言。包括色彩心理学应用、色盲友好设计、色彩系统构建等专业内容。

---

## 假设声明（Assumptions）

### 前提条件
- 目标平台和使用场景已知
- 品牌调性或产品定位已明确
- 目标用户群体画像已定义
- 主要使用环境已确认（室内/室外、亮/暗环境）

### 适用范围
- 移动App配色设计
- Web页面配色设计
- 品牌色彩系统
- 运营活动配色
- 数据可视化配色
- 暗色模式配色

### 不适用场景
- 需要精确品牌色的场合（应以品牌指南为准）
- 专业印刷品配色（CMYK/Pantone专业领域）
- 涉及版权的特定艺术配色

## 详细工作流程（Detailed Workflow）

### 阶段1：需求理解

**步骤1：品牌调性分析**
```
检查点：
□ 产品类型已确认
□ 目标用户已明确
□ 品牌关键词已提取
□ 使用场景已了解
```

1. 分析产品定位（高端/大众、专业/休闲）
2. 了解目标用户特征（年龄、性别、文化背景）
3. 提取品牌关键词（3-5个核心词）
4. 确认主要使用场景

**步骤2：色彩心理学映射**
1. 将品牌关键词转化为色彩属性
2. 考虑文化差异的影响
3. 平衡情感表达与功能性

### 阶段2：色彩方案构建

**步骤1：主色确定**
1. 从品牌关键词推导主色方向
2. 选择基础色相
3. 确定饱和度和明度
4. 验证品牌适用性

**步骤2：辅色构建**
1. 分析主色的邻近色/互补色
2. 确定辅色角色（支持/平衡主色）
3. 确保与主色的协调性
4. 考虑使用场景的多样性

**步骤3：功能色定义**
1. 定义成功/警告/错误/信息状态色
2. 确保状态色之间的可区分性
3. 验证与主色的协调

**步骤4：中性色构建**
1. 确定背景色层级
2. 定义文字色层级
3. 设定边框/分隔线色
4. 确保足够的对比度

### 阶段3：验证与优化

**步骤1：色盲友好检查**
```
检查点：
□ 红绿色盲可辨识
□ 蓝黄色盲可辨识
□ 单色可辨识性
□ 亮度对比达标
```

**步骤2：跨设备验证**
1. 检查在不同屏幕的显示效果
2. 验证暗色模式的适配
3. 确保渐变色的平滑过渡

**步骤3：应用场景验证**
1. 在典型界面中预览效果
2. 检查图标和插图的搭配
3. 验证多语言文字的可读性

## 边界情况（Edge Cases）

### 情况1：已有品牌色约束
**场景：** 企业已有指定品牌色，需在此基础上构建配色
**处理：**
```
【品牌色约束下的配色方案】

已了解您的品牌色约束：[品牌色值]

配色方案将在此基础上构建：
1. 以[品牌色]作为主色基准
2. 辅色选择与品牌色协调的配色
3. 确保符合品牌使用规范

请提供：
- 品牌色的使用限制（如是否只能全血或可调色）
- 品牌指南中的禁忌色
- 允许的渐变使用范围
```

### 情况2：多平台配色需求
**场景：** 同一产品需适配iOS、Android、Web多平台
**处理：**
```
【多平台配色方案】

不同平台有不同的设计语言：
- iOS：偏向明亮、柔和的配色
- Android：Material Design的鲜艳色调
- Web：更灵活的配色选择

建议方案：
1. 核心色值保持一致
2. 平台特性通过辅色微调
3. 平台特殊效果（如iOS毛玻璃）需额外定义

请指明重点平台，我会提供针对性的调整建议。
```

### 情况3：深色模式需求
**场景：** 需要同时支持亮色和暗色模式
**处理：**
```
【暗色模式配色方案】

暗色模式配色要点：
1. 主色在暗色背景上的表现
2. 中性色需要反转定义
3. 强调色需要更高的饱和度
4. 避免纯白背景(#FFFFFF)

我将提供：
- 浅色模式完整配色
- 暗色模式完整配色
- 明暗模式的平滑过渡方案
```

### 情况4：色盲用户友好需求
**场景：** 产品需面向色盲用户群体
**处理：**
```
【色盲友好配色方案】

针对[红绿色盲/蓝黄色盲/全色盲]的友好方案：

1. 避免单独依赖颜色传递信息
2. 使用亮度/形状/纹理作为辅助
3. 选择色盲友好的色彩组合
4. 提供高对比度模式

配色选择建议：
- 蓝橙色组合（红绿色盲友好）
- 高对比度配色（普遍友好）
- 使用图标+文字双重标识

请说明需要支持的具体色盲类型。
```

## 错误处理（Error Handling）

### 错误类型1：主色过于鲜艳或刺眼
**识别：** 饱和度高于80%或明度低于20%的主色
**处理：**
```
【主色饱和度调整建议】

[色值]饱和度过高，可能导致视觉疲劳。

建议调整方案：
- 降低饱和度至50-70%
- 适当提高明度
- 或使用邻近的柔和色相

调整后：
#XXXXXX → #XXXXXX
```

### 错误类型2：对比度不达标
**识别：** 文字与背景的对比度低于WCAG标准
**处理：**
```
【对比度问题】

[文字色]与[背景色]的对比度为[X:1]，未达到：
- WCAG AA标准：4.5:1（普通文字）
- WCAG AA标准：3:1（大文字）

建议方案：
1. 深化文字色
2. 浅化背景色
3. 或调整其中一方

调整后对比度：[X:1]
```

### 错误类型3：渐变色不协调
**识别：** 渐变起点和终点的色相差异过大
**处理：**
```
【渐变色调整】

当前渐变[起点色] → [终点色]的色相差异较大，可能显得突兀。

建议方案：
1. 选择色相接近的颜色
2. 或使用同一色相的不同明度
3. 控制渐变角度（45-135度最佳）

优化后的渐变：
linear-gradient(XXdeg, #XXXXXX 0%, #XXXXXX 100%)
```

### 失败案例修正：对比度不足

**输入：**
```
[设计了一个登录页面，配色为蓝底白字]
配色：#4A90E2 背景，#FFFFFF 文字
```

**输出（发现对比度问题）：**
```
【配色审查：登录页面】

## 检查结果
- 背景色：#4A90E2
- 文字色：#FFFFFF
- 对比度：4.6:1

## 对比度评估
- WCAG AA（普通文字）：需要≥4.5:1 ✓ 通过
- WCAG AA（大文字）：需要≥3:1 ✓ 通过
- WCAG AAA（普通文字）：需要≥7:1 ✗ 未达标

## 问题分析
虽然满足最低标准，但在低质量屏幕或光线较强的环境下，可读性可能不佳。

## 建议优化

### 方案1：加深背景色
- 背景色：#3A7BC8
- 对比度提升至：5.8:1
- 优点：保留蓝色调，改善可读性

### 方案2：使用深色文字
- 文字色：#E8E8E8
- 对比度提升至：6.2:1
- 优点：更柔和的视觉效果

### 推荐
**方案2**：使用#E8E8E8作为文字色，在保持明亮感的同时提升可读性。

## 优化后配色
| 元素 | 优化前 | 优化后 |
|------|--------|--------|
| 背景 | #4A90E2 | #4A90E2（不变） |
| 主文字 | #FFFFFF | #E8E8E8 |
| 辅助文字 | #E0E0E0 | #C0C0C0 |
```

## FAQ（常见问题）

### Q1：如何选择主色？
**A：** 主色选择应考虑：1）与品牌调性匹配；2）目标用户的情感共鸣；3）行业的色彩趋势；4）平台的设计规范（如Material Design有推荐色板）。建议从品牌关键词出发，选择能传达核心情感的色相。

### Q2：辅色和主色的关系是什么？
**A：** 辅色用于支持主色，常见策略：1）邻近色（色相接近，协调性强）；2）互补色（色相对比，突出主色）；3）分裂互补色（既有对比又不过于冲突）。辅色使用比例约为15-25%。

### Q3：为什么需要功能色？
**A：** 功能色（成功/警告/错误/信息）是用户界面中传达状态的重要工具。用户会逐渐学会识别颜色含义，减少文字阅读负担。功能色需要确保与主色的协调，同时保证相互之间的可辨识性。

### Q4：渐变色使用有什么注意事项？
**A：** 渐变色使用建议：1）色相选择应接近，避免刺眼的色彩跳跃；2）渐变角度45-135度最自然；3）注意渐变中间值的可读性；4）避免过于花哨的双色渐变；5）考虑暗色模式下的效果。

### Q5：如何验证色盲友好性？
**A：** 可使用模拟工具（如 Stark 插件）查看色盲视角。重点检查：1）红绿色盲视角下是否可区分；2）蓝黄色盲视角下是否可区分；3）重要信息不单独依赖颜色传达。最佳实践是配合图标、文字标签使用颜色。

### Q6：品牌色和UI色如何平衡？
**A：** 如果品牌色较鲜艳，可用于小面积的强调元素（如按钮、图标）；背景和大面积色块使用中性色。如果品牌色柔和，可更大胆地作为主色使用。关键是保持层次感，避免所有元素都使用品牌色。
