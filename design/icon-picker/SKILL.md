# icon-picker

## 功能说明

帮助选择适合产品风格的图标，提供图标库推荐、风格建议、尺寸规范和使用指导。确保图标的视觉一致性和功能可用性。

---

## 假设声明（Assumptions）

### 前提条件
- 目标平台已明确（iOS/Android/Web/多平台）
- 产品风格已定义
- 目标用户已了解
- 需要图标的功能场景已知

### 适用范围
- 移动App图标选型
- Web页面图标
- 桌面应用图标
- 后台系统图标
- 功能图标选择
- 品牌图标设计

### 不适用场景
- 需要原创设计的情景图标
- 涉及复杂插图的场景
- 版权要求严格的特定图标

## 边界情况（Edge Cases）

### 情况1：多平台一致性需求
**场景：** 同一产品需在iOS、Android、Web保持图标一致
**处理：**
```
【多平台图标方案】

多平台一致性建议：

1. 选择跨平台图标库
   - 推荐：Lucide、Phosphor、Remix Icon
   - 这些图标库提供SVG格式，跨平台兼容性好

2. iOS注意事项
   - 遵循Apple SF Symbols风格
   - 圆角使用系统默认值

3. Android注意事项
   - 可使用Material Symbols
   - 或选择与Material风格接近的图标库

4. Web注意事项
   - 确保字体图标或SVG加载性能
   - 考虑深色模式适配

建议方案：统一使用Lucide图标库，iOS端可使用SF Symbols补充特殊图标
```

### 情况2：混用多种图标库
**场景：** 用户希望结合多个图标库的优点
**处理：**
```
【图标库混用警告】

不建议混用多个图标库，原因：

1. 视觉不一致
   - 不同图标库的线条粗细、比例、风格会有差异
   - 用户能感知到视觉不统一

2. 维护困难
   - 需要管理多套图标资源
   - 更新时需同步修改

3. 增加体积
   - 多个图标库会增加包体积

建议方案：
- 选择能满足80%需求的图标库
- 特殊图标单独定制
- 或统一替换为同一图标库
```

### 情况3：品牌定制图标需求
**场景：** 需要专属品牌风格的图标
**处理：**
```
【品牌图标定制建议】

品牌图标定制流程：

1. 基础风格定义
   - 基于品牌视觉系统确定基础风格
   - 线条粗细、圆角大小、视觉重量

2. 核心图标设计
   - 设计8-12个核心功能图标
   - 确保风格一致性

3. 图标规范文档
   - 定义品牌图标的使用规范
   - 提供各场景的尺寸参考

4. 扩展图标库
   - 核心图标满意后扩展到完整图标库

推荐品牌图标风格选项：
- 线性定制：线条统一，可添加品牌特征
- 填充定制：更具品牌识别度
- 混合风格：核心图标定制，其他使用图标库
```

### 情况4：暗色模式图标适配
**场景：** 需要在浅色和深色模式下使用图标
**处理：**
```
【暗色模式图标方案】

暗色模式图标处理方式：

1. 颜色适配方案
   - 浅色模式：#000000 或深灰色
   - 深色模式：#FFFFFF 或浅灰色
   - SVG图标使用 currentColor 自动适配

2. 方案选择

   方案A：单一SVG + currentColor
   - 图标使用 currentColor
   - 文字颜色变化时自动适配
   - 推荐用于大多数场景

   方案B：双套图标
   - 提供浅色/深色两套图标
   - 通过CSS或平台特性切换
   - 适用于有特殊品牌色的情况

3. 推荐设置
   图标颜色：currentColor
   确保图标容器设置了正确的文字颜色
```

## 错误处理（Error Handling）

### 错误类型1：图标风格与产品调性不匹配
**识别：** 简约产品使用了过于复杂的图标
**处理：**
```
【风格不匹配提示】

当前选择的图标风格与产品调性可能存在差异：

产品风格：简约、现代
图标风格：细节丰富、装饰性强

建议调整：
- 简约产品 → 选择 Feather、Lucide 等简洁风格
- 活泼产品 → 可选择带圆角的图标
- 专业产品 → 选择结构清晰、中性风格的图标

推荐替代：[图标库名称]
```

### 错误类型2：图标尺寸规范缺失
**识别：** 未提供具体的尺寸和间距指导
**处理：**
```
【图标尺寸规范建议】

缺少图标尺寸规范，以下是通用建议：

基础尺寸：
- 功能图标：24x24px
- 装饰图标：16x16px / 20x20px
- 大图标：32x32px / 48x48px

触控区域（移动端）：
- 最小触控：44x44px
- 推荐触控：48x48px
- 图标在触控区域内居中

视觉尺寸 vs 触控尺寸：
┌──────────────────┐
│    44x44px       │  ← 触控区域
│   ┌────────┐     │
│   │ 24x24  │     │  ← 视觉图标
│   └────────┘     │
└──────────────────┘
```

### 错误类型3：图标版权问题
**识别：** 使用的图标库有商业使用限制
**处理：**
```
【图标版权提醒】

您选择的图标库存在使用限制：

[图标库名称]
- License：[许可证类型]
- 商业使用：[允许/需授权/不允许]
- Attribution：[是否需要署名]

免费商用推荐：
- Lucide (MIT)
- Feather (MIT)
- Tabler Icons (MIT)
- Phosphor (MIT)
- Remix Icon (Apache 2.0)

使用前请确认许可证要求，如有商业使用需求，建议选择MIT或Apache 2.0许可证的图标库。
```

## FAQ（常见问题）

### Q1：如何选择图标库？
**A：** 选择图标库考虑因素：1）风格匹配度；2）图标数量是否满足需求；3）更新维护频率；4）许可证（MIT最宽松）；5）社区支持度。推荐Lucide作为首选，它覆盖全面、风格稳定、许可证友好。

### Q2：可以同时使用线性图标和填充图标吗？
**A：** 可以，但需要遵循明确的规则：1）同一功能模块内风格统一；2）如导航用线性，内容区可用填充；3）保持清晰的视觉逻辑。常见做法是：导航/标签栏用线性，功能操作区可用填充。

### Q3：图标需要配合文字使用吗？
**A：** 视场景而定：1）导航图标建议配合文字标签（提低认知成本）；2）功能按钮图标建议配合文字（明确操作含义）；3）装饰性图标可单独使用；4）高频率使用的图标可以单独保留。

### Q4：如何确保图标在不同屏幕下清晰？
**A：** 确保图标清晰的方法：1）基础尺寸使用24px或以上；2）提供@2x和@3x变体；3）线条图标使用2px描边（移动端）；4）确保对比度足够。SVG格式是最佳选择，它支持任意缩放。

### Q5：自定义图标的费用大概是多少？
**A：** 定制图标取决于复杂度：1）简单功能图标（20-50个）：约¥5000-15000；2）完整图标库（200+个）：约¥20000-50000；3）品牌图标系统：¥50000+。建议先定制核心图标（8-12个），后续按需扩展。

### Q6：图标库更新后如何处理？
**A：** 图标库更新处理：1）关注版本更新日志；2）评估更新内容是否需要跟进；3）小版本更新可选择性跟进；4）大版本更新建议评估后统一替换。建议锁定图标库版本，避免自动更新导致界面变化。
