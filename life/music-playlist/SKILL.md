# music-playlist v3.0

音乐播放列表生成技能。2026年集成AI音乐品味分析、智能歌单生成、音乐情绪匹配能力。

## 功能说明

根据用户需求生成音乐播放列表，适合不同场景和心情。

---

## 假设声明（Assumptions）

- 用户有听音乐的需求
- 用户能够使用音乐播放平台
- 用户愿意尝试新的音乐风格

---

## 输入输出（I/O Format）

### 标准输入格式
```
【基本信息】
- 听歌场景：[工作学习/运动健身/通勤/睡前/放松/聚会]

【偏好设置】
- 喜欢的风格：[流行/摇滚/爵士/古典/电子/民谣]
- 喜欢的年代：[70s/80s/90s/00s/10s/都可以]
- 语言偏好：[中文/英文/日语/都可以]

【心情/情绪】
- 当前心情：[欢快/忧郁/励志/平静/浪漫/焦虑]
- 想要的感觉：[提神/放松/治愈/振奋]
```

### 标准输出格式
```
【歌单名称】：XXX

【风格】：...

【推荐曲目】
| # | 歌曲 | 艺人 | 时长 | 特点 |

【在线收听】
- Spotify：...
- 网易云：搜索"XXXX"
```

## 2026年新功能

| 功能 | 说明 |
|------|------|
| **AI音乐品味分析** | 分析用户音乐偏好，预测喜欢程度 |
| **智能歌单生成** | 根据场景和心情自动生成完整歌单 |
| **音乐情绪匹配** | 识别音乐情绪特征，精准匹配需求 |
| **BPM节奏适配** | 运动歌单自动匹配目标心率 |

---

## 工作流程（Workflow）

1. **需求分析** - 确认听歌场景、音乐风格偏好、心情状态
2. **歌单生成** - 选择符合场景的音乐风格，确保多样性
3. **曲目说明** - 说明每首歌的特点和推荐理由
4. **延展建议** - 提供风格相似的其他推荐

---

## 质量验证清单（Quality Checklist）

- [ ] 歌曲信息（名称、艺人、时长）准确
- [ ] 歌曲适合指定场景
- [ ] 风格统一但有多样性
- [ ] 有清晰的推荐理由
- [ ] 提供收听渠道信息
