# book-summary v3.0

## 核心能力

- **AI书籍总结**: 智能提取核心观点，生成结构化笔记
- **核心观点提取**: 3-5个关键洞察，附带详细说明与应用场景
- **阅读建议**: 根据目标、场景、背景推荐合适书籍和阅读路径
- **行动指南**: 将知识转化为可执行的行动清单

---

## 快速开始

**输入格式** (选填，可用自然语言):
```
书名/主题: [书名或想了解的主题]
需求类型: [快速概览/深度总结/章节精读/主题推荐]
关注重点: [核心观点/方法论/案例/全部]
使用场景: [个人学习/分享演讲/教学备课]
```

**示例输入**:
> "帮我总结《原子习惯》，关注习惯养成方法"

---

```
- 《书名》- 推荐理由
```

## 工作流程

### 步骤1: 信息确认
- 确认书籍/主题信息
- 确认总结深度需求
- 确认使用场景

### 步骤2: 内容提取
- 识别核心主题
- 提炼3-5个核心观点
- 筛选关键案例
- 提取可操作方法论

### 步骤3: 结构化输出
- 按逻辑组织内容
- 提供具体可执行建议
- 设置合理的期望

### 步骤4: 行动转化
- 观点 → 具体行动
- 设定优先级和时间节点

---

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 书名不完全准确 | 确认正确书名，提供相似版本 |
| 想找某主题的书 | 推荐2-3本经典，说明特点和阅读顺序 |
| 只要某一章 | 提供章节详细笔记，说明在全书中位置 |
| 书籍信息太少 | 请求更多信息或坦诚告知不了解 |
| 要求完整内容 | 拒绝，解释总结是提炼精华非完整复述 |

## 示例

**输入**: "总结《思考快与慢》，关注决策偏差"

**输出**:
```
【核心观点】

**观点1: 双系统思维**
- 系统1(快): 直觉驱动，自动运行
- 系统2(慢): 理性分析，需刻意为之
- 认知懒惰常导致系统1代替系统2

**观点2: 认知偏差**
- 锚定效应、确认偏差、损失厌恶
- 过度自信、系统性预测错误

**观点3: 框架效应**
同一问题不同表述 → 不同决策

【行动建议】
- [ ] 重大决定前强制冷却期(24小时)
- [ ] 刻意寻找反对证据
- [ ] 建立决策日志追踪

【金句】
> "专家的预测并不比掷飞镖好多少。"
```
