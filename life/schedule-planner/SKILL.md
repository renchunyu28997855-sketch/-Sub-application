# schedule-planner

## 功能说明
帮助用户规划日程，优化时间分配，提升工作和生活效率。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户拥有基本的日程管理意识，愿意投入时间规划和复盘
- 用户能够区分任务的优先级（紧急/重要矩阵）
- 用户有相对固定的工作/生活节奏可供规划参考

### 适用范围
- 适合：学生、职场人士、自由职业者、需要日常规划的人群
- 不适合：时间完全被他人支配（如24小时护理人员）、突发情况较多的工作

### 已知限制
- 本技能不提供具体任务执行的时长精确估算
- 不涉及跨时区的复杂日程协调
- 不包含团队协作或会议安排功能

---

## 2. 输入输出（I/O Format）

### 标准输入格式
```
【基本信息】
- 日期：[具体日期或"今天"/"明天"/"本周"]
- 角色：[学生/职场/自由职业等]
- 可用时间：[总时长]
- 特殊约束：[如有]

【任务信息】
- 待办任务：[任务列表]
- 预计时长：[每个任务预估]
- 截止时间：[如有]
- 优先级：[高/中/低]

【偏好设置】
- 时间块长度：[25min/45min/60min等]
- 休息频率：[每工作X小时休息Y分钟]
- 禁止打扰时段：[如午休等]
```

### 标准输出格式
```
【今日/本周概览】
- 重点任务数：X
- 预计总时长：X小时
- 完成率目标：X%

【时间块规划】
| 时间 | 任务 | 类型 | 状态 |
|------|------|------|------|
| HH:00-HH:00 | 任务名 | 工作/休息 | □ |

【待办清单】
- [ ] 任务1（优先级：高，截止：XX:XX）
- [ ] 任务2（优先级：中）

【复盘提示】
- 建议记录点：...
```

## 3. 详细工作流程（Detailed Workflow）

### 步骤1：信息收集与解析
**检查点**：
- [ ] 确认日期和时间范围
- [ ] 收集所有待办任务
- [ ] 识别截止时间要求
- [ ] 了解用户的角色和场景

### 步骤2：任务分析与分类
**检查点**：
- [ ] 标记有截止时间的任务（紧急）
- [ ] 按优先级排序（高/中/低）
- [ ] 识别任务类型（深度工作/事务性/休息）
- [ ] 估算每个任务的实际时长

### 步骤3：时间块分配
**检查点**：
- [ ] 预留缓冲时间（通常10-15%）
- [ ] 将高优先级任务安排在精力最佳时段
- [ ] 避免在禁止打扰时段安排需要专注的任务
- [ ] 确保休息时间合理分布

### 步骤4：输出格式化
**检查点**：
- [ ] 时间块无重叠
- [ ] 任务名称清晰
- [ ] 包含复盘提示
- [ ] 提供执行建议

### 步骤5：优化建议
**检查点**：
- [ ] 检查是否有过长的时间块（>90分钟需拆分）
- [ ] 检查是否有连续过短的时间块（<15分钟可合并）
- [ ] 检查是否充分利用黄金时段

## 4. 边界情况（Edge Cases）

### 情况1：输入为空
**场景**：用户仅说"帮我规划今天"
**处理方式**：
1. 询问用户今天有哪些任务需要完成
2. 如用户无法明确，假设通用工作日结构（上午深度工作，下午事务性工作）
3. 提供模板让用户填充

### 情况2：任务过多超过可用时间
**场景**：用户列了10小时的任务但只有8小时
**处理方式**：
1. 明确告知时间缺口
2. 按优先级筛选出必须在当天完成的任务
3. 建议将低优先级任务移至其他日期

### 情况3：任务截止时间冲突
**场景**：两个任务都标记为"今天必须完成"
**处理方式**：
1. 识别两个任务的相对紧急程度
2. 建议拆分任务或并行处理
3. 提供超时风险提示

### 情况4：输入格式不完整
**场景**：用户只说"我很忙，没时间"
**处理方式**：
1. 提供标准输入模板
2. 询问关键信息：有哪些任务？截止时间？
3. 基于有限信息给出最小可行规划

### 情况5：跨天任务
**场景**：用户说"这个项目要做一周"
**处理方式**：
1. 将项目拆解为每日任务
2. 按周视图分配任务
3. 标注里程碑节点

---

## 5. 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理策略 |
|----------|----------|----------|
| 时间重叠 | 同一时间安排两个任务 | 重新分配或合并任务 |
| 时长不足 | 任务预估时长超过可用时间 | 压缩任务范围或延期 |
| 优先级矛盾 | 高优先级任务被安排在低效时段 | 调整时间块位置 |
| 缺少缓冲 | 连续任务无休息间隔 | 插入休息时间块 |
| 截止时间错误 | 任务截止时间早于当前时间 | 提示并建议调整 |

## 6. 质量验证清单（Quality Checklist）

### 输出前检查
- [ ] 所有任务都在可用时间内
- [ ] 时间块无重叠冲突
- [ ] 高优先级任务在黄金时段（通常上午）
- [ ] 每个时间块长度合理（15-90分钟）
- [ ] 包含足够的休息间隔
- [ ] 有明确的复盘提示
- [ ] 输出格式符合标准模板

### 验证问题
1. 总任务时长是否超过可用时间？
2. 是否有任务被安排在禁止打扰时段？
3. 深度工作任务是否有足够连续时长？
4. 紧急任务是否被优先安排？

### 边界/异常案例

**输入**：
```
角色：学生
日期：明天
可用时间：6小时（周末）
任务：
- 完成论文（重要，10小时）- 截止：周一
- 打扫房间（中等，2小时）- 截止：周日
- 朋友聚会（社交，4小时）- 今天
- 看电影（放松，2小时）
```

**处理**：
```
【问题识别】
⚠️ 任务总时长超过可用时间（10+2+4+2=18小时 > 6小时）
⚠️ 论文截止时间为周一，必须完成

【建议方案】
方案A（优先论文）：
- 明天：论文6小时（完成60%）
- 周日：论文4小时 + 打扫房间2小时

方案B（平衡生活）：
- 明天：论文3小时 + 朋友聚会3小时
- 周日：论文7小时 + 打扫房间2小时（放弃看电影）

【推荐】
选择方案A，确保论文进度，避免周一熬夜。
```

### 失败案例及修正

**原始规划**（有问题）：
```
9:00-12:00 撰写论文
12:00-13:00 午休
13:00-18:00 继续写论文（5小时连续）
```

**问题识别**：
- 下午连续工作5小时，效率会下降
- 没有留出处理突发事务的时间

**修正后**：
```
9:00-11:30 撰写论文（上午高效期）
11:30-12:00 处理邮件/休息
12:00-13:00 午休
13:00-15:00 撰写论文（下午第一段）
15:00-15:15 短暂休息
15:15-17:00 撰写论文（下午第二段）
17:00-17:30 处理杂事/弹性时间
```

## 8. FAQ（常见问题）

**Q1：如何确定任务的优先级？**
A：使用四象限法则：
- 紧急且重要 → 立即处理
- 重要不紧急 → 计划处理
- 紧急不重要 → 委托他人
- 不紧急不重要 → 考虑删除

**Q2：番茄工作法和时间块有什么区别？**
A：番茄工作法适合事务性任务（25分钟工作+5分钟休息）；时间块适合深度工作（45-90分钟连续工作+15分钟休息）

**Q3：如果任务被意外中断怎么办？**
A：记录中断时间，评估剩余任务是否能在原定时间内完成。如不能，适当压缩任务范围或顺延后续任务。

**Q4：如何处理临时增加的任务？**
A：评估新任务的优先级和时长：
- 高优先级：替换同优先级较低的任务
- 低优先级：移至待办清单，在有空余时处理

**Q5：周末和工作日的规划方式有区别吗？**
A：有。工作日以工作为主，周末建议增加休闲和家庭时间比例。
