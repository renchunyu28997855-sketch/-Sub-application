# recipe-recommender

## 功能说明
根据用户需求推荐合适的食谱，提供烹饪方法。

---

## 1. 假设声明（Assumptions）

### 前提条件
- 用户有基本的烹饪条件和意愿
- 用户能够获取推荐食谱所需的基本食材和调味料
- 用户遵守基本的厨房安全规范

### 适用范围
- 适合：家庭日常饮食、特殊饮食需求（减脂、素食等）、节日准备
- 不适合：专业厨师、特殊医学饮食（需营养师指导）

### 已知限制
- 本技能提供家常菜谱，不适合专业餐饮场景
- 不提供分子料理等高级技术
- 不包含需特殊设备的专业食谱
- 不保证食材的可获取性（因地区和季节而异）

## 3. 详细工作流程（Detailed Workflow）

### 步骤1：需求分析
**检查点**：
- [ ] 确认用餐人数和场景
- [ ] 了解口味偏好和禁忌
- [ ] 确定烹饪时间要求
- [ ] 考虑季节因素

### 步骤2：食材搭配
**检查点**：
- [ ] 考虑应季食材
- [ ] 保证营养均衡（主食+蛋白+蔬菜）
- [ ] 食材之间是否搭配
- [ ] 是否符合特殊饮食要求

### 步骤3：食谱筛选
**检查点**：
- [ ] 筛选难度合适的食谱
- [ ] 确保时间在要求范围内
- [ ] 考虑食材的可获取性
- [ ] 搭配多样性（不重复）

### 步骤4：详细指导
**检查点**：
- [ ] 提供清晰的步骤
- [ ] 说明火候和时机
- [ ] 提供小技巧提升成功率
- [ ] 提醒常见错误

### 步骤5：营养提示
**检查点**：
- [ ] 提供热量估算
- [ ] 说明营养搭配建议
- [ ] 给出特殊人群注意事项
- [ ] 提供保存方式

## 4. 边界情况（Edge Cases）

### 情况1：用户厨房条件有限
**场景**：只有电饭煲，没有炒锅
**处理方式**：
1. 推荐蒸、煮、炖类食谱
2. 教用户用电饭煲做多种菜
3. 提供"一锅熟"食谱

### 情况2：食材极度有限
**场景**：冰箱里只有鸡蛋和米饭
**处理方式**：
1. 提供最简单的蛋炒饭变体
2. 教用户用鸡蛋做出3种不同菜
3. 建议尽快采购的必要食材

### 情况3：时间非常紧
**场景**：只有15分钟做饭
**处理方式**：
1. 推荐快手菜（5-10分钟）
2. 提供微波炉食谱
3. 建议半成品+简单加工

### 情况4：有食物过敏
**场景**：对海鲜过敏
**处理方式**：
1. 排除所有海鲜类食谱
2. 推荐高蛋白替代（鸡胸肉、豆制品）
3. 注意隐藏过敏原（如蚝油含海鲜成分）

### 情况5：节日/特殊场合
**场景**：中秋节想自己做月饼
**处理方式**：
1. 提供简单月饼食谱
2. 说明需要特殊工具（如月饼模具）
3. 提前提醒准备食材

---

## 5. 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理策略 |
|----------|----------|----------|
| 食材不匹配 | 推荐了用户没有的食材 | 提供替代食材或替代食谱 |
| 难度过高 | 用户是新手但推荐复杂菜 | 降低难度或提供简化版 |
| 口味不匹配 | 推荐了辣菜但用户不吃辣 | 更换为不辣或微辣的食谱 |
| 营养不均衡 | 连续推荐多道肉菜 | 加入蔬菜和蛋白质均衡的选项 |

## 8. FAQ（常见问题）

**Q1：如何让青菜保持翠绿？**
A：1. 焯水时加少许盐和油；2. 不要盖锅盖；3. 捞出后立即过凉水；4. 炒制时间要短。

**Q2：炒菜什么时候放盐？**
A：一般情况下，出锅前放盐更健康（减少盐摄入）。但炖菜、红烧菜可以早放，让味道渗透。

**Q3：如何判断油温？**
A：1. 目测：油面有纹路，冒出细烟；2. 木筷测试：筷子周围冒泡；3. 滴一滴水进去，瞬间炸开说明油温够。

**Q4：食材用不完怎么保存？**
A：1. 青菜：纸巾包裹冷藏，3天内食用；2. 肉类：分装冷冻，一周内食用；3. 豆制品：冷藏保存，2天内食用。

**Q5：如何让肉炒出来更嫩？**
A：1. 切肉时逆纹切；2. 提前腌制（加少量淀粉、油）；3. 大火爆炒，快速出锅；4. 不要过早放盐。

**Q6：厨房没有某调料怎么办？**
A：可以用其他调料替代：
- 料酒 ≈ 黄酒/白酒（去腥用）
- 生抽 ≈ 酱油
- 蚝油 ≈ 酱油+少许糖
- 鸡精 ≈ 味精
