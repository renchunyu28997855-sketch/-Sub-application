# 模具工厂 AI 降本增效系统设计规范

**版本**: v1.0
**日期**: 2026-04-13
**状态**: 已批准

---

## 一、项目概述

### 1.1 目标
为模具工厂构建一套完整的 AI 降本增效系统，包含：AI 需求分析报告生成、方案输出、以及支持大屏展示的 12 部门 Agent 协作可视化 Dashboard。

### 1.2 交付范围
- **AI 方案生成器**：输入工厂数据 → 输出完整转型方案 + 可导出 PDF
- **工厂大脑 Dashboard**：12 个部门 Agent 协作可视化 + 业务流动态模拟 + 大屏适配

### 1.3 用户画像
- 模具工厂老板/管理者
- 关注：降本、增效、可落地
- 使用场景：大屏展示、企业汇报、AI 转型决策参考

## 二、系统架构

### 2.1 整体架构（纯前端无后端）

```
┌──────────────────────────────────────────────────────────┐
│                      index.html (单文件)                   │
├──────────────────────────────────────────────────────────┤
│  ┌────────────────┐    ┌────────────────────────────┐   │
│  │  AI 方案生成器   │    │     工厂大脑 Dashboard      │   │
│  │  · 输入表单     │    │  · Canvas 粒子动画          │   │
│  │  · 报告预览     │    │  · Agent 状态环             │   │
│  │  · PDF 导出     │    │  · 协作链路可视化           │   │
│  └────────────────┘    └────────────────────────────┘   │
├──────────────────────────────────────────────────────────┤
│              12部门 Agent 模拟引擎 (JavaScript)           │
│  · 事件驱动总线   · 状态机   · 消息传递   · 业务逻辑模拟  │
└──────────────────────────────────────────────────────────┘
```

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 展示层 | HTML5 + CSS3 + Vanilla JS | 单文件，无构建工具 |
| 可视化 | Canvas API + ECharts | 粒子系统 + 图表 |
| PDF 导出 | jsPDF + html2canvas | 报告生成 |
| 数据持久化 | LocalStorage | 保存工厂数据 |
| AI 调用 | DeepSeek API (前端代理) | 方案生成 |

### 2.3 12 个部门 Agent 定义

| 编号 | 部门 | 核心职能 | 输入依赖 | 输出 |
|------|------|----------|----------|------|
| 1 | 订单管理 | 接收客户订单、订单分配 | 客户需求 | 生产工单 |
| 2 | 设计部 | 产品设计、CAM编程 | 订单需求 | 设计图纸 |
| 3 | 生产车间 | 加工制造、装配 | 设计图纸+工单 | 半成品/成品 |
| 4 | 质检部门 | IQC/OQC/过程检 | 在制品 | 质检报告 |
| 5 | 设备维护 | 点检/保养/维修 | 设备状态 | 维护工单 |
| 6 | 能源管理 | 电力/气/油监控 | 设备运行数据 | 能耗报告 |
| 7 | 仓储管理 | 原料/成品入库出库 | 生产需求 | 出库单 |
| 8 | 采购部 | 原料采购、供应商管理 | 库存预警 | 采购单 |
| 9 | 财务部 | 成本核算、付款 | 业务数据 | 财务报表 |
| 10 | 售后跟踪 | 客户服务、投诉处理 | 出库信息 | 服务记录 |
| 11 | 人力资源 | 排班、绩效、培训 | 生产计划 | 排班表 |
| 12 | 数据中枢 | 工厂大脑、跨部门协调 | 全部数据 | 分析建议 |

## 三、业务流程模拟

### 3.1 核心业务流（深度模拟）

```
[客户下单]
    ↓
[订单管理] → 解析订单 → 紧急度评估
    ↓
[设计部] → 根据订单要求设计 → 输出工艺文件
    ↓
[生产车间] → 按工艺加工 → 各工序流转
    ↓                          ↓
[质检部门] ← 抽检/全检 ←      [设备维护] → 加工中设备状态监控
    ↓
[仓储管理] → 成品入库
    ↓
[采购部] ← 库存预警 ← 原料消耗监控 → [能源管理]
    ↓
[财务部] → 成本计算 → 利润分析
    ↓
[售后跟踪] → 客户反馈收集
    ↑
[数据中枢] ← 全流程数据汇聚 → AI 分析建议
```

### 3.2 事件驱动模拟

| 事件类型 | 触发条件 | 影响部门 | 可视化表现 |
|----------|----------|----------|------------|
| 新订单进入 | 定时/随机 | 订单管理 | 粒子从左侧进入 |
| 质检告警 | 不良品率>5% | 质检+生产 | 红色告警脉冲 |
| 设备预警 | 温度/振动异常 | 设备维护 | 橙色闪烁节点 |
| 库存不足 | 某原料<20% | 仓储+采购 | 黄色警告气泡 |
| 能源超耗 | 能耗>阈值 | 能源管理 | 绿色波动曲线 |
| 订单完成 | 流程结束 | 全部 | 蓝色粒子爆发 |
| AI建议 | 定时分析 | 数据中枢 | 全局高亮 |

### 3.3 Agent 协作模式

**消息传递示例：**
```
质检部门 ──[不良率>阈值]──→ 设备维护 ──[检查设备]──→ 生产车间
                              │
                              ↓
                         数据中枢 ──[AI分析建议]──→ Dashboard
```

**工作流编排：**
- 订单优先级：紧急订单 → 设计部优先 → 生产车间绿色通道 → 质检快速通道
- 瓶颈检测：某工序积压 > 10件 → 告警 + 建议增派人员
- 动态调度：根据设备负载重新分配工单

### 4.3 部门节点设计

每个部门显示为环形状态指示器：

```
        ┌─────────┐
       /  设计部   \      ← 部门名称（霓虹青）
      │  ┌─────┐  │
      │  │ ◉◉◉ │  │      ← 状态环（3个点=忙碌）
      │  └─────┘  │
       \  ████  /       ← 今日任务量进度条
        └─────────┘
```

**状态颜色：**
- 空闲：`#00ff88`（霓虹绿）
- 忙碌：`#00d4ff`（霓虹青）
- 告警：`#ff6b35`（橙色）
- 故障：`#ff3366`（红色）

### 4.4 粒子系统规范

- **粒子来源**：各节点边缘
- **粒子颜色**：根据传输内容类型着色
  - 任务流：霓虹青 `#00d4ff`
  - 告警流：橙色 `#ff6b35`
  - 建议流：绿色 `#00ff88`
- **粒子大小**：2-6px 随机
- **粒子速度**：50-150px/秒
- **连线**：半透明渐变线条，连接相关节点
- **流动方向**：业务流方向

### 4.5 右侧信息面板

**实时数据卡片（每 2 秒刷新）：**
| 指标 | 值 | 趋势 |
|------|-----|------|
| 今日订单 | XX 单 | ↑/↓ |
| 在产工单 | XX 单 | ↑/↓ |
| 质检良品率 | XX% | ↑/↓ |
| 设备利用率 | XX% | ↑/↓ |
| 库存周转率 | XX 次 | ↑/↓ |
| 能耗kWh | XX | ↑/↓ |

## 五、AI 方案生成器设计

### 5.1 输入表单

| 字段 | 类型 | 说明 |
|------|------|------|
| 工厂名称 | text | 公司名称 |
| 员工人数 | number | 必填 |
| 厂房面积 | text | 如 5000㎡ |
| 年产值 | number | 万元 |
| 净利润率 | number | % |
| 主要产品 | checkbox | 注塑/冲压/压铸/其他 |
| 核心痛点 | multiselect | 最多选2个 |
| 预算范围 | radio | 低/中/高 |
| 期望周期 | radio | 快速/渐进/长期 |

### 5.2 输出报告（PDF）

**文档结构：**
1. 封面
2. 目录
3. 工厂现状分析
4. AI 转型方案（按6维度展开）
5. 实施路线图
6. 预期效果
7. 附录：技术架构

### 5.3 方案模板结构

```
# XX模具工厂 AI 降本增效方案

## 一、工厂概况分析
- 数据概览
- 痛点识别
- 优先级矩阵

## 二、解决方案（6维度）
### 2.1 生产流程优化
- 现状问题
- AI 方案
- 预期效果
- 实施措施（3-5条）

[其他5个维度...]

## 三、AI 接入架构
- 推荐方案（P0/P1/P2）
- 技术架构图
- 实施周期

## 四、效果预估
- 成本降低
- 效率提升
- ROI 测算

## 五、实施计划
- 第一阶段（1-2月）
- 第二阶段（3-6月）
- 第三阶段（6-12月）
```
