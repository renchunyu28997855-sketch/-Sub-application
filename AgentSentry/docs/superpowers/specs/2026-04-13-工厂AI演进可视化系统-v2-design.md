## 一、项目概述

### 1.1 目标
在现有数字孪生 Dashboard 基础上，新增工厂 AI 演进可视化模块，展示工厂从"全人力"逐步演进到"人机协作"的完整过程。包含双Agent架构（进化Agent + 审查Agent）、部门KPI审查机制、人机比例动态变化。

### 1.2 核心概念
- **进化Agent**：居中，发出进化工作流，推动部门AI化
- **审查Agent**：监控各部门KPI，对比初始值，审查通过后进化Agent才能工作
- **工作流**：沿部门连接线流动的光带/波纹，而非粒子
- **人机比例**：部门演进时，人形减少、机器人在节点内增加
- **磨合期颜色**：AI化程度低时橙色 → 高时绿色

## 二、系统架构

### 2.1 布局结构

```
              [审查 Agent]  ← 发出审查工作流（紫色波纹）
                    ↓
              [进化 Agent]  ← 居中，发出进化工作流（青绿波纹）
                    ↓ 沿连接线流动
    ┌─────────────────────────────────────────────┐
    │  [订单] → [设计] → [生产] → [质检]         │
    │     ↓           ↘           ↓              │
    │  [数据中枢]  [设备维护] ←──────┘            │
    │                         ↓                   │
    │  [人力] ←── [仓储] ← [采购] ← [财务]       │
    │     ↗                                      │
    │  [售后] ←──────── [人力资源]                │
    └─────────────────────────────────────────────┘
```

**视觉缩放规则**：
- 节点内人形/机器人数目按比例缩放，最大显示10个
- 例如：50人 → 显示5个火柴人；100人 → 显示10个

### 2.3 部门连接关系（业务流+数据流）

```
订单 → 设计 → 生产 → 质检
                ↓
  数据中枢 → 订单、设计、生产、质检、财务（数据支持）
                ↓
  质检 → 仓储 → 采购 → 财务 → 售后 → 订单（业务闭环）
                ↓
  设备维护 ↔ 能源管理 ↔ 生产（设备+能源支持）
                ↓
  人力资源 → 生产（人员支持）
```

## 三、Agent 设计

### 3.1 进化 Agent
- **位置**：画布中央 (x: 0.5, y: 0.45)
- **形态**：大型机器人 + 青色光环
- **职责**：发出进化工作流，推动部门AI化
- **颜色**：青色 #00d4ff + 绿色 #00ff88

### 3.2 审查 Agent
- **位置**：进化Agent上方 (x: 0.5, y: 0.2)
- **形态**：机器人 + 紫色光环
- **职责**：监控各部门KPI，对比初始值，审查通过后放行
- **颜色**：紫色 #9b59b6
- **工作流**：紫色波纹沿连接线流向各部门

### 3.3 工作流形态
- **进化工作流**：青绿渐变光带，沿连接线流动，宽度8px
- **审查工作流**：紫色光带，沿连接线流动，宽度6px
- **流动速度**：沿路径长度计算，3-5秒流完整个路径

## 四、审查机制

### 4.1 审查流程
```
审查Agent持续监控各部门KPI
           ↓
KPI优于初始值？ ─否→ 继续监控
           ↓是
发出审查工作流（紫色波纹）流向该部门
           ↓
审查工作流到达部门，部门进入"审查中"状态
           ↓
审查动画完成（波纹扩散），KPI达标确认
           ↓
进化Agent发出进化工作流（青绿波纹）
           ↓
部门开始演进（人→机器比变化，磨合期颜色变化）
```

### 4.2 审查判定
- 各部门KPI当前值对比初始值
- 达标条件：KPI优于初始值（方向性已定义）
- 达标后进入3秒审查动画
- 审查动画完成后触发演进

## 五、部门演进动画

### 5.1 人机比例可视化
- 节点大小固定，节点内人形/机器人数量按比例
- AI化进程：人减少，Agent增多
- **生产车间示例**：
  - AI率0%：显示5个火柴人
  - AI率50%：显示3个火柴人 + 2个机器人
  - AI率100%：显示1个火柴人 + 4个机器人

### 5.2 磨合期颜色
- **橙色 #ff6b35 (0-20%)**：新替代，生疏
- **黄色 #ffcc00 (20-50%)**：适应中
- **青色 #00d4ff (50-80%)**：熟练
- **绿色 #00ff88 (80-100%)**：完全融合，成熟

### 5.3 演进触发条件
- 审查通过
- 进化工作流到达部门
- 部门开始磨合期，AI率逐步提升

## 六、工作流可视化

### 6.1 进化工作流
- **颜色**：青色 #00d4ff → 绿色 #00ff88 渐变
- **宽度**：8px
- **形态**：沿连接线的光带，带有节点流动效果
- **速度**：500px/秒

### 6.2 审查工作流
- **颜色**：紫色 #9b59b6
- **宽度**：6px
- **形态**：沿连接线的波纹，带有脉冲效果
- **速度**：400px/秒

### 6.3 数据流
- **颜色**：青色 #00d4ff，半透明
- **粒子**：沿连接线来回流动的小圆点
- **数量**：每条连接线3-5个粒子

## 七、动画规范

### 7.1 工作流流动动画
```
波纹节点沿路径移动
   ↓
 波纹到达时，路径高亮
   ↓
 目标节点脉冲扩散
   ↓
 节点内人形/机器人比例变化
```

### 7.2 人形/机器人变形动画
```
演进开始
   ↓
 人形节点开始减少（每2秒减少1个）
   ↓
 机器人节点开始增加（每2秒增加1个）
   ↓
 磨合期脉冲（颜色从橙色渐变到绿色）
   ↓
 演进完成
```

### 7.3 审查动画
```
紫色波纹到达部门
   ↓
 部门节点被紫色光圈包围
   ↓
 光圈脉冲3次（模拟审查过程）
   ↓
 光圈扩散消失
   ↓
 进化工作流开始发出
```

## 八、技术实现

### 8.1 工作流类
```javascript
class WorkflowWave {
    constructor(startX, startY, endX, endY, color, speed) {
        this.startX = startX; this.startY = startY;
        this.endX = endX; this.endY = endY;
        this.progress = 0; // 0-1
        this.color = color;
        this.speed = speed; // px/秒
        this.alive = true;
    }
    update(dt) {
        const pathLength = Math.sqrt((this.endX-this.startX)**2 + (this.endY-this.startY)**2);
        this.progress += (this.speed * dt) / pathLength;
        if (this.progress >= 1) this.alive = false;
    }
    draw(ctx) {
        const x = this.startX + (this.endX - this.startX) * this.progress;
        const y = this.startY + (this.endY - this.startY) * this.progress;
        ctx.save();
        ctx.shadowColor = this.color; ctx.shadowBlur = 20;
        ctx.fillStyle = this.color;
        ctx.beginPath(); ctx.arc(x, y, 8, 0, Math.PI * 2); ctx.fill();
        // 绘制光带尾巴
        ctx.globalAlpha = 0.5;
        ctx.beginPath(); ctx.arc(x - 20, y, 5, 0, Math.PI * 2); ctx.fill();
        ctx.restore();
    }
}
```
