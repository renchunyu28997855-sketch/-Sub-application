# exam-prep

## 功能说明
为用户提供考试备考的全面指导，包括计划制定、知识点梳理、模拟训练和应试技巧。

## 假设声明（Assumptions）

### 前提条件
- 用户有明确的考试目标
- 用户有可用的备考时间
- 用户愿意按照计划执行

### 适用范围
- **国内考试**：中考、高考、考研、公务员笔试
- **语言考试**：托福、雅思、GRE、GMAT
- **专业考试**：CPA、法考、医考、工程类考试

### 不适用范围
- 需要实地操作的考试
- 完全随机的选拔考试
- 时间极短（少于1周）的紧急备考

## 详细工作流程（Detailed Workflow）

### 步骤1：信息收集
- 了解考试类型和目标分数
- 评估当前水平和可用时间
- 识别薄弱科目

### 步骤2：策略制定
- 确定总体备考策略
- 划分备考阶段
- 制定阶段目标

### 步骤3：计划细化
- 制定每周学习任务
- 安排每日学习时间
- 预留模拟考试时间

### 步骤4：技巧指导
- 传授各科应试技巧
- 提供时间分配建议
- 心理调适方法

### 检查点
- [ ] 计划切实可行
- [ ] 时间分配合理
- [ ] 重点突出
- [ ] 留有调整余地

## 边界情况（Edge Cases）

### 情况1：时间极短（少于1个月）
**处理方式**：突出重点，聚焦高频考点，调整为突击策略

### 情况2：多科同时备考
**处理方式**：合理分配时间，确保每科基础分

### 情况3：零基础备考
**处理方式**：从基础开始，分轮复习，延长总时间

### 情况4：已有较好基础
**处理方式**：侧重难点突破，模拟训练

---

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|---------|---------|---------|
| 时间不足 | 备考时间过短 | 调整策略为突击重点 |
| 目标过高 | 目标与实际差距大 | 建议设定合理目标 |
| 计划不切实际 | 任务量过大 | 压缩任务，突出重点 |
| 缺乏自律 | 用户反映执行困难 | 提供监督方法和激励机制 |

### 失败案例及修正

**输入**：
```
考试类型：考研
目标分数：400分
可用时间：200天
```

**问题**：200天时间较长，可能导致后期疲劳

**建议调整**：
```
【考试类型】考研
【目标分数】400分
【可用时间】200天
【总体策略】分阶段递进，保持状态
【调整建议】
1. 分为基础期（1-80天）、强化期（81-160天）、冲刺期（161-200天）
2. 基础期打牢基础，不要急于求成
3. 强化期大量练习，形成知识体系
4. 冲刺期模拟考试，保持最佳状态
5. 每40天安排一次调整周，避免疲劳

【易错提醒】
- 不要以为时间多就慢慢来，要有紧迫感
- 不要只做难题，基础分也很重要
- 不要忽视政治，后期要留足时间
- 不要熬夜，保持良好作息
```

## FAQ（常见问题）

**Q1：时间不够怎么办？**
A：突出重点，放弃偏难怪，聚焦高频考点。

**Q2：怎么克服拖延症？**
A：制定每日任务清单，使用番茄工作法，给自己设定奖励机制。

**Q3：模拟题什么时候开始做？**
A：一般在中后期开始，前期以打基础为主。

**Q4：考前最后一天怎么安排？**
A：轻松复习，检查考试用品，保持良好作息。
