## 多个示例（Multiple Examples）

### 典型案例

**输入**：
```
学习内容：Python中的递归
当前水平：入门
学习目标：理解递归原理，能够实现阶乘和斐波那契
```

**输出**：
```
【语言/框架】Python
【难度等级】入门→基础
【概念讲解】
递归就是函数调用自己。

想象俄罗斯套娃，每个娃娃里面还有一个更小的娃娃，直到最小的一个。

递归需要两个关键部分：
1. 基础情形（不再递归的条件）——最小的那个娃娃
2. 递归情形（调用自己）——打开娃娃后发现里面还有一个

【代码示例】
```python
# 阶乘的递归实现
def factorial(n):
    # 基础情形：n为0或1时，阶乘为1
    if n <= 1:
        return 1
    # 递归情形：n! = n * (n-1)!
    return n * factorial(n - 1)

# 斐波那契数列
def fibonacci(n):
    # 基础情形
    if n <= 1:
        return n
    # 递归情形
    return fibonacci(n - 1) + fibonacci(n - 2)
```

【运行结果】
```python
# 阶乘
factorial(5) = 5 * 4 * 3 * 2 * 1 = 120
factorial(0) = 1

# 斐波那契
fibonacci(6) = 8  # 0, 1, 1, 2, 3, 5, 8
```

【知识点拆解】
1. factorial(5) 调用过程：
   factorial(5) → 5 * factorial(4)
                    4 * factorial(3)
                      3 * factorial(2)
                        2 * factorial(1)
                          1（基础情形返回）
                        2 * 1 = 2
                      3 * 2 = 6
                    4 * 6 = 24
                  5 * 24 = 120

2. 递归深度太大会导致栈溢出（Stack Overflow）
   Python默认递归深度约1000层

【练习题】
1. 用递归实现：1+2+3+...+n
2. 用递归实现：求数组最大值
3. 用递归实现：字符串反转
4. 用递归实现：计算一个数的各位之和

【常见错误】
- 忘记基础情形 → 无限递归 → Stack Overflow
- 递归深度过大 → 内存消耗大
- 用递归做简单循环 → 效率低

【答案提示】
```python
# 练习1：求和
def sum_n(n):
    if n == 1:
        return 1
    return n + sum_n(n - 1)

# 练习2：求最大值
def max_in_list(lst):
    if len(lst) == 1:
        return lst[0]
    rest_max = max_in_list(lst[1:])
    return lst[0] if lst[0] > rest_max else rest_max
```

### 边界/异常案例

**输入**：
```
学习内容：Python中的装饰器
当前水平：进阶
学习目标：理解装饰器原理，能够自己实现装饰器
```

**输出**：
```
【语言/框架】Python
【难度等级】进阶
【概念讲解】
装饰器就是"给函数戴帽子"——在不修改原函数的情况下，给函数增加新功能。

可以理解为：
- 包装纸 = 装饰器
- 原函数 = 礼物
- 包装后的礼物 = 装饰后的函数

【代码示例】
```python
# 简单的装饰器
def my_decorator(func):
    def wrapper():
        print("函数开始执行")
        func()
        print("函数执行结束")
    return wrapper

# 使用装饰器
@my_decorator
def say_hello():
    print("Hello!")

# 这等价于：
# say_hello = my_decorator(say_hello)

# 调用
say_hello()
```

【运行结果】
```
函数开始执行
Hello!
函数执行结束
```

【知识点拆解】
1. 装饰器是一个函数
2. 接收一个函数作为参数
3. 返回一个新函数
4. 新函数包装了原函数

【实际应用】
```python
# 计时的装饰器
import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} 运行时间: {end - start:.2f}秒")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(2)
    print("完成")

slow_function()
```

【练习题】
1. 写一个装饰器，记录函数被调用了多少次
2. 写一个装饰器，检查函数参数类型

【常见错误】
- 装饰器顺序搞错（多个装饰器时）
- 忘记保留原函数的信息（使用functools.wraps）
- 装饰器带参数时结构混乱
```

### 失败案例及修正

**输入**：
```
学习内容：Python面向对象
当前水平：入门
反馈：看了很多教程但还是不懂
```

**问题**：概念太多，顺序不对

**修正输出**：
```
【语言/框架】Python
【难度等级】入门
【概念讲解顺序】
学习面向对象，不要一上来就看类和对象。要从生活实例出发！

Step 1：什么是对象？
现实世界中的对象：
- 属性：对象的特征（长什么样、什么属性）
- 方法：对象能做什么（会动、会叫）

例如：一只猫
- 属性：名字、颜色、年龄、体重
- 方法：跑、跳、吃东西、喵喵叫

Step 2：什么是类？
类就是"模板"！
- 类是创建对象的蓝图
- 对象是类的实例

例如：猫是一个类
- 你家的"旺财"是一个对象
- 我家的"小白"是另一个对象

Step 3：为什么需要类？
不用类（面向过程）：
```python
# 记录一个人的信息
name1 = "张三"
age1 = 20
score1 = 85

name2 = "李四"
age2 = 22
score2 = 92
```

用类：
```python
class Student:
    def __init__(self, name, age, score):
        self.name = name
        self.age = age
        self.score = score

    def introduce(self):
        print(f"我叫{self.name}，今年{self.age}岁")

stu1 = Student("张三", 20, 85)
stu2 = Student("李四", 22, 92)

stu1.introduce()  # 我叫张三，今年20岁
```

【理解要点】
1. 类是模板，对象是实例
2. __init__是初始化方法，创建对象时自动调用
3. self是对象的引用，指向当前对象
4. 方法就是函数，属性就是变量

【练习】
创建一个Dog类，有名字、品种属性，有叫和跑的方法
```
