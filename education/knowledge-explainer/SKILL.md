# knowledge-explainer

## 功能说明
用通俗易懂的语言解释复杂的专业概念，配合生活实例和类比，帮助用户真正理解而非死记硬背。

## 假设声明（Assumptions）

### 前提条件
- 用户希望深入理解概念，而非仅知道定义
- 用户愿意通过类比和实例来学习
- 用户具备基本的常识和生活经验

### 适用范围
- 数学：微积分、线性代数、概率统计
- 物理：相对论、量子力学、电磁学
- 计算机：算法、数据结构、网络原理
- 经济：供需关系、通货膨胀、边际效应
- 其他任何需要解释的专业概念

### 不适用范围
- 需要大量前置知识的深奥理论
- 纯技术操作类（不适合概念讲解）
- 完全主观的价值判断

重点关注：[可选，如"实际应用"、"推导过程"]
```

### 输入示例

当前理解程度：完全不了解

...

...

...
【相关概念】
...
【记忆方法】
```

## 详细工作流程（Detailed Workflow）

### 步骤1：概念识别
- 识别用户想了解的概念
- 评估用户的理解程度
- 确定讲解深度

### 步骤2：专业定义
- 给出准确的专业定义
- 解释术语的含义
- 说明适用条件

### 步骤3：通俗转化
- 用生活类比解释
- 用简单语言重新表述
- 找出常见理解误区

### 步骤4：实例讲解
- 给出具体实例
- 分析实例中的概念应用
- 展示实际应用场景

### 步骤5：延伸关联
- 介绍相关概念
- 构建知识网络
- 提供记忆方法

### 检查点
- [ ] 概念解释准确
- [ ] 类比恰当
- [ ] 实例具体
- [ ] 用户能理解

---

## 边界情况（Edge Cases）

### 情况1：概念过于专业
**处理方式**：只讲核心思想，不深究细节

### 情况2：用户完全零基础
**处理方式**：从最基础的开始讲，类比更生活化

### 情况3：相似概念混淆
**处理方式**：对比讲解，区分差异

### 情况4：概念有争议
**处理方式**：客观介绍不同观点，不做唯一判断

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|---------|---------|---------|
| 概念不存在 | 无法解释 | 说明并建议更具体的概念 |
| 类比不恰当 | 可能引起误解 | 更换类比 |
| 用户理解偏差 | 反馈理解错误 | 重新解释 |
| 超出能力范围 | 需要专业背景 | 说明并尽力解释核心 |

## 多个示例（Multiple Examples）

### 典型案例

**输入**：
```
概念名称：边际效应
当前理解程度：完全不了解
重点关注：生活中的例子
```

**输出**：
```
【概念名称】边际效应
【专业定义】在一定时间内，每增加一单位投入所带来的产出增量。
【通俗解释】
"边际"就是"多出来的那个"的意思。边际效应就是"多做一点这件事，会带来多少额外收获"。

就像吃包子，吃第一个的时候最香，第二个还行，第三个开始有点腻，第四个吃不下去了。每一个新增的包子带来的"满足感"在递减。
【生活类比】
- 喝水：第一杯很解渴，第二杯还行，第三杯喝不下
- 花钱：第一个1000块很兴奋，第10个1000块就没感觉了
- 游戏：第一次通关很激动，第10次通关就无聊了
- 加班：第1小时效率高，第5个小时效率很低
【实例讲解】
一家奶茶店，原来每小时卖50杯，现在做促销：
- 第51-60杯：每增加1杯，总利润增加10元（边际利润高）
- 第61-80杯：每增加1杯，总利润只增加3元（边际利润下降）
- 第81杯起：可能还要降价卖，边际利润可能为负

这就是为什么奶茶店不会无限接单——边际利润在下降！
【相关概念】
- 边际成本 - 多生产一件产品的额外成本
- 边际收益 - 多卖一件产品的额外收入
- 规模效应 - 规模扩大带来的效益变化
- 机会成本 - 选择A而放弃B的价值
【记忆方法】
"边际"="多出来的那一个"，边际效应="多出来的那个带来的效果"
想象你吃东西，越吃越饱，最后一口的"效果"就是边际效应
```

### 边界/异常案例

**输入**：
```
概念名称：微积分中的"极限"
当前理解程度：高中生，有点了解
重点关注：为什么需要极限
```

**输出**：
```
【概念名称】极限
【专业定义】当自变量趋近于某个值时，函数的趋势走向。
【通俗解释】
极限就是研究"越来越接近"这件事。

想象你跑步：
- 你永远到不了终点，但你可以越来越近
- 极限就是描述你"最终会停在哪儿"的工具

比如：你从1米开始跑，每次跑剩下距离的一半（1/2, 1/4, 1/8...）
你永远到不了终点，但你极限上停在0米
【为什么需要极限】
1. 处理"无穷"问题
   - 无限接近但永远不到
   - 无穷小、无穷大

2. 定义导数的基础
   - 导数就是"瞬间变化率"
   - 用极限定义

3. 计算不规则图形面积
   - 用无限多个小矩形逼近
   - 极限让计算成为可能

【实例讲解】
求曲线 y=x² 在 x=2 处的切线斜率：
- 不能直接除，因为没有"瞬间"
- 用极限：Δx 越来越小，看斜率趋近于多少
- 结果是 4

【生活类比】
- 0.999...（无限个9）= 1
- 你的身高每天长一点点，一年后高了很多——积分思想
- 照片放大到极限变成像素点——极限思想
【相关概念】
- 导数 - 变化的极限
- 积分 - 求和的极限
- 连续 - 极限值等于函数值
【记忆方法】
极限就是研究"无限接近但不到"的情况
想象追乌龟的悖论——芝诺的极限
```

### 失败案例及修正

**输入**：
```
概念名称：AI中的"神经网络"
```

**错误解释**（假设）：
```
神经网络就是模拟人脑的神经细胞工作的
```

**问题**：解释太模糊，可能让人以为真的像大脑一样

**正确解释**：
```
【概念名称】神经网络
【专业定义】由多层节点组成的计算模型，通过调整连接权重来学习数据模式。
【通俗解释】
神经网络其实没有听起来那么神秘。它本质上是一个"函数拟合器"。

想象一个生产线：
- 原材料（输入数据）
- 经过多个加工程序（隐藏层）
- 产出成品（输出结果）

每个加工程序做的就是：把收到的材料加权组合，加上偏置，然后决定输出多少
【类比】
就像做菜：
- 输入：食材
- 每层：洗菜→切菜→炒菜→调味
- 每一步都有"秘方"（权重）
- 最后出锅（输出）

训练神经网络就是：调整每个步骤的秘方，让做出来的菜越来越好
【关键点】
- 神经网络不"理解"数据
- 它只是学会了从输入到输出的映射
- 100层也不比10层"更聪明"——只是学到的模式不同
【相关概念】
- 深度学习 - 多层神经网络
- 权重 - 调整的参数
- 反向传播 - 学习算法
【记忆方法】
神经网络 = 很多层处理 + 调整参数让输出越来越好
```

## FAQ（常见问题）

**Q1：类比会不会误导理解？**
A：会，所以类比后要强调类比的局限性，指出本质区别。

**Q2：概念太抽象怎么办？**
A：用更多实例，从不同角度解释，让用户慢慢建立理解。

**Q3：可以用数学公式吗？**
A：可以，但要说明公式中每个符号的含义，不要让公式成为障碍。

**Q4：用户基础太差怎么办？**
A：从更基础的概念开始讲，不要假设任何前置知识。
