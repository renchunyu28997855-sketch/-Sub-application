# study-plan

## 功能说明
帮助用户制定科学合理的个性化学习计划，包括学科规划、时间管理和进度追踪。

## 假设声明（Assumptions）

### 前提条件
- 用户有明确的学习目标
- 用户有可用的学习时间
- 用户愿意执行计划并定期反馈

### 适用范围
- 学期学习计划
- 考前复习计划
- 技能学习计划
- 自学成长计划

### 不适用范围
- 需要系统性培训的复杂技能
- 需要导师长期指导的专业学习
- 完全零基础的陌生领域（需要先入门）

---

## 输入输出（I/O Format）

### 输入格式
```
学习目标：[具体目标，如"Python编程"、"英语口语"]
总时长：[如"3个月"、"一学期"]
每周可用时间：[如"10小时"、"每天2小时"]
当前水平：[如"零基础"、"中级"]
学习风格：[可选，如"视觉型"、"听觉型"、"动手型"]
```

### 输入示例

```
【学习风格】XXX

...

...

...

...

```

## 详细工作流程（Detailed Workflow）

### 步骤1：目标分析
- 了解学习目标和动机
- 明确学习内容的范围
- 确定优先级

### 步骤2：水平评估
- 评估现有水平
- 识别基础和薄弱环节
- 确定学习起点

### 步骤3：计划制定
- 确定学习重点和难点
- 制定阶段性目标
- 安排每周学习任务

### 步骤4：时间规划
- 安排每日学习时段
- 平衡各科时间分配
- 预留休息和调整时间

### 步骤5：反馈机制
- 设置进度检查点
- 建立调整策略
- 提供激励建议

### 检查点
- [ ] 目标明确具体
- [ ] 计划切实可行
- [ ] 时间分配合理
- [ ] 有检查和调整机制

---

## 边界情况（Edge Cases）

### 情况1：时间零散
**处理方式**：化整为零，利用碎片时间学习

### 情况2：多任务并行
**处理方式**：确定主次任务，合理分配精力

### 情况3：学习瓶颈
**处理方式**：分析原因，调整方法，增加练习

### 情况4：计划执行偏差
**处理方式**：灵活调整，保持灵活性

---

## 错误处理（Error Handling）

| 错误类型 | 识别方式 | 处理方式 |
|---------|---------|---------|
| 目标不明确 | 学习方向模糊 | 帮助明确具体目标 |
| 任务过重 | 执行困难 | 压缩任务，突出重点 |
| 时间不足 | 无法完成 | 调整计划，降低预期 |
| 执行力差 | 总拖延 | 提供监督和激励方法 |

## 多个示例（Multiple Examples）

### 典型案例

**输入**：
```
学习目标：Python编程
总时长：3个月
每周可用时间：10小时
当前水平：零基础
学习风格：动手型
```

**输出**：
```
【学习目标】Python编程入门到独立做项目
【总时长】3个月，每周10小时
【学习风格】动手型（适合边做边学）
【总体方案】
- 第1个月：Python基础语法、数据类型、函数
- 第2个月：面向对象、文件操作、常用库
- 第3个月：项目实战、代码规范、版本控制

【周计划示例】
第1-4周：Python基础

第1周：环境搭建和变量
- 周一：安装Python和VS Code
- 周二：变量和数据类型
- 周三：运算符
- 周四：动手练习（计算器程序）
- 周五：复习总结
- 周末：休息

第2周：控制流
- 条件语句
- 循环语句
- 列表基础
- 实战练习

第3周：函数
- 函数定义和调用
- 参数传递
- 返回值
- 匿名函数

第4周：综合练习
- 小项目：学生成绩管理系统
- 复习本周内容
- 阶段测试

【每日学习表】（以周末为例）
09:00-10:30 视频学习 + 笔记
10:45-12:00 编码练习
14:00-15:30 项目实战
15:45-16:30 总结复盘

【进度检查点】
- 第4周：完成基础语法小测试（正确率>80%）
- 第8周：完成面向对象编程作业
- 第12周：完成期末项目

【调整策略】
1. 如果某周任务完不成，将剩余任务分配到后面几周
2. 如果觉得进度太慢，跳过已经掌握的内容
3. 如果遇到难点，可以先跳过，后面再回头看
4. 每两周回顾一次，根据反馈调整计划

【学习资源推荐】
- 视频：B站小甲鱼Python教程
- 书籍：《Python编程从入门到实践》
- 练习网站：LeetCode简单题、Hackerrank
```

### 边界/异常案例

**输入**：
```
学习目标：英语口语
总时长：6个月
每周可用时间：5小时（非常零散）
当前水平：初中级
```

**输出**：
```
【学习目标】英语口语提升
【总时长】6个月，每周5小时
【学习风格】待评估
【总体方案】
- 第1-2个月：发音纠正、基础口语表达
- 第3-4个月：日常对话、场景练习
- 第5-6个月：流利表达、复杂话题

【零散时间利用策略】
1. 通勤时间（30分钟/天）
   - 听力材料泛听
   - 单词背诵App

2. 午休时间（20分钟）
   - 跟读练习
   - 口语短对话

3. 睡前时间（20分钟）
   - 复习当天所学
   - 听力输入

4. 周末（2-3小时）
   - 系统学习
   - 口语实战

【周计划示例】
周一：通勤听力 + 单词
周二：跟读练习 + 日常用语
周三：通勤听力 + 场景对话
周四：跟读练习 + 复习
周五：周末内容预习
周末：系统学习1小时 + 实战练习1小时

【进度检查点】
- 第2个月：发音有明显改善
- 第4个月：可以进行日常对话
- 第6个月：能够表达复杂观点

【调整策略】
如果某周无法完成计划：
1. 优先保证每天10分钟的跟读
2. 周末时间可以补上工作日的缺失
3. 如果连续两周无法完成，考虑减少任务量
```

### 失败案例及修正

**输入**：
```
学习目标：考研数学
总时长：8个月
每周可用时间：20小时
当前水平：较差
```

**问题**：计划过于笼统，没有具体执行方案

**修正输出**：
```
【学习目标】考研数学（数一）达到120分
【总时长】8个月，每周20小时
【总体方案】
第一阶段（1-2月）：基础巩固
- 高等数学上册
- 线性代数基础
- 概率论基础

第二阶段（3-5月）：强化提升
- 高等数学下册
- 线代强化
- 概率强化
- 大量做题

第三阶段（6-7月）：专题突破
- 针对薄弱章节
- 总结解题方法
- 建立知识体系

第四阶段（8月）：冲刺模拟
- 真题练习
- 模拟考试
- 查漏补缺

【具体周计划示例】（第一阶段第3周）
周一：高等数学 - 函数极限（2小时）
周二：线性代数 - 行列式（2小时）
周三：高等数学 - 连续性（2小时）
周四：概率论 - 随机事件（2小时）
周五：错题整理（2小时）
周六：本周内容复习 + 练习（4小时）
周日：休息

【每日任务清单】
早上30分钟：复习前一天知识点
下午1小时：新内容学习
晚上30分钟：做练习题

【易错提醒】
- 不要只看不练，数学必须大量做题
- 不要忽视基础，基础题也要做
- 不要追求速度，确保每道题真正理解
- 错题本必须坚持，记录和复习同样重要
```

## FAQ（常见问题）

**Q1：计划执行不下去怎么办？**
A：分析原因，是任务太重还是时间不够，然后调整计划。

**Q2：如何克服拖延？**
A：使用番茄工作法，设定小目标，给自己奖励。

**Q3：学习效果怎么评估？**
A：通过定期测试、作业完成情况、自我感觉评估。

**Q4：计划需要调整吗？**
A：每两周回顾一次，根据执行情况和效果调整。
